## О данном руководстве программиста EME-БД

Данное руководство предназначено для программистов, которые собираются разрабатывать прикладные системы на основе базы данных ЕМЕ-БД. Руководство не является полным и исчерпывающим описанием всех функций, возможностей и правил ЕМЕ-БД — база данных постоянно развивается (как ядро, так и метапроект) и накопила столь богатый объём программных кодов и функций, что исчерпывающее описание потребовало бы десятков человеко-лет.

Руководство решает следующие задачи:

- Объясняет, с чего следует начинать разработку нового проекта.
- Знакомит с программными языками, применяемыми в разработке прикладных и системных («ядерных») программ.
- Приводит пример разработки простейшего проекта.
- Перечисляет доступные функции и механизмы.

в языке EME-L программисты могут найти ответы на дополнительные вопросы на сайте www.eme.ru.

## История развития ЕМЕ-БД

### Эпоха ДОС — зарождение реляционной базы данных

База данных ЕМЕ-БД возникла в 1991 году как побочный продукт при разработке прикладного проекта автоматизации работы предприятия. С самого начала это — реляционная база данных с прямыми ссылками и дублированием файлов на рабочих станциях. ЕМЕ-БД не является файловой системой прикладного проекта — это самостоятельное изделие, хотя база данных оформлена как встроенный в проект код. Все прикладные проекты надстраиваются над ЕМЕ-БД.

в системе EME.WMS реляционная архитектура с прямыми ссылками обеспечивает высокую скорость доступа к данным без необходимости индексного поиска.

### Переход на Windows — MFC и эмуляция ДОС

Во времена ДОС система собиралась при помощи компилятора Watcom. В 1998 году в ЕМЕ-БД был проинтегрирован интерфейс MFC, все проекты стали «виндоусными». Ядерный и прикладной код был переведён под компилятор Visual Studio. Для плавной миграции досовых проектов в ядро был встроен интерфейс досовых функций и эмулятор EMEDOS с отображением досового экрана в консольном окне. Это позволило на уровне исходных кодов, без перепрограммирования, перевести досовые проекты под Windows.

В досовую эпоху исходный код прикладных проектов хранился в разных директориях, а ядро было общим. Это решало проблему взаимного влияния проектов, но затрудняло перенос идей между проектами и повторное использование кода.

### Концепция Метапроекта — единое пространство имён

Для решения проблемы повторного использования кода одновременно с переходом под Windows была разработана концепция Метапроекта. Идея заключалась в применении одинаковых названий записей и полей базы данных (таблиц и столбцов) во всех прикладных проектах, когда эти записи и поля описывают одни и те же категории реального мира. Система имён таблиц, диалогов, полей диалогов, отчётов и ячеек отчётов создавала логическое информационное пространство для всех проектов фирмы ЕМЕ.

Прикладные программы метапроекта были оторваны от пользовательского интерфейса — «чистые алгоритмы», которые можно было применить где угодно, если они могли найти по именам нужные записи, поля, диалоги и ячейки.

в языке EME-L данная идея не оправдала себя по двум причинам: во-первых, многообразие прикладных задач столь велико, что вероятность повторного использования одного и того же алгоритма без изменений очень мала; во-вторых, программисты неохотно документировали метафункции и предпочитали писать свои новые функции. В настоящее время метапроект представляет собой .cpp (Си++) расширение емельных программ.

### Многозадачность и язык EME-L

В 2001 году был реализован механизм многозадачной виртуальной памяти, что позволило реализовать многопоточную схему работы как сервера, так и рабочих станций. К этому времени был разработан собственный язык программирования проектов ЕМЕ — EME-L (EME-Language, «ЕМЕля»). Программы на этом языке хранятся непосредственно в базе данных проекта и не требуют применения внешнего компилятора (компилятор работает «на лету», незаметно для программиста). Также был разработан интерпретатор языка SQL — в усеченном по отношению к стандарту варианте с одной стороны и расширенном с другой (добавлена возможность параметризации вызовов).

в системе EME.WMS сложилась современная архитектура: встроенный язык EME-L, SQL-интерпретатор, многозадачная виртуальная память.

### HTTP-сервер и «Тонкий клиент»

В 2001 году был реализован полноформатный HTTP-сервер, полностью проинтегрированный в базу данных. Это позволяет реализовывать проекты с «тонким клиентом». В режиме тонкого клиента организуется работа терминалов сбора данных с Wi-Fi-точкой доступа.

### Разделы в ядре 50 — мастер данные и данные раздела

В ядре 50 все данные в БД делятся на две группы: мастер данные и данные раздела. Мастер данные во всех разделах одинаковые, данные раздела во всех разделах разные. Системные записи всегда являются мастер данными. Все разделы делятся на две группы: раздел 0 называется «мастер раздел», остальные разделы — «рабочие разделы».

в EME БД механизм разделов позволяет изолировать данные различных бизнес-подразделений при сохранении общих справочников.

## Изделие «Базовая версия EME.WMS» — состав и компоненты

Изделие «Базовая версия EME.WMS» состоит из следующих компонентов:

| Компонент | Описание |
|-----------|----------|
| Инструкция по развёртыванию | Пошаговое руководство по установке изделия |
| Сервер WMS (Windows/Linux) | Серверное ПО базы данных EME |
| Приложение для ТСД (Android) | Два apk-файла: для физических устройств и эмуляторов |
| Монитор Заданий | Десктопная версия (Windows/Linux) и мобильная версия (Android-планшеты) |
| Apache Superset | Отчётная среда с аналитической БД PostgreSQL, настройки стилей ЕМЕ и отчётный модуль склада |
| Инструкции по вспомогательному ПО | Установка Docker, Java |
| Рабочая станция EME.WMS | Опциональный компонент |
| Обменная БД PostgreSQL | Для интеграции WMS с ERP заказчика или ERP поклажедателя (3PL-склад) |
| Хэш-БД (опционально) | Для работы с марками ЕГАИС и Честный знак |
| Веб-портал 3PL (опционально) | Обработка заявок поклажедателей: поставка, отгрузка, бронирование таймслотов |
| Книга (инженерная записка) | Документация к Базовой версии, адаптированная под Емельяна |
| Сервер ИИ (Linux) | Серверное ПО для работы искусственного интеллекта на складе |
| Емельян | ИИ компании EME: веб-интерфейс Нейроконсультанта и Монитор Заданий |
| Веб-версия Монитора Заданий | Опциональный компонент |
| Модуль «Дашборды» | Опциональный компонент |

в системе EME.WMS сервер ИИ не является предустановленным компонентом — если он не развёрнут, изделие работает стабильно, но без функционала ИИ.

## Методология формирования релиза EME.WMS

Релиз всего изделия состоит из релизов его компонентов. Руководитель проекта использует соответствующие ссылки на получение релизов всех компонентов.

**Источники релизов компонентов:**

- **GitLab** — ссылки на релизы компонентов необходимо получить на соответствующих релизных страницах GitLab (потребуется доступ к репозиторию).
- **SVN / внутренняя система контроля версий** — ссылки на релизы компонентов необходимо получить у программиста Базовой версии.
- **Инструкция по развёртыванию** — передаётся программистом команды Базовой версии руководителю проекта лично с пояснениями.

в системе EME.WMS для отдела продаж формируется рабочий стенд с текущим релизом Базовой версии, что позволяет демонстрировать функционал потенциальным заказчикам.
