# Создание срезов данных и уроков в EME.WMS

> **Поисковые синонимы:** срез данных, урок EME, демонстрационный сценарий, обучающий сценарий, data slice, подготовка данных, срез урока, Монитор Заданий, бизнес-процесс урок, обучение WMS, события процессов, параметры шаблона процесса, очистка БД, раздвоитель ключей, EME.WMS lesson, data slice creation

## Общие сведения

В системе EME.WMS реализована возможность создавать **обучающие (демонстрационные) сценарии** для прохождения складских бизнес-процессов без долгой настройки рабочей станции. Такие сценарии называются **уроками**, а калиброванные наборы данных для них — **срезами**.

Уроки доступны к прохождению в **монопольном режиме** со следующими ключами: `/bc` (мозговой компьютер), `/http` (http-сервер) и `/d_dataslice`.

**Важно:** срез данных необходимо опубликовать на центральном сервере, поэтому база данных должна быть запущена в режиме **FullVM**. В моно-режиме создать новый срез невозможно.

в системе EME.WMS срезы позволяют быстро разворачивать демонстрационные среды для тестирования и обучения складским бизнес-процессам.

---

## Пошаговая инструкция создания срезов

1. **Подготовить данные** для среза (архив состояния ЕМЕ БД) через диалог «Очистка БД».
2. **Зарегистрировать срез** на центральном сервере через диалог «Срезы данных».
3. **Создать урок** через диалог «Описание бизнес-процессов».
4. **Настроить урок** для работы с Монитором Заданий через диалоги «Параметры шаблона процесса» и «События процессов».

---

## Подготовка данных для среза

Создание среза всегда начинается с подготовки данных. Как правило, необходимо скорректировать список товаров, добавить документы, скрыть лишние процессы и провести дополнительную настройку системы.

Часто удобно создавать срез на основе другого (простого) среза — это сокращает время на настройку склада и уточнение топологии.

в EME БД подготовка данных среза включает раздвоение ключей для предотвращения конфликтов при загрузке и использовании нескольких срезов.

### Порядок действий

1. **Раздвоение ключей:**
   - Открыть раздел «Структура Базы Данных» (Ctrl+Shift+S).
   - Нажать кнопку «Раздвоитель ключей» и подтвердить выбор.

2. **Сохранение состояния БД:**
   - Перейти: «Утилиты БД → СИСТЕМА → Очистка БД».
   - В выпадающем списке выбрать «Все несистемные записи».
   - Нажать «Экспорт данных».

3. **Сохранение архива:**
   - Сохранить zip-файл с данными среза.
   - **Важно:** в пути к папке и её названии не должно быть пробелов — это мешает загрузке данных из архивов.

## Регистрация среза на центральном сервере

Для привязки данных к уроку необходимо создать срез через диалог **«Срезы данных»**: «Утилиты БД → СИСТЕМА → Срезы данных».

### Порядок действий

1. Нажать кнопку **«Новый»** — добавить объект в список срезов.
2. Заполнить **название среза**.
3. В поле «Тип» указать **«Срез урока»**.
4. Нажать **«Загрузить»** — выбрать архив с данными состояния БД.
5. **Сохранить** данные в диалоге.

### Кнопки диалога «Срезы данных»

| Кнопка | Описание |
|--------|----------|
| **Загрузить** | Выбор архива, данные которого загружаются в срез. |
| **Обновить** | Создаёт данные из текущего состояния БД, подключается к серверу и обновляет срез. **Не рекомендуется** — при некорректном состоянии данных прошлое состояние будет потеряно. |
| **Выгрузить** | Извлечение архива среза из системы. Полезно для резервного копирования. |

в языке EME-L срезы данных хранятся в специализированных записях БД и управляются серверными механизмами репликации для доставки на рабочие станции.

## Создание урока

Создание и настройка уроков выполняется в режиме рабочей станции (залить новые процессы через модуль заливки невозможно).

Диалог: **«Описание бизнес-процессов»** → вкладка **«Учебники»**.

### Списки уроков

| Список | Описание |
|--------|----------|
| **Конструируется** | Уроки, разработка которых ещё не завершена. |
| **Самостоятельное знакомство** | Уроки, полностью реализующие функционал складского бизнес-процесса. |

### Порядок действий

1. Выбрать нужный список.
2. Выбрать любой существующий урок, пролистать вниз.
3. Нажать **«Создать урок»**.
4. Выбрать созданный урок, правая кнопка мыши → **«Параметры»**.
5. В открывшемся диалоге привязать срез через пункт **«Срез урока»**.
6. **Сохранить** данные в диалоге.

## Настройка урока для работы с Монитором Заданий

При загрузке урока система проверяет разрешённые вечные задания, снимает с них признак «не используется» и проставляет его для остальных заданий — это обеспечивает чистый вид Монитора Заданий.

### Порядок действий

1. В диалоге **«Параметры шаблона процесса»** нажать **«Новое событие»**.
2. Выбрать одно из событий процессов.
3. Нажать **«Применить»** для сохранения.

### Определение события для задания

События связаны с заданиями и часто содержат название задания в своём имени, но это правило иногда нарушается.

Для точного определения события:
- Открыть диалог **«События процессов»**: «Классификаторы → БИЗНЕС-ПРОЦЕССЫ → События процессов».
- Найти привязку нужного события к заданию.

в EME БД корректная привязка событий процессов к заданиям урока гарантирует, что Монитор Заданий отобразит только релевантные задания для конкретного обучающего сценария.
