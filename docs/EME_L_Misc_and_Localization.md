## Тест интерфейса диалогов в ЕМЕ БД

В EME БД имеется модуль, проверяющий работоспособность всех диалогов путём их последовательного открытия. Запускается через "Система -> Ресурсы -> Тестирование -> Тест интерфейса".

Проверка всех диалогов занимает примерно три минуты. Во время теста не нажимаются кнопки и не закрываются окна сообщений, поэтому программа должна пройти тест без ручного вмешательства. Режим тестирования проверяется функцией `is_ui_test_mode()`.

### Использование is_ui_test_mode()

```EME-L
Dialog_OnLoad()
{
    If (ContextFindParamsRec.GetNoOfLines() == 0 & ~is_ui_test_mode())
        If (is_message(tr("Внимание"), tr("Создать настройки поиска?"), "YESNO", "QUESTION") == "YES")
            CreateDefault();
        End If
    End If
}
```

в языке EME-L функция `is_ui_test_mode()` позволяет отличать обычный запуск диалога от тестового, чтобы избежать блокировки теста интерактивными сообщениями.

### Проверка потерянных полей

Тест также выполняет поиск полей, использующих CEMERec классы, но отсутствующих в базе. Проверка реализована в EME-L классе `Error` (метод `CheckLostFields()`) с применением функции `is_lost_fields()`. Если проверяемые поля не должны быть в проекте — они добавляются в исключения.

Тест считается пройденным, если после его выполнения не осталось открытых диалогов и сообщений об ошибках.

в системе EME.WMS тест интерфейса является быстрым способом проверки целостности диалогов после внесения изменений в структуру БД.

## Роботы-тестировщики в ЕМЕ БД

Робот-тестировщик запускается через "Система -> Ресурсы -> Тестирование -> Робот". База данных должна быть запущена в моно-режиме.

В ЕМЕ БД реализованы тесты для:

- **Классификаторов:** склады, сектора, товары, транзакции
- **Операций:** приход, отгрузка, перемещение, изменение характеристик, списание

Некоторые роботы имеют зависимость от других и требуют строгой последовательности выполнения. Роботы могут явно вызывать друг друга. Для передачи параметров используются контейнеры класса `CRobotDataContainer`.

### Добавление нового теста робота

Для добавления нового диалога для тестирования роботом:

1. Создать новый класс
2. Реализовать метод `RobotBatchGo_RST(Robot AS "CIRobot")`
3. Прописать класс и метод в конструкторе класса `AllRobots`

Требования к классам:

- Класс тестирования классификаторов (ячейки, сектора) наследуется от `GuideRobots`
- Класс тестирования диалогов складских операций наследуется от `OperationRobots`
- Класс тестирования диалога отчётов наследуется от `ReportsRobots`

### Запуск тестирования роботом

После открытия диалога робота:

1. Поставить галочку "Останавливать выполнение, если шаг не выполнен"
2. Выставить пошаговую задержку 0 мс
3. Выставить "Ожидание до" 50 секунд
4. Выбрать имя класса робота (AllRobots — тестирование всех)
5. Нажать "Старт"

При обнаружении ошибки выполнение остановится. Прохождение теста всеми роботами занимает 10 минут.

в системе EME.WMS роботы-тестировщики обеспечивают автоматизированную проверку основного функционала складских операций.

## Передача аргументов в EME-L по ссылке

Одно из ограничений EME-L — единственное возвращаемое значение из методов. Это решено нововведениями:

- Новый тип данных "Ссылка"
- Операторы `ByRef` и `ByVal` — при определении переменной
- Операторы `RefOf` и `ValOf` — при обращении к переменной

### Сценарии использования ссылок

1. Ссылка формируется при определении метода:
```EME-L
MyMethod(MyReference ByRef)
```

2. Передача значения по ссылке:
```EME-L
Result = MyMethod(RefOf MyValue)
```

3. Получение значения по ссылке:
```EME-L
Check = ValOf MyReference > 0
```

4. Изменение значения по ссылке:
```EME-L
MyReference ByVal = 5
```

в языке EME-L передача по ссылке позволяет методам возвращать несколько значений и является важным расширением возможностей языка.

## Функция tr() для перевода интерфейса

Функция `tr()` необходима для перевода строк с русского на иностранный язык. Принимает один параметр — строку текста. Все строки, отображаемые в интерфейсе на русском, должны быть обёрнуты в `tr()`. Программа переводит русский текст транслитом, если перевода нет.

Язык системы проверяется при запуске. Для принудительного тестирования английского языка используется ключ запуска `/ENG`.

### Правильное использование

```EME-L
is_format(tr("На складе %s включена инвентаризация"), m_r_Warehouse.GetCode());
```

### Частые ошибки

```EME-L
'Неправильно — конкатенация внутри tr():'
tr("На складе " + m_r_Warehouse.GetCode() + " включена инвентаризация");

'Неправильно — раздельный перевод частей строки:'
tr("На складе ") + tr(" включена инвентаризация");

'Неправильно — внутренние идентификаторы не переводятся:'
GetCollectionMap(tr("wh_" + m_r_Warehouse.GetLine()), 103);

'Неправильно — нет человеко-читаемого текста:'
is_message(tr("!"), csText, 1);

'Неправильно — названия методов и классов не переводятся:'
is_message(tr("ERPUtilities::RepairRemainsQty"), Error, "OK", "STOP");
```

в системе EME.WMS функция tr() является основным средством интернационализации интерфейса.

## Механизм перевода рабочей станции EME.WMS

Перевод хранится в файле `kernel.x64\Exe\Resource\ENG.txt` вместе с оригинальными репликами на русском. Файл в кодировке UTF16, выравнивается с сервера на все рабочие станции.

Структура файла:

- Код контекста (тег "root") — используется при поиске: EMEL, CXX и т.д.
- Строковый контекст (тег "child") — для быстрого поиска в исходном коде
- Исходная строка (тег "text")
- Перевод (тег "translation")

Файл передаётся в группу ИИ для перевода языковой моделью, затем обработанный файл сохраняется в SVN ядра.

## Утилиты для перевода (класс LocalizationUtilities)

Утилиты для извлечения строк в файл перевода находятся в EME-L классе `LocalizationUtilities`:

- `TranslateContextFindParams()` — строки записи "Параметры контекстного поиска", "Типы документов", "Статусы кода продукта"
- `GetTextFromUserMenu()` — тексты из верхнего меню WMS
- `TranslateAllSystemDialogs()` — тексты из rc-файлов ядра
- `GetTextFromTREMEL()` — тексты из исходного кода EME-L, обёрнутые в `tr()`
- `GetTextFromTRKernel()` — тексты из исходного кода ядра
- `GetTextFromTRMetaproj()` — тексты из метапроекта
- `GetTextFromReports()` — тексты из шаблонов отчётов
- `TranslateBrowsers()` — тексты из шаблонов браузеров
- `TranslateSettings()` — тексты из настроек модулей и рабочей станции
- `TranslateDBNames()` — тексты имён записей, полей и атрибутов БД
- `TranslateDBRUS()` — тексты из сообщений ядра БД (Db.rus)
- `TranslateDialogs()` — тексты из шаблонов диалогов
- `TranslateProcessTemplates()` — тексты из Монитора Заданий

в EME БД утилиты класса LocalizationUtilities обеспечивают полный охват всех компонентов системы при подготовке файлов для перевода.
