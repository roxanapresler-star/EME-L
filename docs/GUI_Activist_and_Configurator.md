## Диалог «Конфигуратор»

В ЕМЕ БД имеется механизм описания конфигураций программных модулей, реализованный в диалоге **«Конфигуратор»** — интерактивном инструменте для определения программных компонентов модуля и их соответствия общей структуре БД.

Доступ: **«Система → Ресурсы → Конфигуратор»**.

в языке EME-L конфигуратор обеспечивает связь между программными компонентами (EME-L классами, диалогами, запросами) и модулями системы, формируя единую карту проекта.

### Структура диалога

- **Левая часть** — интегратор с деревом вложенности модулей. При раскрытии дерева до названия модуля можно быстро перейти в редактор компонентов через кнопку **«Открыть»**.
- **Таблица конфигурации** — для каждого компонента указывается:
  - **Тип**: Запись, Диалог, EME-L класс, Запрос.
  - **Название** компонента.
- Над таблицей отображаются **автор конфигурации** и **дата/время формирования**.

### Управление конфигурациями

| Действие | Описание |
|----------|----------|
| **Добавить на текущем уровне** | Создание новой конфигурации на текущем уровне дерева. |
| **Создать подуровень** | Создание вложенного уровня конфигурации. |
| **Редактирование** | Выбор значений из выпадающего списка в колонке «Тип». |
| **Экспорт/Импорт** | Перенос конфигураций в другие проекты. |

в системе EME.WMS конфигуратор позволяет структурировать все программные компоненты модуля, упрощая навигацию и управление зависимостями при командной разработке.

## EME-L классы для работы с модулем «Активист»

Центральным классом для модуля «Активист» является **`ActivistTasks`**. Все обработчики активностей должны быть наследниками этого класса — иначе их нельзя будет выбрать в диалоге рабочей станции.

в языке EME-L наследование от ActivistTasks обязательно для всех классов-обработчиков активностей, поскольку базовый класс реализует интерфейс взаимодействия с подсистемой мониторинга и диспетчеризации задач.

### Иерархия классов-обработчиков

```
ActivistTasks (базовый класс)
├── ActivistDialog
├── ActivityIncomingGoodsForgotten
├── ActivityOutcomeRegularMissing
├── ActivityTemplate
└── ActivityWDCMeasurement
```

### Описание классов

#### ActivistDialog

Класс-обработчик диалога для работы с задачами Активиста. Обеспечивает интерфейсное взаимодействие с подсистемой задач в диалоге рабочей станции.

#### ActivityIncomingGoodsForgotten

- **Назначение:** активность «Простой грузовых мест в приёмке (зависшие размещения)».
- **Интервал простоя по умолчанию:** 30 минут.
- **Код активности:** `IncomingGoodsForgotten`.
- **Логика:** проверяет, что паллеты не задерживаются в зоне приёмки — у них есть приказ на размещение и этот приказ создан не слишком давно.

в EME БД данная активность предотвращает образование «зависших» placements, автоматически выявляя грузовые места, находящиеся в зоне приёмки дольше допустимого времени.

#### ActivityOutcomeRegularMissing

- **Назначение:** активность «Анализ частых недостач».
- **Интервал анализа недостач по умолчанию:** 7 дней.
- **Код активности:** `OutcomeRegularMissing`.
- **Логика:** проверяет повторяющиеся ситуации с недостачей за период — выявление хищений или некорректного использования системы.
- **Триггеры:**
  - По ячейке.
  - По сотруднику.
  - По товару.

Каждый триггер поддерживает два режима работы в зависимости от параметра `bCheckOnly`:
- `bCheckOnly = True` — не заполнять список, вернуть результат проверки наличия.
- `bCheckOnly = False` — заполнить запрос или вернуть результирующее значение.

в языке EME-L триггеры активностей позволяют реализовать гибкую логику анализа данных, работая как в режиме проверки, так и в режиме формирования задач.

#### ActivityTemplate

Класс-шаблон, созданный для ускорения разработки новых активностей. Используется как основа при создании собственных обработчиков.

#### ActivityWDCMeasurement

- **Назначение:** активность «Измерение ВГХ» (весогабаритных характеристик).
- **Интервал создания активности по умолчанию:** 0.
- **Код активности:** `WDCMeasurement`.
- **Логика:** формирует задание в Мониторе Заданий для распределения приказа на отложенное измерение весогабаритных характеристик товара.

в системе EME.WMS автоматизация измерения ВГХ через модуль Активист обеспечивает своевременный сбор весогабаритных данных для корректной работы алгоритмов размещения и маршрутизации.
