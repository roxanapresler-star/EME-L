# Редактор диалогов — настройки, таблицы и органы диалогов в ЕМЕ БД

> **Поисковые синонимы:** редактор диалогов, свойства диалога, настройка диалогов, таблицы диалогов, органы диалогов, привязка к БД, броузер диалога, EME-L класс в диалоге, DLL метапроекта, реляционный путь, столбцы таблицы, диалог EME, dialog editor, dialog properties, table settings, control settings

---

## Открытие редактора диалогов

Для настройки интерфейса диалогов необходимо открыть редактор диалогов через пункты меню **«Система → Ресурсы → Редактор диалогов»**. Затем выбрать **«Файл → Открыть диалог»** и в броузере указать требуемый диалог.

Для перехода к настройкам диалога нужно нажать на левый верхний угол окна диалога левой кнопкой мыши — откроется окно **«Свойства»**. Основные настройки расположены на вкладке **«Свойства диалога»**.

в языке EME-L диалоги являются основным средством взаимодействия пользователя с данными БД, а привязка EME-L классов к диалогам обеспечивает программную логику обработки событий.

---

## Описание общих настроек интерфейса диалога

### Свойства окна

| Настройка | Описание |
|-----------|----------|
| **Софтовое имя** | Имя диалога для обращения в программе. Только латинские буквы. |
| **Заголовок диалога** | Имя диалога, отображаемое в левом верхнем углу. |
| **Шаг выравнивания органов управления** | Размер сетки в интерфейсе редактора диалогов. |
| **Номер раздела справки** | Настройка не используется. |
| **URL справки** | Настройка не используется. |

### Настройки интерфейса

| Настройка | Описание |
|-----------|----------|
| **Базовая запись диалога** | Указывается запись БД, к которой привязан диалог. |
| **Поле сортировки** | Поле записи с атрибутом «поле-сортировка». Если отсутствует — выставляется «Сортировка по физике». |
| **Не привязывать диалог к записи БД** | Отвязывает диалог от записи БД. |
| **Основной DLL диалога** | DLL метапроекта, используемая диалогом. Кнопка «<= EMEL» привязывает EME-L класс. |
| **Основная функция диалога** | Функция из DLL, привязанная к диалогу. Выбор через кнопку «…». |
| **Параметры** | Параметры функции: «Класс», «Ссылка», «Область видимости объекта», «Видеть диалог». «Класс» и «Ссылка» заполняются автоматически от привязанного EME-L класса. |
| **Не удалять** | Убирает иконку удаления с панели. |
| **Не добавлять** | Убирает иконку добавления с панели. |
| **Не листать** | Убирает иконки пролистывания с панели. |
| **Выводить броузер** | Открывает диалог в формате броузера. |
| **Работать с отчётами** | Активирует кнопку «Вывести отчёт» на панели. |
| **Пакетная печать** | Настройка не используется. |
| **Пос-ть сооб. после печати** | Настройка не используется. |
| **Выводить справку** | Флаг не редактируется. |
| **Запрет дублирования диалогов** | Запрещает одновременное открытие двух экземпляров диалога. |
| **Не выводить панель инструментов** | Скрывает верхнюю панель инструментов. |
| **Создавать новую строку при старте** | Диалог открывается с пустыми значениями параметров органов. |
| **Проверять отпечаток** | Настройка не используется. |
| **Не менять владельца** | Настройка не используется. |
| **Защищать информацию от одновременного доступа серверным семафором** | Активирует механизм защиты от одновременного доступа к общим ресурсам. |
| **Отправлять почту** | Настройка не используется. |
| **TopLevelFlag** | Настройка не используется. |
| **Запретить скриншот диалога** | Запрещает снимок окна диалога через Ctrl+W. |

в системе EME.WMS настройки интерфейса диалога определяют доступные пользователю операции: добавление, удаление, навигация и печать — а также поведение диалога при старте.

## Дополнительные настройки параметров диалога

| Настройка | Описание |
|-----------|----------|
| **Дополнительные функции диалога** | Цепочка из до 4 DLL с EME-L классами, именами функций и параметрами. Можно менять порядок. |
| **Броузеры диалога** | Формирование перечня броузеров. Добавление — кнопка «←», удаление — кнопка «→». Доступно редактирование свойств, базовой записи и поля сортировки. |
| **Отчёты диалога** | Формирование перечня отчётов. Добавление и удаление аналогично броузерам. |
| **Переменные диалога** | Создание и редактирование переменных, сформированных в диалоге. |
| **Основной Броузер >>** | Привязка броузера к диалогу (сделать основным). Три пункта: «привязать», «редактировать», «отвязать». |
| **Заливка** | Цвет фона диалога. |
| **Шрифт** | Шрифт текста диалога. |
| **Ширина** | Ширина диалога в пикселях. |
| **Высота** | Высота диалога в пикселях. |

в EME БД дополнительные функции позволяют создавать цепочки обработчиков, где каждый EME-L класс последовательно обрабатывает события диалога.

## Настройки таблиц диалогов

Для редактирования таблицы: открыть редактор диалогов, выбрать диалог, выбрать таблицу, нажать правой кнопкой мыши → «Редактировать». Все настройки — на вкладке **«Настройка таблицы»**.

Имя таблицы — софтовое имя (латинские буквы) для обращения в программе.

### Флаги таблицы

| Флаг | Описание |
|------|----------|
| **Удалять сразу из БД** | Удаление из таблицы автоматически удаляет данные из БД. Программа блокирует удаление при наличии ссылок. |
| **Запретить добавление и удаление** | Убирает возможность добавлять/удалять строки. |
| **Скрытая таблица** | Настройка никогда не активна. |
| **Столбец нумерации** | Активирует столбец нумерации в левой части таблицы. |
| **Доступна для всех филиалов** | Настройка не используется. |
| **Не сортируется пользователем** | Отключает сортировку столбцов пользователем. |
| **Невидимая** | Скрывает таблицу с диалога. |
| **Не закрашивать фокусную ячейку** | Отключает подсветку активной ячейки. |
| **Рамка вокруг окна** | Отключает обводящую рамку. |
| **Только для чтения** | Запрещает изменение данных. |
| **Не затенять недоступные ячейки** | Отключает затенение нередактируемых ячеек. |
| **Не отображать пустые значения** | Пустые ячейки не показываются. |
| **Автопоказ верт. полосы скроллинга** | Автоматическая вертикальная полоса прокрутки при переполнении. |
| **Множественное выделение и удаление строк** | Позволяет выделять и удалять несколько строк. |
| **Разрешить перетаскивание** | Позволяет менять порядок строк. Актуально для приоритетов складских операций. |
| **Закрашивать фокусную строку** | Подсвечивает активную строку. |
| **Всплывающая подсказка** | Текст подсказки при наведении курсора на левую часть таблицы. |

### Привязка таблицы

| Настройка | Описание |
|-----------|----------|
| **Не привязана к БД** | Таблица не привязывается к записям БД. |
| **К записи диалога** | Привязка к записи диалога. |
| **К записи БД** | Привязка к указанной записи БД. |
| **Независимо от диалога** | Таблица не взаимодействует с другими органами диалога. |
| **Таблица работает с массивом ссылок** | Для записей с полями «Массив ссылок». Появляется поле «Поле массива ссылок». |
| **Таблица работает с цепочками** | Для записей с полями «Ссылка на заголовок цепочки». Появляется поле «Поле обратной ссылки» и галочка «Удалять только ссылки на заголовок». |

в языке EME-L привязка таблиц к записям БД через массивы ссылок и цепочки обеспечивает навигацию по сложным реляционным структурам.

### Параметры таблицы

- **Шрифт** — шрифт, используемый в таблице.
- **Функции таблицы** — цепочка до 4 DLL с EME-L классами, функциями и параметрами.
- **Высота заголовка** — высота заголовка в пикселях.
- **Строк для итогов** — количество подсвечиваемых строк при открытии.
- **Макс. число строк** — максимально допустимое число строк.
- **Ширина статусных кнопок** — ширина статусной кнопки в левой части.
- **Кол-во статич. столбцов** — количество статичных столбцов.

### Управление столбцами

- **Удалить** — удалить столбец.
- **Добавить** — добавить столбец (заполняется заголовок, имя органа, номер, атрибут «Невидимый»).
- **Свойства** — открыть свойства столбца как самостоятельного органа.
- **Вверх / Вниз** — изменить порядок столбцов.

### Настройки столбца

| Настройка | Описание |
|-----------|----------|
| **Комбинированный список** | Столбец в формате комбо-бокса. |
| **Редактор (поле ввода)** | Столбец в формате поля ввода. |
| **Кнопка выбора** | Столбец в формате кнопки с выбором вариантов. |
| **Обычная кнопка** | Столбец в формате кнопки без выбора. |
| **Шрифт заголовка столбца** | Шрифт заголовков столбцов. |
| **Создавать новую строку** | Формирование новой строки при заполнении значения столбца. |
| **Отображать с новой строки** | Перенос значения столбца на новую строку. |
| **Декоративный столбец** | Убирает название и скрывает данные столбца. |
| **Запрашивает отображаемый текст** | Настройка не используется. |
| **Запрашивает формат** | Настройка не используется. |
| **Фиксированная ширина** | Фиксированная ширина столбца. |
| **Ширина** | Ширина столбца в пикселях. |

> **Важно:** для смены типа органа управления столбца необходимо удалить столбец и создать новый с нужными свойствами.

## Настройки органов диалогов

Для редактирования: открыть редактор диалогов, выбрать диалог, выделить орган, правая кнопка мыши → «Редактировать». Привязка к БД — на вкладке **«Привязка»**.

в системе EME.WMS органы диалогов — это визуальные элементы управления (поля ввода, кнопки, списки), каждый из которых может быть привязан к полю БД и обрабатываться EME-L классом.

### Свойства органа

| Настройка | Описание |
|-----------|----------|
| **Имя органа управления** | Софтовое имя (латинские буквы) для обращения в программе. |
| **Всплывающая подсказка** | Текст при наведении курсора на орган. |
| **Привязать к полю в БД** | Привязка значения органа к полю БД. |
| **Использовать внешнюю функцию** | Привязка DLL из метапроекта и EME-L класса. |
| **Тип поля взять из БД** | Использование типа данных привязанного поля БД. |
| **Привязать к другому органу управления** | Привязка к другому органу на вкладке «Привязка». |
| **Вычисляемое поле** | Значение органа становится вычисляемым. |
| **Не сохранять орган в БД** | Пользовательский ввод не сохраняется в БД. |
| **Всегда проверять перед сохранением** | Актуально при сохранении данных органа в БД. |
| **Текст контекстной справки** | Текст справки, отображаемый через «Справка об элементе». |
| **Скопировать привязку** | Копирование настроек из другого органа диалога. |

### Привязка к БД

| Настройка | Описание |
|-----------|----------|
| **Реляционный путь к БД** | Путь к записи БД (несколько строк для цепочки ссылок: поле → поле → …). |
| **Знаковый** | Разрешает отрицательные и дробные числа. |
| **Не проверять на пустой** | Отключает проверку на пустое значение ссылки. |
| **Тип поля** | Тип поля. Не редактируется, если включён «Тип поля взять из БД». |
| **Знаков после запятой** | Только для численных атрибутов. |

### Функция обработчик

- **Имя DLL** — DLL метапроекта. Кнопка «<= EMEL» для привязки EME-L класса.
- **Имя функции** — функция из DLL. Кнопка «…» для выбора. Параметры: «Класс», «Ссылка», «Область видимости объекта», «Видеть диалог».

### Список стандартных функций

- **Настройка** — цепочка до 4 DLL с EME-L классами и функциями. Можно менять порядок.

### Связь с другим органом управления

- **Имя органа** — софтовое имя органа для связи. Активно только при включённом флаге «Привязать к другому органу управления».

в EME БД механизм привязки органов диалога к БД через реляционные пути позволяет автоматически отображать и редактировать данные из связанных записей, а EME-L классы-обработчики управляют логикой валидации и поведения.
