# Диалог «Заливка проекта» — развёртывание и контроль версий в ЕМЕ БД

> **Поисковые синонимы:** заливка проекта, развёртывание проекта, деплой, деплоимент, заливка изменений, базовая выгрузка, TortoiseSVN, мердж, слияние, патч, откат изменений, заливка в сеть, версия проекта, EME project deployment, project merge, version control, commit changes

## Общие сведения

Диалог **«Заливка проекта»** предназначен для контроля совместной разработки проекта группой программистов в ЕМЕ БД. Доступен по пунктам меню **«Утилиты БД → Заливка проекта → Заливка изменений»**.

**Важно:** любые изменения объектов ЕМЕ БД (классы, запросы, отчёты и т.д.) разрешены **только в моно-режиме**. Для работы с диалогом необходимо установить **TortoiseSVN**.

в языке EME-L заливка проекта обеспечивает контроль версий EME-L классов, диалогов и других программных компонентов при командной разработке.

---

## Базовая выгрузка проекта

Перед осуществлением изменений необходимо выполнить **базовую выгрузку**, фиксирующую текущее состояние всех объектов ЕМЕ БД. Для выполнения достаточно нажать соответствующую кнопку (если кнопка неактивна — выгрузка уже была выполнена).

в системе EME.WMS базовая выгрузка создаёт эталонный снимок состояния БД, относительно которого в дальнейшем отслеживаются все изменения.

---

## Просмотр истории изменений

Для просмотра истории изменений, выполненных программистами, нужно нажать кнопку **«Строки заливки»**. Откроется браузер с таблицей изменений в разрезе даты, времени и автора.

---

## Просмотр собственных изменений

1. Нажать кнопку **«Сравнить проект с базовой выгрузкой»**.
2. Проверить список объектов со статусами: **«Изменено»**, **«Добавлено»** или **«Удалено»**.
3. Выбрать нужный объект и нажать **«Показать изменения»**.
4. Ознакомиться с правками в окне **TortoiseMerge**.

в EME БД сравнение проекта с базовой выгрузкой использует TortoiseMerge для визуального отображения diff-различий между текущим и эталонным состояниями.

---

## Заливка изменений в сетевую версию

**Последовательность действий:**

1. Нажать кнопку **«Заливка в сеть»**.
   - **Важно:** перед заливкой желательно запустить базу с ключом `/nodllalign`, иначе произойдёт выравнивание с неактуальными DLL с сервера.

2. В таблице нижней части диалога выставить галочки:
   - **«Залить»** — включить объект в заливку.
   - **«Мердж»** — выполнить слияние.
   - **«Отложить»** — заливка не произойдёт.
   - **«Откатить»** — возврат к сетевой версии с отменой изменений.

3. Нажать кнопку **«Мердж»** и просмотреть изменения в отредактированных файлах.

4. Заполнить **комментарий** и нажать **«Залить изменения»**.

> **Ограничение:** заливку в сеть может выполнять **только один пользователь**. Попытки других пользователей будут блокироваться.

в языке EME-L ключ `/nodllalign` предотвращает автоматическое выравнивание DLL-библиотек с сервером, что критически важно при заливке новых версий EME-L классов.

## Восстановление после аварийного завершения

Для восстановления данных после аварийного завершения ЕМЕ БД в процессе заливки:

1. Выбрать опцию **«Из папки выгрузки»**.
2. Нажать кнопку **«Загрузить изменения»**.
3. В открывшемся окне обзора выбрать директорию **«Выгрузка проекта»**.

---

## Откат изменений

Доступна возможность откатить уже совершённую заливку в сеть (за исключением изменений в структуре БД):

1. Нажать кнопку **«Откат изменений»**.
2. В открывшемся браузере выбрать нужную заливку для возврата к её состоянию.

---

## Работа с патчами

Изменения можно сохранять без заливки в сеть:

1. Вместо «Залить изменения» нажать **«Создать патч»**.
2. Изменения оформляются в виде патча и вносятся в запись **«Патчи заливок»**. При этом изменения удаляются из проекта.
3. Для загрузки из патча: выбрать опцию **«Из патча в БД»**, нажать **«Загрузить изменения»** и указать нужный патч.

в EME БД механизм патчей позволяет программистам сохранять промежуточные результаты работы, не влияя на сетевую версию, и загружать их обратно при необходимости.
