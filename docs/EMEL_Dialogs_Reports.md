# Диалоги и отчёты в EME-L

**Описание:** Диалоги — пользовательские интерфейсы в EME.WMS. Отчёты — печатные формы и документы. Создаются через специальные редакторы и привязываются к EME-L классам.
**Сленг:** диалог, отчёт, печатная форма, интерфейс, форма, шаблон диалога

## Создание диалогов

Диалоги создаются через редактор диалогов: Система → Ресурсы → Редактор диалогов.

**Синонимы для поиска:** создание диалога, редактор диалогов, интерфейс пользователя, форма EME, шаблон диалога

### Шаги создания диалога

1. Открыть редактор диалогов
2. Двойной клик на заголовке — свойства диалога
3. Заполнить параметры:
   - Софтовое имя диалога
   - Заголовок
   - Базовая запись
4. Разместить органы управления
5. Сохранить: Ctrl-S

### Базовые свойства диалога

| Параметр | Описание |
|----------|----------|
| Софтовое имя | Имя диалога для обращения в EME-L |
| Заголовок | Текст в заголовке окна |
| Базовая запись | Запись БД для привязки данных |
| Класс | EME-L класс-обработчик |

## Органы управления диалога

На диалоге размещаются различные органы управления через панель инструментов.

**Синонимы для поиска:** органы диалога, элементы управления, controls, поля ввода, кнопки

### Основные органы

| Орган | Кнопка | Описание |
|-------|--------|----------|
| Текстовое поле | ab | Ввод текста и чисел |
| Статический текст | Aa | Метка, заголовок |
| Группа | XYZ | Объединение элементов |
| Таблица | Таблица | Табличная часть |
| Кнопка | Кнопка | Кнопка действия |
| Выпадающий список | Список | Выбор из списка |
| Календарь | Календарь | Ввод даты |

### Привязка органа к полю БД

1. Выбрать орган на диалоге
2. Открыть свойства органа
3. Вкладка "Свойства органа":
   - Имя органа (с именем привязанного класса)
   - Галочка "Привязать к полю БД"
   - Галочка "Тип поля взять из БД"
4. Вкладка "Привязка":
   - Указать поле записи БД

'Пример привязки органа к полю в EME-L'

## Обработчики событий диалога

Диалоги имеют специальные методы-обработчики для различных событий.

**Синонимы для поиска:** обработчики диалога, события диалога, Dialog_OnLoad, Dialog_OnCheck, онлоад, ончек

### Основные обработчики

| Обработчик | Когда вызывается | Сленг |
|------------|-----------------|-------|
| `Dialog_OnLoad()` | При загрузке диалога | онлоад |
| `Dialog_OnCheck(Transaction)` | Перед сохранением | ончек |
| `Dialog_OnClose(CloseType)` | При закрытии | онклоуз |
| `Dialog_OnUpdate()` | После сохранения | онапдейт |
| `Dialog_OnAfterUpdate()` | После обновления | онэфтерапдейт |
| `Dialog_OnReport(Name)` | При формировании отчёта | онрепорт |
| `Dialog_OnButton(Button)` | При нажатии кнопки | онбаттон |
| `Dialog_OnTimer()` | По таймеру | онтаймер |

### Пример Dialog_OnLoad

```eme-l
'Инициализация диалога при загрузке (онлоад)'
Dialog_OnLoad()
{
    Dialog = Object("Dialog");
    
    'Установка значений по умолчанию'
    If (StartDate.GetValue() == 0)
        StartDate.SetValue(is_date());
    End If
    
    If (EndDate.GetValue() == 0)
        EndDate.SetValue(is_date());
    End If
    
    'Настройка связанного календаря'
    ctrlStart = Object("Control",,"StartDate");
    ctrlEnd = Object("Control",,"EndDate");
    Dialog.SetControlDateRange(ctrlStart, ctrlEnd);
    
    'Загрузка драйвера сканера'
    Dialog.LoadScannerDriver(0);
}
```

### Пример Dialog_OnCheck

```eme-l
'Валидация данных перед сохранением (ончек)'
Dialog_OnCheck(Transaction)
{
    'Если транзакция уже открыта, пропускаем проверки'
    If (~Transaction)
        'Проверка обязательных полей'
        If (Name.IsEmpty())
            is_message("Внимание!", "Не заполнено наименование", 1);
            Name.SetFocus();
            Return 1;
        End If
        
        If (Code.IsEmpty())
            is_message("Ошибка", "Не заполнен код", "OK", "STOP");
            Code.SetFocus();
            Return 1;
        End If
        
        'Проверка уникальности кода'
        r_GoodsItem = Object("dsDB", "GoodsItem");
        r_GoodsItem.SetSkipMode();
        r_GoodsItem.GetCodeFld().MustBeEQ(Code.GetValue());
        r_GoodsItem.GetGoodsItemRefFld().MustBeRefNE(is_dlg_line());
        r_GoodsItem.SetFirstLine();
        
        If (r_GoodsItem.IsValidLine())
            is_message("Ошибка", "Товар с таким кодом уже существует", "OK", "STOP");
            Return 1;
        End If
    End If
    
    Return 0;
}
```

### Пример Dialog_OnClose

```eme-l
'Обработка закрытия диалога (онклоуз)'
Dialog_OnClose(CloseType)
{
    Dialog = Object("Dialog");
    
    'Выгрузка драйвера сканера'
    Dialog.UnloadScannerDriver(0);
    
    'Проверка несохранённых изменений'
    If (CloseType == 2)  'Закрытие без сохранения'
        If (is_edit_data("IsModified") == 1)
            If (is_message("Подтверждение", "Сохранить изменения?", "YESNO", "QUESTION") == "YES")
                Dialog.Save();
            End If
        End If
    End If
}
```

### Пример отчёта с запросом

```eme-sql
'Запрос для отчёта'
SELECT
    [Номер документа],
    [Дата создания],
    [@Клиент].[Наименование] AS [Клиент],
    [Сумма]
FROM
    [Документы]
WHERE
{
    startDate = Const(is_edit_data("Report.StartDate"));
    endDate = Const(is_edit_data("Report.EndDate"));
    Return is_query(,"Дата создания") >= startDate 
       & is_query(,"Дата создания") <= endDate;
}
ORDER BY
    [Дата создания]
```

## Отчёты по будильнику

Отчёты могут формироваться и отправляться автоматически по расписанию.

**Синонимы для поиска:** отчёт по будильнику, автоматический отчёт, ReportRunner, отправка отчёта, рассылка

### Создание отчёта по будильнику

1. Создать запрос и отчёт
2. Создать EME-L класс-обработчик (наследник ReportsForMail)
3. Реализовать методы:
   - `GetReportCaption()` — название отчёта
   - `GetNameReport()` — имя шаблона отчёта
   - `RunReports()` — формирование отчёта
   - `CreateReportFile()` — сохранение в файл

### Пример класса отчёта по будильнику

```eme-l
'Класс для автоматического формирования отчёта в EME-L'
Class DailySalesReport extends ReportsForMail
{
    'Конструктор класса'
    objReport = Object("IClass", "ReportRunner", "ReportRunner");
    
    'Название отчёта'
    GetReportCaption()
    {
        Return "Ежедневный отчёт по продажам";
    }
    
    'Имя шаблона отчёта'
    GetNameReport()
    {
        Return "DailySalesReport";
    }
    
    'Формирование отчёта'
    RunReports()
    {
        'Установка параметров'
        objReport.SetReportName(GetNameReport());
        objReport.SetQueryName("DailySalesQuery");
        objReport.SetCaption(GetReportCaption());
        
        'Формирование файла отчёта'
        CreateReportFile(objReport, GetNameReport());
    }
    
    'Создание файла отчёта'
    CreateReportFile(m_report, m_reportName)
    {
        'Создание отчёта и сохранение в HTML'
        Report = Object("Report", m_reportName);
        Report.Create();
        Report.SaveToFile("daily_sales_report.html");
    }
}
```

### Настройка будильника

1. Открыть: Утилиты БД → Навигатор канальной схемы → Редактор событий

## Правила оформления диалогов

При создании диалогов следует соблюдать стандарты оформления.

**Синонимы для поиска:** дизайн диалога, оформление окон, шрифты, кнопки, отступы, выравнивание

### Стандарты шрифтов и размеров

- Основной шрифт системы EME.WMS — **Arial**, обычное начертание, размер 10
- Надписи на кнопках — Arial 11, обычный
- Отступ от левого верхнего края диалога (или закладки) — **14 пикселей**
- Расстояние между статическим текстом и текстовым редактором — **4 пикселя**
- Высота текстового поля — **22 пикселя** (по умолчанию с панели)
- Расстояние между полями друг над другом — **8 пикселей**
- Расстояние между связанными полями (код и наименование) — **2 пикселя**
- Поля выравниваются к левому краю, образуя колонки

### Размеры диалогов

- Простые диалоги: высота не более **524 пикселя**
- Сложные диалоги: высота не более **750 пикселей**
- Ширина диалога: не более **1000 пикселей**
- Размер по умолчанию не должен вызывать скроллирование

### Кнопки на диалогах

| Тип кнопки | Высота | Описание |
|------------|--------|----------|
| Основная (Применить/Закрыть) | 36 px | Зелёная «Применить» RGB(0,164,65), белая «Закрыть», синяя для вторых основных действий RGB(14,121,245) |
| Средняя | 28 px | Выделяется среди малых кнопок, может быть синей |
| Малая | 25 px | Основная масса кнопок, Arial 10 |
| Квадратная | 42×42 px | Кнопка с пиктограммой |

- Кнопка «Применить» — неактивна при отсутствии модификаций, активна при наличии. При нажатии сохранение без дополнительных вопросов (галочка «Сохранять данные в БД при нажатии» НЕ должна стоять)
- Кнопка «Закрыть» — всегда доступна, предлагает сохранить изменения перед закрытием
- Оптимальная ширина кнопки = ширина текста + 13 пикселей
- На кнопках использовать стандартные пиктограммы (добавить, удалить, копировать, открыть документ и т.д.)

### Правила размещения

- Не использовать двоеточия у названий полей («Склад:» — неправильно, «Склад» — правильно)
- Двоеточие после статического текста не используется, если в столбик идёт больше одного поля. Если поля в строчку — ставится двоеточие
- Обязательные к заполнению поля — с текстом «Введи!»
- Все надписи — только на русском языке
- Заполнять «Всплывающую подсказку» и «Текст контекстной справки» для всех органов, включая таблицы
- Использовать кнопки [?] справа от ссылочного поля для перехода в классификаторы
- На диалогах с привязкой к БД: кнопка «I» (информация о статусах) в правом верхнем углу, «Применить» и «Закрыть» в правом нижнем углу

### Выравнивание элементов

Для выравнивания элементов используйте панель инструментов в редакторе диалогов. При выборе двух и более элементов и нажатии на инструменты выравнивания автоматически предлагается правильный интервал. Можно выравнивать все элементы между собой (кнопки, поля ввода, чекбоксы, радиобаттоны, текст) как по вертикали, так и по горизонтали.

## Правила блокировки контролов

Блокировка кнопок и полей ввода на диалоге методом `Disable` создаёт проблемы: пользователь не знает причину неактивности.

**Синонимы для поиска:** блокировка контролов, Disable, неактивный контрол, доступность кнопок, запрещение редактирования

### Правила блокировки

1. **Блокировку кнопок нужно исключить.** При нажатии на кнопку проверять возможность выполнения действия, и в случае невозможности выводить message box с сообщением. Сообщение должно содержать:
   - Причину запрета
   - Название запрещённой операции
   - Способы снятия запрета
   
   Пример хорошего сообщения: «Администратор заблокировал Вам доступ к операции "Откат строк" в диалоге сотрудника»
   
   Пример плохого сообщения: «Доступ запрещен»

   В заголовке сообщения — название модуля, создавшего сообщение.

2. **Блокировку полей ввода** нужно сопровождать текстом контекстной справки. В метод `Control::Disable` добавлен второй параметр — текст контекстной справки, отображаемый при наведении мыши на контрол:

```EME-L
If(dsDB.IsValidLine() & dsDB.IsClosed())
    ctrlReasonName.Disable(1, "Нельзя редактировать значение у поля \"Основание прихода\" в документе на статусе \"Закрыт\"");
End If
```

3. **Кнопку «Применить» можно блокировать** как раньше — её поведение очевидно (серая = нет изменений).

4. Можно задать кнопке текст контекстной справки вызовом метода `Control::SetHelp`. Тогда пользователь увидит проблему при наведении курсора мыши на кнопку.

## Как правильно добавить кнопку на диалог

**Синонимы для поиска:** добавление кнопки, размещение кнопки, привязка кнопки, пиктограмма кнопки

Частые ошибки при добавлении кнопок:

1. Надпись на английском языке — все надписи должны быть на одном языке
2. Размер кнопки не совпадает с соседними — выравнивайте размеры кнопок в одном ряду
3. Привязка к верхнему краю вместо нижнего — кнопка окажется под таблицей при расширении
4. Отсутствуют всплывающая подсказка и справка — через несколько дней программист не вспомнит назначение кнопки
5. Кнопка расположена не на своей закладке

**Правильная последовательность:**
1. Разместить кнопку на нужной закладке
2. Задать размеры, равные соседним кнопкам
3. Удалить текст и поставить стандартную пиктограмму (не рисовать самостоятельно!)
4. На закладке «Положение» привязать к нижней границе диалога
5. На закладке «Основные свойства» заполнить «Текст контекстной справки» и «Всплывающая подсказка»
6. Проверить, что кнопка остаётся на месте при разворачивании диалога

## Кастомный список отчётов диалога

При классическом подходе отчёты привязываются к шаблону диалога, что создаёт проблемы: дублирование, отсутствие сортировки, невозможность вложенных подменю.

**Синонимы для поиска:** свой список отчётов, диалог отчётов, ShowReports, Menu, выпадающее меню отчётов, кастомное меню

### Решение: перехват кнопки печати

Если сказать ядру, что доступен всего один отчёт, ядро не покажет меню со списком, а сразу вызовет `Dialog_OnReport`. Название отчёта игнорируется — важен сам факт нажатия кнопки. Затем формируется своё меню.

```EME-L
Dialog_OnAfterUpdate()
{
    arrReports = Object("Array");
    'список печатных форм заполняется в GetReport()'
    'привязка отчётов в редакторе диалогов игнорируется'
    'добавим один отчёт, чтобы получать Dialog_OnReport без меню'
    'какой именно отчёт — неважно, важно что он привязан к шаблону'
    arrReports.Add("Лист вложения");
    Dialog.ShowReports(arrReports, TRUE);
}

Dialog_OnReport(Name)
{
    'Name всегда содержит "Лист вложения", меню формируем сами'
    dsDB.SetLine(is_dlg_line());
    
    'создаём меню, отображаем его, возвращаем выбранный отчёт'
    reportName = GetReport(dsDB);
    
    If(reportName != "")
        PrintReport(reportName, dsDB);
    End If
    
    'ядро ничего не делает — отчёт формируем в EME-L'
    Return TRUE;
}

GetReport(r_Order)
{
    'список доступных форм зависит от статуса'
    nStatus = r_Order.GetStatusAsNumber();
    
    menu = Object("Menu", TRUE);
    arrReports = Object("Array");
    
    'подменю с вложенными отчётами'
    submenu = Object("Menu", TRUE);
    submenu.AppendMenu("STRING", arrReports.Add("ТТН"), "ТТН");
    submenu.AppendMenu("STRING", arrReports.Add("Накладная"), "Накладная");
    submenu.AppendMenu("STRING", arrReports.Add("Счет фактура"), "Счет фактура");
    submenu.AppendMenu("STRING", arrReports.Add("МХ 3"), "МХ 3");
    menu.AppendMenu("POPUP", submenu, "НАКЛАДНЫЕ");
    
    If (nStatus >= osHasLeftWarehouse & nStatus != osPartlyAssembled & nStatus <= osFinished)
        menu.AppendMenu("STRING", arrReports.Add("Заявление о страховой стоимости"), "Заявление о страховой стоимости");
    End If
    
    If(arrReports.GetSize() == 0)
        Return "";
    End If
    
    action = menu.TrackPopupMenuWithMousePos();
    
    If(action < 0 | action >= arrReports.GetSize())
        Return "";
    End If
    
    Return arrReports.GetAt(action);
}
```

## Правильное создание отчёта

**Синонимы для поиска:** создание отчёта, дизайн отчёта, шрифты отчёта, выравнивание, таблица отчёта, интерактивный отчёт

### Требования к диалогу запуска отчёта

1. Название в заголовке окна = название в меню = заголовок окна отчёта
2. Сверху — группа «Период отчёта»
3. Далее — группа «Фильтры» с кнопками, сдвинутыми влево. Надписи во множественном числе: «Склады», «Ячейки», «Товары»
4. Под фильтрами — остальные поля и чекбоксы. Двоеточия не ставить
5. Кнопки «Отчёт» (слева) и «Закрыть» (справа) внизу диалога

**Стандартные размеры диалогов отчётов:**
- «Малыш» — 378 × 410
- «Гигант» — 378 × 650
- «Супергигант» — 491 × 823

**Шрифт:** MS Sans Serif 10, основной. Установить шрифт диалогу, а не отдельным органам.

**Размеры кнопок:** фильтры — 158 × 25, «Отчёт» — 102 × 36, «Закрыть» — 108 × 36

### Настройка заголовка отчёта

Чтобы заменить софтовое название отчёта на人道ное, в редакторе отчётов (Настройка → Свойства отчёта, группа «Функции отчёта») привяжите функцию `rs_main_proxy` из `MetC_Sys.dll` и укажите ваш EME-L класс:

```EME-L
OnBeginReport()
{
    Object("Report").SetCaption(tr("Статус заказов на комплектацию"));
}
```

### Правила оформления таблиц отчёта

**Заголовок:** жирный Arial 18pt, по центру. Даты периода — в одной строке с заголовком (если короткий) или ниже на 2-4 pt меньше. Версия отчёта — мелким шрифтом 6-8 pt. Дата и время формирования, имя хоста — ниже версии.

**Легенда:** при 2+ цветовых обозначениях — добавить «легенду» в конце отчёта, объясняющую каждый цвет.

**Нижний колонтитул:** № страницы, количество страниц, количество записей, применённые фильтры, дата и время печати.

**Выравнивание текста:**
- Названия, текст — влево
- Количество, цена, сумма — вправо
- Дата — по центру
- Заголовки колонок — жирный, по центру
- «Итого» — влево, итоговая сумма — вправо

**Шрифт:** Arial 8-10pt. Запрещены разные размеры в одной строке, декоративные и моноширинные шрифты.

**Границы:** тонкие. Для разделения внутри группы — пунктир, между группами — двойная линия. Столбец «Количество» и суммовой столбец отделить горизонтальной границей.

**Перенос строк:** запрещён для № документа, кода товара, партии, количества, цены, суммы, адреса ячейки. Допускается для названия товара, контрагента, банковских реквизитов (не более 3 строк).

### Интерактивные возможности отчёта

- Ячейки с кликабельным текстом (подчёркнутые) для перехода в диалоги
- Плюсики/минусики для сворачивания/разворачивания данных
- Проверка конвертации в Excel, OpenOffice, PDF, HTML

### Отчёты по будильнику — дополнительная информация

Отчёты и их запросы не должны обращаться к диалогу и органам напрямую при работе через планировщик. Запрещены `is_edit_data()`, `is_hewed()`, `Object("Dialog")`, `Object("Control")`.

Для передачи параметров из диалога в запрос используется класс `ReportRunner`. Класс содержит интерфейсные методы для чтения фильтров, к которым можно обращаться из отчёта и запроса:

```EME-L
'Получить дату начала периода из ReportRunner:'
Object().GetDateBegin();

'Получить параметры запуска:'
Object().GetParameters();
```

Пример передачи параметров:

```EME-L
'В классе диалога читаем параметр из поля:'
SectorWith = Object("Control",,"SectorWith");
Param.SetParam("SectorWith", SectorWith.Get());

'В классе отчёта по будильнику устанавливаем константу:'
MinSector = 1;
Param.SetParam("SectorWith", MinSector);

'В запросе получаем значение:'
r_Parameters = CONST(Object().GetParameters());
SectorWith = CONST(is_type(r_Parameters.GetParam("SectorWith"),1));
```

Методы класса `ReportRunner`: `GetReportCaption()`, `GetNameReport()`, `RunReport()`, `CreateReportFile()`, `GetDateBegin()`, `GetParameters()`.

Переменная «Видимость отчёта» (`bShow`): при запуске с диалога = `TRUE`, при запуске по будильнику = `FALSE`. Вместо `is_edit_data()` использовать `is_report_param()`.

## Выпадающий список при вводе в ссылочное поле

Начиная с ядра 45.5 доступен выпадающий список при вводе в текстовое поле по ссылке.

**Синонимы для поиска:** автозаполнение, autocomplete, выпадающий список, ссылочное поле, подсказка ввода

### Настройка

1. Добавить в структуру БД поле типа **«Древовидная сортировка»** в запись, на которую ссылается текстовое поле
2. В сортировке указать поле, значение которого должно отображаться в выпадающем списке
3. Пример: текстовое поле привязано к записи «Клиенты-партнёры», в записи поле «Индекс Код клиента» с типом «Древовидная сортировка», в сортировке указано поле «Код клиента»

## 3D-программирование на диалоге (Irrlicht)

В EME.WMS доступен графический движок Irrlicht (начиная с версии ядра 46) для отображения 3D-сцены на диалоге.

**Синонимы для поиска:** 3D, Irrlicht, IrrDevice, IrrNode, IrrCamera, IrrLight, IrrMesh, визуализация, 3D-орган

### Создание 3D-органа управления

В редакторе диалогов нет специального 3D-органа, поэтому используется технология динамической замены:

1. На диалог размещается орган типа «редактор»
2. К нему подключается метафункция `ts_proxy_3d` из библиотеки `MetC_Sys.dll`
3. На сообщении `TSM_INIT` метафункция удаляет редактор и подставляет объект класса `CIrrlictCtrl`

### Интерфейс 3D-органа с EME-L

Создаётся EME-L класс с методом `<Имя3DОргана>_OnAttach` (один параметр — объект `IrrDevice`):

```EME-L
VizexIrrlicht_OnAttach(Irrlicht)
{
    m_VizexIrrlicht = Irrlicht;
    ConstructVizex3D(m_VizexIrrlicht);
    m_VizexIrrlicht.SetAutoUpdate(m_Dialog.IsTabItemActive("Vizex"));
    m_VizexIrrlicht.GetActiveCamera().SetFOV(0.5);
}
```
