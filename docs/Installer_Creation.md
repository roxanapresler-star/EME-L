# Создание установочного файла (инсталлятора) EME.WMS

> **Синонимы для поиска:** инсталлятор EME WMS, создание установщика, NSISBI, генератор инсталлятора, сборка инсталлятора, версия WMS, ProjectVersion, Professional версия, Standard версия, NSI скрипт, metc_clr, кодировка Windows-1251, лицензионное соглашение, InstallerDoc, autorun.inf

---

## Общие сведения

в системе EME.WMS для создания установочного файла требуется программа **NSISBI** (`https://sourceforge.net/projects/nsisbi/`).

Перед сборкой инсталлятора нужно убедиться, что в ядре существует файл `kernel.x64\exe\Irrlicht.dll`. Без него инсталлятор соберётся, но установленный проект не будет нормально работать.

Также необходимо скачать директорию с комплектом файлов для записи на диск и документами лицензионных соглашений — расположена по адресу `G:\Для записи на диск`.

### Пошаговая инструкция создания инсталлятора

1. Выполнить сборку метапроекта
2. Проверить и при необходимости исправить номер версии программы
3. Выполнить генерацию скрипта сборщика через диалог **«Генератор инсталлятора»**
4. Закрыть рабочую станцию и выполнить скрипт сборки с помощью NSISBI
5. Скопировать созданный инсталлятор в директорию «Для записи на диск»

---

## Проверка и редактирование номера версии программы EME.WMS

в системе EME.WMS менеджер продаёт конкретную версию, поэтому при несоответствии версий придётся переделывать весь дистрибутив. Места для проверки:

### 1. EME-L класс ProjectVersion

Открыть класс `ProjectVersion` в редакторе классов. Методы `GetMajorVersion()` и `GetMinorVersion()` должны возвращать старшую и младшую цифру версии WMS.

### 2. Заставка программы

Меню: **«Система → Ресурсы → Заставка программы»** — проверить, что в поле «Строки» указана требуемая версия.

### 3. Файл autorun.inf

в EME БД в файле `autorun.inf` (из директории для записи на диск) проверить строку `open=wms{название версии}.exe`.

### 4. Диалог ProjectName.Dialog

Открыть диалог `ProjectName.Dialog` через редактор диалогов, запустить (`Ctrl + R`) и указать название проекта.

### 5. Лицензионное соглашение

Из директории «Для записи на диск» взять архив `InstallerDoc.zip`, убедиться в актуальной версии WMS в каждом документе и разархивировать по пути: `{Рабочий каталог ЕМЕ БД}\metaproj\src\outerWMS` с заменой файлов.

## Генерация скрипта сборщика инсталлятора

в системе EME.WMS генератор инсталлятора доступен по пунктам меню **«Утилиты БД → Система → Генератор инсталлятора»**.

Особенности генератора:

- Сохраняет введённые данные (кнопка «Применить» — данные сохраняются в `kernel.ini`)
- Можно указать путь к NSISBI на вкладке «Файлы» — программа откроется в контексте генерации

### Вкладка «Типы установки»

| Тип | Содержимое |
|-----|-----------|
| Профессиональная версия | Полный набор: сервер, службы интеграции, рабочая станция, Java-Монитор, веб-порталы (дашборды, 3PL), учебная станция |
| Стандартная версия | Базовый набор: сервер, службы интеграции, рабочая станция, Java-Монитор |
| Только рабочая станция | Рабочая станция + Java-Монитор |
| Свой набор | Самостоятельный выбор модулей на вкладке «Дерево модулей» |

### Вкладка «Дерево модулей»

Выбор модулей для включения: серверные службы, рабочая станция, учебная станция, SQL-базы, Сервер веб-монитора, Дашборды, 3PL. Указать пути до всех папок с данными и дополнительными инсталляторами.

### Вкладка «Файлы»

Пути до:
- Папки для создания скрипта сборки
- Компонентов ЕМЕ БД (Ядро, Метапроект, данные, ТСД, обучающие видео, Хэш-БД)

Галочки: демо-версия (заменяет `LicenseAgreement.rtf` на `LicenseAgreementTrial.rtf`), включение пользовательских данных.

### Вкладка «Ресурсы»

Папка с декоративными элементами и файлами лицензионного соглашения: `C:\{Рабочий каталог ЕМЕ БД}\metaproj\src\outerWMS\`.

### Вкладка «PostgreSQL»

Путь к инсталлятору PostgreSQL и PSQL ODBC (опционально).

### Вкладка «Монитор Заданий»

Путь к исходникам Монитора Заданий, инсталлятору JDK и веб-Монитору.

### Вкладка «Веб-сервисы»

Путь к исходникам портала 3PL (Publish, сертификаты) и портала дашбордов.

### Генерация и сборка

После настройки всех вкладок вернуться на «Типы установки» и нажать **«Создать»**. Программа предложит сгенерировать скрипт — согласиться. На вопрос о запуске сборки — выбрать **«Да»**.

в EME БД при первой попытке сборки будет ошибка (рабочая станция открыта). Нужно:
1. Закрыть рабочую станцию и сервер
2. Начать процесс сборки заново (`Ctrl + R`)

Результат: файл скрипта и инсталлятор в директории, указанной на вкладке «Файлы».

## Решение возможных проблем при создании инсталлятора

### Ошибка: файл не найден

в системе EME.WMS если при создании инсталлятора возникает ошибка вида `File: "...\kernel..\exe\metc_clr.dll" no files found` — нужно открыть `.nsi` файл в блокноте, найти и удалить данную строку. Это актуально, если метапроект собран без 3PL-портала.

Аналогичное решение применяется для всех файлов, которых нет в метапроекте, но которые скрипт инсталлятора ищет для сборки.

### Кодировка NSI-файла

в EME БД файл `.nsi` должен быть сохранён в кодировке **Windows-1251**. При сохранении в UTF-8 произойдёт ошибка при сборке инсталлятора.
