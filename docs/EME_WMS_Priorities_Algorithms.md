## Системные приоритеты в EME.WMS

<!-- Синонимы: системные приоритеты, приоритеты резервирования, критерии важности, стратегии размещения, параметры прихода, параметры подборки -->

### Назначение

Системные приоритеты в EME.WMS определяют критерии важности применения правил резервирования товара в приёмке/подборке в рамках формирования соответствующих стратегий размещения в EME.WMS.

### Диалог "Системные приоритеты"

Наименования системных приоритетов находятся в диалоге "Системные приоритеты", который расположен по пунктам меню: "Утилиты БД -> Система -> Системные приоритеты".

В диалоге указывается:
- **Название класса** с реализацией приоритета (в стиле Camel Case)
- **Комментарий** к приоритету
- **Галочка "Уход"** — если приоритет нужно использовать в параметрах подборки для стратегии размещения

Значения в данном диалоге могут редактироваться только на филиале разработчика.

**Важно:** Системные приоритеты формируются в метапроекте и недоступны в EME-L.

### Использование приоритетов

Приоритеты из данного диалога используются в специальных таблицах стратегий размещения (параметры прихода/подборки). Порядок расстановки приоритетов может быть определён программой по умолчанию или вручную посредством браузера "Приоритеты системные".

Для управления последовательностью применения приоритетов нужно выставить соответствующие значения для каждого приоритета в колонке "П".

### Создание нового системного приоритета

Чтобы создать новый системный приоритет, необходимо выполнить следующий перечень действий:

1. **Создать новый файл приоритета в метапроекте.** За образец брать файл `sc_ByRef.cpp`.

2. **Придумать название для класса приоритета**, например: `"CSearchClassBy - объект приоритета"`.

3. **Добавить инициализацию приоритета** в метод `INNER(PriorityOperations)::FillAlgoStringArrayDefaultOutcome` (файл `r_PriorityOperations.cpp`) — добавить две строчки инициализации приоритета (не считая комментария) в соответствующее приоритету место.

4. **Объявить новый приоритет** в рабочей станции по пунктам меню: "Утилиты БД -> Система -> Системные приоритеты". Проставить галочку "Уход" — если приоритет нужно использовать в параметрах подборки для стратегии размещения. Иначе приоритет будет доступен только в параметрах прихода.

5. **Проверить**, что приоритет можно выбрать в стратегиях размещения (параметры прихода/подборки).

## Фильтры и алгоритмы пополнения/оптимизации

<!-- Синонимы: алгоритмы пополнения, алгоритмы оптимизации, фильтры прихода, фильтры подборки, PPU, заполнение приоритетов -->

### Диалог "Алгоритмы пополнения"

Используемые в EME.WMS фильтры и алгоритмы пополнения/оптимизации настраиваются в диалоге, расположенном по следующим пунктам меню: "Утилиты БД -> Система -> Алгоритмы пополнения".

В дальнейшем алгоритмы отображаются в различных браузерах диалогов в качестве применяемых параметров (например: параметр "Алгоритм оптимизации" в диалоге "Оптимизация склада").

**Важно:** Значения в данном диалоге могут редактироваться только на филиале разработчика.

### Параметры алгоритмов

Для каждого алгоритма выставляются следующие параметры:

| Параметр | Описание |
|----------|----------|
| **Не используется** | Если флаг выставлен, алгоритм нельзя выбрать ни в одном браузере специализированного диалога |
| **PPU** | Алгоритм предназначен для пополнения пикинга. Можно задать как значение параметра "Алгоритм" в диалоге "Задание на пополнение пикинга" |
| **Фильтр прихода** | Алгоритм предназначен для поиска ячейки для размещения товара. Можно задать как значение параметра "Дополнительный фильтр" в параметрах прихода для стратегии размещения |
| **Фильтр подборки** | Алгоритм предназначен для резервирования ячеек в рамках подбора заказов. Можно задать как значение одноимённого параметра в параметрах подборки для стратегии размещения |
| **Заполнение приоритетов** | Алгоритм предназначен для динамического вычисления приоритетов типов ячеек отдельно для каждого товара при приходе. Можно задать как значение параметра "Динамическое заполнение приоритетов" в параметрах продукта для стратегии размещения |
| **Код** | Наименование алгоритма (желательно указывать на русском языке) |
| **Класс** | EME-L класс реализации алгоритма |
| **Метод** | Метод EME-L класса, содержащий реализацию алгоритма |
| **!** | Флаг, означающий использование данного алгоритма по умолчанию |
| **Описание** | Текстовое описание алгоритма |

### Классификация алгоритмов

Если у алгоритма не проставлена ни одна из следующих галочек: "PPU", "Фильтр прихода", "Фильтр подборки" и "Заполнение приоритетов" — алгоритм автоматически классифицируется как **"алгоритм оптимизации склада"**. Такие алгоритмы можно выбрать в браузере диалога "Оптимизация склада".

## Применение в стратегиях размещения

<!-- Синонимы: стратегии размещения, параметры прихода, параметры подборки, параметры продукта, динамическое заполнение приоритетов -->

### Параметры прихода

В параметрах прихода для стратегии размещения используются:
- **Системные приоритеты** — определяют порядок применения правил размещения
- **Фильтр прихода** — алгоритм для поиска ячейки для размещения товара
- **Динамическое заполнение приоритетов** — алгоритм для вычисления приоритетов типов ячеек

### Параметры подборки

В параметрах подборки для стратегии размещения используются:
- **Системные приоритеты** (с галочкой "Уход") — определяют порядок резервирования
- **Фильтр подборки** — алгоритм для резервирования ячеек

### Параметры продукта

В параметрах продукта для стратегии размещения используется:
- **Динамическое заполнение приоритетов** — алгоритм вычисления приоритетов типов ячеек отдельно для каждого товара

## Пример использования

### Настройка приоритетов для прихода

1. Открыть диалог "Системные приоритеты" ("Утилиты БД -> Система -> Системные приоритеты")
2. Выбрать нужные приоритеты из списка
3. В браузере "Приоритеты системные" выставить значения в колонке "П" для управления последовательностью
4. Сохранить изменения

### Настройка алгоритма пополнения пикинга

1. Открыть диалог "Алгоритмы пополнения" ("Утилиты БД -> Система -> Алгоритмы пополнения")
2. Найти или создать алгоритм с галочкой "PPU"
3. Указать EME-L класс и метод реализации
4. В диалоге "Задание на пополнение пикинга" выбрать этот алгоритм в параметре "Алгоритм"

### Настройка фильтра прихода

1. В диалоге "Алгоритмы пополнения" найти алгоритм с галочкой "Фильтр прихода"
2. Убедиться, что EME-L класс и метод указаны корректно
3. В параметрах прихода для стратегии размещения выбрать этот алгоритм в параметре "Дополнительный фильтр"
