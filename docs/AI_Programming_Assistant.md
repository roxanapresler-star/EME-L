## Назначение ассистента программиста

Группой искусственного интеллекта разработан модуль «Ассистент программиста», доступный в рабочей станции EME.WMS (веб-интерфейс в редакторе классов) и в Android Studio (расширение ProxyAI). Ассистент помогает писать код на EME-L и Java соответственно. Программист взаимодействует с ассистентом на естественном языке в формате чата.

### Языковые модели

| Вариация | Модель | Назначение |
|---|---|---|
| Ассистент на EME-L | Qwen2.5-14B-chat-instruct-Q8 | Дообучена на языке EME-L |
| Ассистент на Java | Qwen3-coder:30b-a3b-q8_0 | Помощь в разработке на Java |

**Важно:** Ассистент может допускать ошибки (try-catch внутри транзакции, несоответствующие объекты EME-L и т.д.). Необходима валидация сгенерированного кода. Может потребоваться несколько итераций для получения корректного результата.

## Ассистент программиста на EME-L (в EME.WMS)

### Доступ и регистрация

Ассистент не требует особой настройки. Нажмите кнопку **«E»** в правом верхнем углу редактора классов. Работает только во внутренней сети ЕМЕ или через VPN.

Веб-интерфейс: `http://10.1.6.108:3000/`

Новому пользователю нужно ввести логин и пароль для регистрации.

### Интерфейс LibreChat

После авторизации открывается интерфейс **LibreChat** с функционалом:

- **Системный промпт** — задаётся через меню настроек (правая часть интерфейса)
- **Параметры модели** — температура (степень креативности), максимальное количество токенов
- **Новый чат** — кнопка в правом верхнем углу
- **Загрузка файлов** — скрепка в левом нижнем углу → «Загрузить как текст»
- **История чатов** — кнопка в левом верхнем углу, группировка по датам
- **Архивирование/удаление** чатов

### Ограничения ассистента на EME-L

- Работает исключительно с текстовым контентом
- Поддерживает до **8192 токенов** (≈5–6 сообщений в одном чате). При достижении лимита — создайте новый чат
- Чем подробнее сформулирован вопрос, тем выше вероятность приемлемого ответа
- Модель **не знает** EME-SQL и синтаксис запросов — ответы по этой теме часто некорректны

### Обратная связь

В активном чате нажмите «Палец вниз» → выберите категорию проблемы → нажмите повторно для детального описания. Отзывы помогают выявлять пробелы в знаниях модели и улучшать будущие версии.

### Векторный поиск по кодовой базе EME.WMS

Ассистент использует векторную базу данных с ≈3200 методов EME-L из базовой версии EME.WMS. Процесс формирования ответа:

1. Пользователь задаёт вопрос (например: «что такое while?»)
2. Модель генерирует несколько уточняющих вопросов по тематике запроса
3. Вся пачка вопросов векторизуется, из базы выбираются релевантные блоки кода
4. Дублирующиеся блоки устраняются
5. Формируется 10 блоков кода, прогоняются через **модель-реранкер** для уточнения релевантности
6. Итоговый ответ формируется на основе **4 наилучших кандидатов**

## Ассистент программиста в Android Studio (ProxyAI)

### Установка расширения

1. Версия Android Studio — не ниже **2025.1.1**
2. Загрузите расширение ProxyAI: `H:\Kpi\CodeGPT-3.5.0-241.1.zip` (не распаковывать)
3. `Ctrl+Alt+S` → **Plugins** → шестерёнка → **Install Plugin from disk** → укажите zip-файл
4. Перезапустите IDE

### Настройка провайдера модели

1. `Ctrl+Alt+S` → **ProxyAI → Providers → Custom OpenAI**
2. Нажмите **Import Settings** → укажите `H:\kpi\EMECoderAiSettings.json`
3. Нажмите **Apply**
4. В выпадающем окне назначьте модель для всех вариантов: Chat, Auto apply, Commit messages, Edit code, Name lookups, Autocomplete
5. Нажмите **Update Models**, при необходимости перезапустите IDE

Модель доступна через иконку ProxyAI в правой части интерфейса Android Studio.

### Формирование контекста беседы

- **Файл как контекст:** введите `@`, затем имя файла
- **Блок кода как контекст:** выделите фрагмент левой кнопкой мыши (подключится автоматически)
- Рекомендуется работать с отдельными блоками кода для лучшей точности

### Системный промпт

Кнопка **Default Persona** в верхней части расширения. В разделе **Prompts** — набор шаблонов. Промпты можно писать на русском и английском.

### Генерация и применение кода

Код генерируется в окне-листинге с автоматическим форматированием. Доступны:

- **Apply changes** — применить сгенерированный код к файлу
- **Шестерёнка → Compare** — сравнение старого и нового кода с возможностью принять/отклонить изменения

### Редактирование кода в файле

Выделите блок кода → правая кнопка мыши → **ProxyAI → Edit code** → опишите требуемые изменения. Изменённый блок помечается символами ```` ``` ````, которые нужно удалить вручную.

### Генерация юнит-тестов

Выделите фрагмент кода → правая кнопка мыши → **ProxyAI → Write Tests**. Сохраните через шестерёнку → **Create New File**.

### Быстрые действия через контекстное меню ProxyAI

- **Find Bugs** — поиск багов в коде
- **Explain** — объяснение кода
- **Refactor** — рефакторинг кода
- **Optimize** — оптимизация узких мест
