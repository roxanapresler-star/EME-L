# Программирование SQL-запросов в EME.WMS

> Синонимы для поиска: SQL запрос EME БД, SELECT WHERE GROUP BY ORDER BY EME-SQL, NEW{} источник запроса, TOTAL UNION EME, WITH WITH(Rec) HAVING, CHAIN JOIN реляционное чтение, NULL_REF NULL_RECORD, агрегатные функции EME-L, is_query is_edit_data, Const оптимизация запроса, SQL уроки EME.WMS, редактор запросов EME, интерпретируемый SQL, вычисляемое поле запроса

## Общие сведения о SQL в ЕМЕ БД

В ЕМЕ БД представлен полноценный интерфейс SQL, реализованный в виде интерпретируемого языка запросов. Запросы хранятся непосредственно в проекте базы данных вместе с другими прикладными данными. Это означает, что внесение изменений в эту часть программы доступно программисту в любой момент времени, не требует обновления версий и включается в работу сразу же после сохранения в редакторе запросов. в системе EME.WMS SQL-запросы являются основным инструментом для формирования отчётов и выборки данных.

### Специфика SQL в ЕМЕ БД

1. Фильтрация в запросе может осуществляться с помощью операторов `WHERE`, `WITH(Rec)`, `WITH` или `HAVING`, внутри которых описывается логика фильтрации на языке EME-L.

2. Простые поля описываются в квадратных скобках `[Имя_поля]`, а вычисляемые — через агрегатные функции или конструкции вида `{выражение}:ТИП AS [Имя_поля]`.

3. Поля-ссылки начинаются с символа `@`, используется синтаксис `[@ПолеСсылка].[ПолеВСвязаннойЗаписи] AS [Имя_поля]`. Вложенность реляционного запроса может быть любой. Выражением `[@]` обозначаются номера строк в текущей записи.

4. Любой блок запроса может содержать код на EME-L.

5. Синтаксис `{1}:INT` описывает константное поле, `SUM({1}:INT)` имитирует работу агрегатной функции `COUNT`.

6. `CHAIN JOIN` — вид объединения таблиц, в результате которого подтягиваются все ссылки из связанной записи.

7. `TOTAL` — применяется после оператора `UNION`, чтобы дальнейшие операторы применялись ко всему результату, а не к последней записи.

8. `NULL_REF` — константа, обозначающая неверный или отсутствующий номер строки.

## Пример простейшего SQL-запроса в ЕМЕ БД

```EME-SQL
SELECT
    [@Клиент].[Фамилия] AS [Фамилия],

    SUM({
        Return 1;
    }:INT ) AS [Количество звонков]

FROM
    [Звонки клиентов]

WHERE
{
    NeedData = Const(is_edit_data("Report.Date"));
    Return is_query(,"Дата") == NeedData;
}

GROUP BY
    [Фамилия]
```

### Пояснения к запросу

- В конструкции `SELECT` перечисляются поля с полным реляционным путём и задаются названия для результирующей таблицы с помощью ключевого слова `AS`.
- Для подсчёта количества звонков одного и того же клиента строки группируются по значению поля «Фамилия» и производится суммирование с помощью агрегирующей функции `SUM`.
- В конструкции `FROM` указывается источник информации.
- Конструкция `WHERE` используется для задания условий включения строк в результат. в языке EME-L внутри WHERE используется вычисляемое поле.
- В конструкции `GROUP BY` указывается поле группировки.

### Ключевые EME-L функции в запросах

- `is_edit_data("Орган диалога")` — возвращает значение, находящееся в органе диалога.
- `is_query("Имя запроса","Реляционный путь")` — возвращает значение поля, указанного реляционным путём.

**Важно:** в фигурных скобках формируется «вычисляемое поле». Результат вычислений возвращается оператором `Return`. После чего указывается тип поля. Пробелы после фигурных скобок и до указания типа **недопустимы**.

Оператор `Const(выражение)` необходим там, где выполняется код, всегда возвращающий один и тот же результат. Если при выполнении запроса к записи с 1000 строк его не использовать — из органа диалога 1000 раз будет получено одно и то же значение.

## SQL-запрос с оператором LEFT JOIN

```EME-SQL
SELECT
    {
    Time=is_time(8+is_line(),0);
    Return Time;
    }:TIME AS [Время],

    [Количество звонков]

FROM
    NEW{11}

LEFT JOIN
    [Запросы по клиентам.Звонки клиентов по времени]
ON
    [Время] = [Запросы по клиентам.Звонки клиентов по времени]![Время]

GROUP BY
 [Время], [Фамилия]
```

- `is_line()` — возвращает номер текущей строки, созданной запросом.
- `is_time(час, минута, секунда)` — возвращает время типа `TIME`.
- `NEW{11}` — запрос формирует 11 новых строк (временные значения от 08:00 до 18:00), не извлекая данные из БД.

## SQL-запрос с оператором UNION и TOTAL

```EME-SQL
SELECT
 {
    Time=is_time(8+is_line(),0);
    return Time;
 }:TIME AS [Время],

 {  0
 }: INT AS [Количество звонков],

 {  0
 }: INT AS [Фамилия],

FROM
 NEW{11}

LEFT JOIN
 [Запросы по клиентам.Звонки клиентов по времени]
ON
 [Время] = [Запросы по клиентам.Звонки клиентов по времени]![Время]

UNION

SELECT *

FROM
 [Запросы по клиентам.Звонки клиентов по времени]

TOTAL

GROUP BY
 [Время],[Фамилия]
```

`UNION` объединяет вывод двух запросов в один. Объединять можно только таблицы с одинаковым набором полей. `TOTAL` применяется для того, чтобы `GROUP BY` работал на всём результирующем наборе данных.

## SQL-запрос с оператором WITH

Оператор `WITH` предназначен для отбора строк в запрос, если источником является запись БД. Настоятельно рекомендуется использовать `WITH` вместо `WHERE` или запроса с типом «Входной фильтр». в языке EME-L код оператора `WITH` вызывается один раз, поэтому не нужно использовать `Const`.

```EME-SQL
SELECT
    ...
FROM
    [Строки документа]

WITH
{
    objQuery = is_context("Query");
    r_Document = Object("dsDB", "Document");
    r_Document.SetSkipMode();

    r_Document.GetCreateDateFld().MustBeGE(is_edit_data("ReportIncoming.StartDate"));
    r_Document.GetCreateDateFld().MustBeLE(is_edit_data("ReportIncoming.EndDate"));
    r_Document.GetDocumentTypeRefFld().MustBeRefEQ(0);

    bbDocs = Object("BitBuffer", is_No_of_lines("Документы"));
    bbDocs.LoadLines(r_Document);

    r_DocLines = Object("dsDB", "DocLines");
    r_DocLines.SetSkipMode();
    r_DocLines.GetDocumentRefFld().MustBeRefInBitBuffer(bbDocs);

    bbGI = Object("BitBuffer");
    bbGI.LoadHewVariableSelection(Object("Dialog"), "Товар");

    If (bbGI.GetNoOfHewedItems() > 0)
        r_DocLines.GetGoodsItemRefFld().MustBeRefInBitBuffer(bbGI);
    End If

    r_DocLines.GetFlagsFld().MustBeBitSet(is_or(r_DocLines.GetGoodsInBufferBit(), r_DocLines.GetGoodsLoadedBit()));

    objQuery.LoadLines(r_DocLines);
}
```

## SQL-запрос с оператором WITH(Rec)

```EME-SQL
SELECT
    [Дата] AS [Дата создания],
    [Время] AS [Время создания],
    [Номер заказа],
    [@Склад].[Наименование] AS [Склад],
    [@Перевозка].[@Водитель].[Фамилия] AS [Фамилия водителя]

FROM
    [Заказы]

WITH(Rec)
{
    Rec.SetSkipMode();
    Rec.GetInputDateFld().MustBeGE(is_date(1, 5, 2022));
}
```

Код оператора `WITH(Rec)` вызывается один раз. Можно использовать `SetSkipMode` или `SetChainMode`. В отличие от `WHERE`, где код вызывается для каждой строки.

## SQL-запрос с оператором HAVING

```EME-SQL
SELECT
    [Дата],
    [Время],
    [@Покупатель].[Фамилия]

FROM
    [Звонки покупателей]

HAVING
{
    NeedSurname = Const(is_edit_data("Buyers.Surname"));
    return is_query(, "Фамилия") == NeedSurname;
}
```

в языке EME-L оператор `HAVING` фильтрует данные после формирования запроса. Рекомендуется использовать `WHERE` или `WITH` вместо `HAVING` для фильтрации до формирования.

## Сортировка в SQL-запросах (ORDER BY)

Сортировка по убыванию:

```EME-SQL
ORDER BY
[Дата] DESC, [Время] DESC
```

Сортировка по возрастанию:

```EME-SQL
ORDER BY
[Дата] ASC, [Время] ASC
```

## Реляционное чтение в SQL-запросах

Конструкция `[@Склад].[Наименование]` выполняет реляционное чтение. в системе EME.WMS это работает так:

1. Из текущей строки записи читается ссылка из поля `@Склад`.
2. В записи «Склады» из вычисленной строки читается поле `Наименование`.
3. Глубина реляционного чтения не ограничена: `[@Перевозка].[@Водитель].[Фамилия]`.

## Уроки по SQL-запросам в EME.WMS

Уроки реализованы в 63 филиале ЕМЕ, проходить их можно в 0 разделе (запрос «Учебник по запросам» в редакторе запросов).

### Урок 1: Источник запроса — NEW

```EME-SQL
SELECT
{
    A;
    A = A + 1;
    Return A;
}:INT AS [Номер дня],
{
    DayNo = is_query(".", "Номер дня");
    Return is_date(DayNo, 2, 2022);
}:DATE AS [Дата]

FROM NEW
{
    Return is_days_in_month(2, 2022);
}
```

Конструкция `FROM NEW{}` формирует запрос без обращения к записи БД. В фигурных скобках указывается количество строк. Функция `is_line()` возвращает номер текущей строки.

### Урок 2: Источник запроса — запись БД

```EME-SQL
SELECT
    [Дата] AS [Дата создания],
    [Время] AS [Время создания],
    [Номер заказа],
    [@Склад].[Наименование] AS [Склад],
    [@Перевозка].[@Водитель].[Фамилия] AS [Фамилия водителя]

FROM
    [Заказы]

WHERE
{
    DateStart = CONST(is_date(1, 5, 2022));
    Return is_query(, "Дата ввода заказа") >= DateStart;
}
```

### Урок 3: Объединение результатов двух запросов (UNION)

```EME-SQL
SELECT
    [Дата ввода заказа] AS [Дата],
    SUM({1}:INT) AS [Счетчик]
FROM
    [Заказы]
WITH
{
    Query = is_query();
    r_Orders = Object("dsDB", "Orders");
    r_Orders.SetSkipMode();
    r_Orders.GetInputDateFld().MustBeGE(is_date(1, 5, 2022));
    Query.LoadLines(r_Orders);
}
GROUP BY
    [Дата]

UNION

SELECT
{
    A;
    A = A + 1;
    Return CONST(is_date(1, 5, 2022)) + A;
}:DATE AS [Дата],
{0}:INT AS [Счетчик]

FROM NEW
{
    Return is_dos_date() - is_date(1, 5, 2022);
}

TOTAL GROUP BY
    [Дата]DESC
```
