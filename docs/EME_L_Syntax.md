## Переменные и комментарии в языке EME-L

Переменные в языке EME-L служат для идентификации данных в памяти. Когда говорится, что мы помещаем или извлекаем значение из переменной, подразумевается, что происходит извлечение или помещение данных в ячейку памяти данных, ассоциированную с именем переменной.

Имена переменных, is-функций и методов классов EME-L могут содержать только буквы от `a` до `z` и от `A` до `Z`, цифры от `0` до `9`, а также знак подчеркивания `_`. Переменные, входящие в состав выражения, должны быть определены ранее.

в языке EME-L ссылки могут быть определены только при декларации метода. Справа от имени переменной надо добавить через пробел ключевое слово `ByRef`.

### Способы определения переменных в EME-L

Определить переменную в EME-L можно пятью способами:

1. **Декларация** — `Variable;` (переменной присваивается значение `Empty`).
2. **Присваивание** — `Variable = 10;`.
3. **Декларация метода** — `SendWebRequest(address, Params)`.
4. **Перехват исключений** — `Try(Err) ... Catch`.
5. **Формирование запроса** — `Select (Query) ... End Select`.

Декларации переменных в списке формальных параметров метода отделяются через запятую:

```EME-L
CheckDocument(Document As "Document", Result ByRef)
```

Декларация переменной внутри метода заканчивается символом `;`:

```EME-L
DocLines AsExternal As "DocLines";
```

Ключевое слово `ByRef` может быть использовано только при декларации метода. Ключевое слово `AsExternal` может быть использовано только внутри метода.

в языке EME-L комментарии бывают двух видов:

- `'Текст комментария'` — используется внутри методов.
- `/* Текст комментария */` — используется вне методов.

## Типы данных в языке EME-L

Прежде чем рассматривать типы данных языка EME-L, следует рассмотреть класс `COleVariant` и его взаимодействие с EME-L. Класс `COleVariant` инкапсулирует тип данных `VARIANT`. В объектах этого класса содержатся как данные, так и сведения об их типе (член `VARTYPE vt`).

В EME-L класс `COleVariant` используется для хранения переменных, констант, литералов и т.д. Описание переменных в EME-L не требует явного указания типа данных.

в языке EME-L примеры объявления переменных:

```EME-L
Variable;       'Тип – Empty,   значение - нет'
Variable = 10;  'Тип – Integer, значение - 10'
Variable = 10.1;'Тип – Real,    значение - 10.1'
Variable = "10";'Тип – String,  значение - "10"'
```

### Тип данных Empty

`Empty` — пустое значение. Переменные, которые были определены, но которым еще не было присвоено ни одного значения, имеют тип `Empty`. Если нужно явно задать переменную с пустым значением — можно использовать встроенную константу `EMPTY`.

в языке EME-L характеристики типа Empty:

| Среда | Соответствие |
|-------|-------------|
| C++ | `void` |
| COleVariant | `VT_EMPTY` |
| EME-DB | отсутствует |

```EME-L
Variable = "EMPTY"; 'Тип – String, значение – "EMPTY"'
Variable = EMPTY;   'Тип – Empty,  значение - нет'
```

### Тип данных Integer

`Integer` — целое число. Литералы целых чисел бывают двух типов — десятичные и шестнадцатеричные. Десятичные литералы могут содержать цифры от `0` до `9`. Шестнадцатеричные литералы должны иметь префикс `0x` и могут содержать цифры от `0` до `9`, а также буквы от `A` до `F`.

в системе EME.WMS характеристики типа Integer:

| Среда | Соответствие |
|-------|-------------|
| C++ | `long` |
| COleVariant | `VT_I4` (значение в `VARIANT::lVal`) |
| EME-DB | `THIS_IS_REFERENCE_ATTRIBUTE`, `CHAIN_BEGIN_ATTRIBUTE`, `CHAIN_BACK_REF_ATTRIBUTE`, `INT_ATTRIBUTE`, `SHORT_ATTRIBUTE` |

Для облегчения программирования EME-DB в EME-L встроены константы: `NULL_REF`, `NULL_RECORD`, `NEW_LINE_MARK`, `NULL_FIELD`.

```EME-L
Variable = 1234;  'Тип – Integer, значение – 1234'
Variable = 0x1234;'Тип – Integer, значение - 4660'
```

### Тип данных Real

`Real` — число с плавающей точкой.

в языке EME-L характеристики типа Real:

| Среда | Соответствие |
|-------|-------------|
| C++ | `double` |
| COleVariant | `VT_R8` (значение в `VARIANT::dblVal`) |
| EME-DB | `FLOAT_ATTRIBUTE`, `MONEY_ATTRIBUTE` |

Литерал числа с плавающей точкой должен иметь форму: `{Целая часть}.{Дробная часть}`, где обе части могут содержать цифры от `0` до `9`.

### Тип данных String

`String` — набор символов, заключенный в двойные кавычки.

в системе EME.WMS характеристики типа String:

| Среда | Соответствие |
|-------|-------------|
| C++ | `char*` |
| COleVariant | `VT_BSTR` (значение в `VARIANT::bstrVal`) |
| EME-DB | `TEXT_STRING_ATTRIBUTE` |

Для ввода в строку специальных символов можно использовать обратный слэш (его нужно экранировать).

```EME-L
Variable = "STRING"; 'Тип – String, значение – "STRING"'
```

### Тип данных Boolean

`Boolean` — предназначен для хранения логических значений (`TRUE` — истина, `FALSE` — ложь).

в языке EME-L характеристики типа Boolean:

| Среда | Соответствие |
|-------|-------------|
| C++ | `BOOL` |
| COleVariant | `VT_BOOL` (значение в `VARIANT::boolVal`) |
| EME-DB | отсутствует |

Для обозначения значения ИСТИНА используется константа `TRUE`, а для значения ЛОЖЬ — `FALSE`. В отличие от C++ константа `TRUE` соответствует `VARIANT_TRUE`, которая равна целому числу `-1`.

### Тип данных DateTime

`DateTime` — предназначен для хранения даты и времени.

в EME БД характеристики типа DateTime:

| Среда | Соответствие |
|-------|-------------|
| C++ | `double` |
| COleVariant | `VT_DATE` (значение в `VARIANT::date`) |
| EME-DB | `DATE_ATTRIBUTE`, `TIME_ATTRIBUTE` |

Дата и время хранятся в виде числа с плавающей точкой: целая часть — количество дней от 31.12.1899, дробная часть — часть дня. Дата не может быть ранее 01.01.0100 и не позднее 31.12.9999. Литералы для типа `DateTime` не предусмотрены — используются соответствующие is-функции.

### Тип данных Array

`Array` — байтовый массив.

в языке EME-L характеристики типа Array:

| Среда | Соответствие |
|-------|-------------|
| C++ | `unsigned char[]` |
| COleVariant | `VT_ARRAY|VT_UI1` (значение в `VARIANT::parray`) |
| EME-DB | `FLAGS_ATTRIBUTE`, `ARRAY_ATTRIBUTE` |

Литералы не предусмотрены. Используется функция `is_array`:

```EME-L
Variable = is_array(255, 1, 127); 'Тип – Array[3], значение – FF 01 7F'
```

### Тип данных Dispatch

`Dispatch` — объект COM-Автоматизации.

в системе EME.WMS характеристики типа Dispatch:

| Среда | Соответствие |
|-------|-------------|
| C++ | `IDispatch*` |
| COleVariant | `VT_DISPATCH` (значение в `VARIANT::pdispVal`) |
| EME-DB | отсутствует |

Предназначен для доступа к серверам COM-Автоматизации.

### Тип данных Object

`Object` — предназначен для доступа к объектам EME-L классов и классов, экспортируемых в EME-L.

в языке EME-L характеристики типа Object:

| Среда | Соответствие |
|-------|-------------|
| C++ | `CEME*` |
| COleVariant | `VT_NULL` (значение в `VARIANT::byref`) |
| EME-DB | отсутствует |

Ключевое слово `Object` создает объект класса и возвращает указатель на него. Формат:

```EME-L
Object({Класс}[, {Аргумент1}, [{Аргумент2} …]])
```

### Тип данных Reference

`Reference` — предназначен для доступа к данным других типов по ссылке. Используется при передаче аргументов вызова, чтобы сделать возможной их модификацию. Попытка выполнения операций над переменными этого типа вызывает ошибку. Единственной доступной операцией является операция `ValOf`. Описание переменных этого типа требует явного указания типа доступа к данным через ключевое слово `ByRef`.

в EME БД характеристики типа Reference:

| Среда | Соответствие |
|-------|-------------|
| C++ | `COleVariant*` |
| COleVariant | `VT_VARIANT | VT_BYREF` (значение в `VARIANT::byref`) |
| EME-DB | отсутствует |

## Типы объектов в EME-L

Тип объекта — это имя класса C++, экспортируемого в EME-L. Хотя описание переменных в EME-L не требует явного указания типа данных, тем не менее EME-L требует от программиста задать тип объекта в том случае, если используются объекты в программе. Это необходимо для проверки на этапе компиляции.

в языке EME-L для определения типа объекта используется ключевое слово `As` после имени переменной. Тип объекта указывается после ключевого слова `As` в виде текстовой строки, заключенной в двойные кавычки.

```EME-L
'Тип объекта определяет компилятор'
r_Document = Object("dsDB", "Document");
'Тип объекта определяет программист'
String As "String";
```

## Операции в EME-L

Выражение состоит из операндов, операций и скобок. Порядок вычислений определяется приоритетом операций. Операции бывают двух видов — унарные и бинарные.

### Неявное преобразование типов в языке EME-L

Неявное преобразование вступает в силу, когда типы операндов в бинарной операции не совпадают. Приоритеты правил:

1. Если один из операндов имеет тип `String` — другой приводится к `String` (например: `1234` → `"1234"`).
2. Если один из операндов имеет тип `Real` или `DateTime` — оба преобразуются к `Real`.
3. Если один из операндов имеет тип `Integer` или `Boolean` — оба преобразуются к `Integer`.
4. Во всех остальных случаях — оба операнда преобразуются к типу `Array`.

Для логических операций операнды преобразуются к типу `Boolean`: `0` воспринимается как `FALSE`, не `0` — как `TRUE`.

### Приоритет операций в EME-L

Операции в порядке убывания приоритета (римская цифра — приоритет, I — наивысший):

| Операция | Символ | Приоритет | Описание |
|----------|--------|-----------|----------|
| Логическое отрицание | `~` | I | Унарная |
| Унарный минус | `-` | I | Унарная |
| Унарный плюс | `+` | I | Фиктивная операция |
| Умножение | `*` | II | Произведение операндов |
| Деление | `/` | II | Частное от деления |
| Сложение | `+` | III | Сумма операндов |
| Вычитание | `-` | III | Разность операндов |
| Меньше | `<` | IV | Сравнение |
| Меньше или равно | `<=` | IV | Сравнение |
| Больше | `>` | IV | Сравнение |
| Больше или равно | `>=` | IV | Сравнение |
| Равно | `==` | V | Сравнение |
| Не равно | `!=` | V | Сравнение |
| Логическое И | `&` | VI | Бинарная логическая |
| Логическое исключающее ИЛИ | `^` | VII | Бинарная логическая |
| Логическое ИЛИ | `\|` | VIII | Бинарная логическая |
| RefOf | `RefOf` | IX | Возвращает ссылку на переменную |
| ValOf | `ValOf` | IX | Возвращает значение по ссылке |

Операции с одинаковым приоритетом выполняются слева направо.

### Операнды в выражениях EME-L

в языке EME-L в качестве операндов могут выступать:

- Литералы (`1234`, `"1234"`, `1.234`)
- Переменные
- Логический оператор `If` (`If(Cond, True, False)`)
- is-функции (`is_dos_date()`)
- Оператор `Const` — вычисляет выражение один раз за время работы программы (`Const(is_dos_date())`)
- Объект EME-L класса (`Object("Automation", "VSS.VSSDatabase")`)
- Вызовы методов классов (`Object().GetContext()`, `Parent::GetMethod()`)
- Выражения (`2*2`, `(-Variant+1234)`)

### Особенности сложения разных типов в EME-L

**Левый операнд DateTime:** неявное преобразование не производится. Правый операнд приводится к `Real`, результат — `DateTime`.

```EME-L
Variable = is_date(1, 1, 2001) + "13";
'Тип – DateTime, значение – 14.01.2001'
```

**String:** возвращает строку — результат конкатенации (слияния) строк-операндов.

```EME-L
Variable = 2 + "-EME-" + 2; 'Тип – String, значение – "2-EME-2"'
```

**Real, Integer:** возвращает сумму операндов.

### Особенности вычитания разных типов в EME-L

**Левый операнд DateTime:** если правый операнд — `DateTime`, возвращается `Real` (разность). Иначе правый приводится к `Real`, возвращается `DateTime`.

```EME-L
Variable = is_date(14, 1, 2001) - is_date(1, 1, 2001, 12, 0, 0);
'Тип – Real, значение – 12.5'
Variable = is_date(14, 1, 2001) - "12.5";
'Тип – DateTime, значение – 01.01.2001 12:00:00'
```

## Операторы и программа в EME-L

Программа на EME-L — это последовательность операторов. Повторяющиеся фрагменты кода можно поместить в метод класса.

### Формат метода класса EME-L

```EME-L
{Метод}({Аргумент1}, [{Аргумент2} ..])
{
    ...
}
```

### Вызовы методов классов

**Вызов метода текущего объекта:**
```EME-L
[{Область}::]{Метод}([{Аргумент1}, [{Аргумент2} ...]])
```
где `{Область}` — имя одного из классов в цепочке наследования.

**Вызов метода внешнего объекта:**
```EME-L
{Объект}.{Метод}([{Аргумент1}, [{Аргумент2} ...]])
```
где `{Объект}` — операнд типа `Object` или `Dispatch`.

### Оператор присваивания (=)

в языке EME-L каждый из следующих операторов является оператором присваивания:

```EME-L
Document = Object("dsDB"); 'создать объект'
is_transaction(1);         'открыть транзакцию'
Date = is_dos_date() - 1;  'рассчитать дату'
Document.PutDate(Date);    'сохранить дату в БД'
is_transaction(-1);        'закрыть транзакцию'
```

### Оператор ByVal

Оператор `ByVal` применяется в операторе присваивания для присваивания значения, переданного по ссылке. Ключевое слово `ByVal` указывается справа от имени переменной типа `Reference` через пробел.

### Оператор AsExternal

Оператор `AsExternal` применяется для инициализации переменной, доступной в запросах на языке EME-SQL (выполняются при помощи оператора `Select`). Может быть использован только внутри метода.

### Оператор As

Оператор `As` применяется для определения типа объекта. Тип указывается после `As` через пробел, заключенный в двойные кавычки.

### Оператор If ... Else ... End If

Условный оператор `If ... Else ... End If` вычисляет значение выражения и проверяет его по условию. Результат приводится к типу `Boolean`. Если `TRUE` — выполняются операторы между `If` и `Else`, иначе — между `Else` и `End If`.

```EME-L
ClientRef;
PLAT_RECORD = 42; 'запись – Платежи наличные'
If (Doc.GetRecord() != PLAT_RECORD)
If (Doc.TuneVendorToRef())
    ClientRef = Doc.GetVendorToRef();
Else
    ClientRef = Doc.GetClientRef();
End If
Else
If (Doc.GetPayer().IsVendor())
    ClientRef = Doc.GetRecipientRef();
Else
    ClientRef = Doc.GetPayerRef();
End If
End If
```

### Оператор While ... End While

Оператор цикла `While ... End While` выполняет тело цикла до тех пор, пока результат выражения, приведенный к типу `Boolean`, равен `TRUE`.

```EME-L
Count = 0;
Doc.SetFirstLine();
While (Doc.IsValidLine())
    If (Doc.IsClosed())
        Count = Count + 1;
    End If
    Doc.SetNextLine();
End While
```

### Оператор For ... End For

Оператор цикла `For ... End For` выполняет тело цикла до тех пор, пока результат выражения, приведенный к типу `Boolean`, равен `TRUE`.

```EME-L
Count = 0;
For (Doc.SetFirstLine(); Doc.IsValidLine(); Doc.SetNextLine())
If (Doc.IsClosed())
    Count = Count + 1;
End If
End For
```

### Оператор Loop ... End Loop

Оператор цикла `Loop ... End Loop` предназначен для обхода всех строк в классах записей ЕМЕ БД. По сути, это тот же `For`, для которого не нужно прописывать методы `SetFirstLine()`, `IsValidLine()` и `SetNextLine()`.

в системе EME.WMS метод `SetFirstLine()` выполняется перед первым выполнением тела цикла. Метод `SetNextLine()` выполняется после тела цикла перед проверкой `IsValidLine()`.

```EME-L
Count = 0;
Loop (Doc)
If (Doc.IsClosed())
    Count = Count + 1;
End If
End Loop
```

### Оператор Break

Оператор `Break` используется внутри циклов `While` и `For` для прекращения исполнения тела цикла. Управление передается оператору, следующему после цикла.

```EME-L
Doc.SetFirstLine();
While (Doc.IsValidLine())
   If (Doc.IsClosed())
      Break
   End If
   Doc.SetNextLine();
End While
```

### Оператор Continue

Оператор `Continue` используется внутри циклов `While` и `For` для перехода на начало цикла. Операторы между `Continue` и концом цикла не выполняются.

```EME-L
Count = 0;
Line = Doc.GetNoOfLines();
While (Line != 0)
   Line = Line - 1;
   Doc.SetLine(Line);
   If (~Doc.IsValidLine())
      Continue
   End If
   Count = Count + 1;
End While
```

### Оператор Return

Оператор `Return` используется для завершения работы программы или метода класса EME-L. Сокращенная форма `Return;` возвращает значение типа `Empty`.

```EME-L
/* Метод GetLineSumma возвращает сумму строки документа */
GetLineSumma(DocLine)
{
   Return DocLine.GetQty() * DocLine.GetPrice();
}
/* Метод GetDocSumma возвращает сумму всего документа */
GetDocSumma(Document)
{
   Summa = 0.;
   DocLine = Document.GetDocLines();
   DocLine.SetFirstLine();
   While (DocLine.IsValidLine())
      Summa = Summa + GetLineSumma(DocLine);
      DocLine.SetNextLine();
   End While
   Return Summa;
}
```

### Оператор Try ... Catch

Оператор `Try ... Catch` отлавливает исключения `CException` и `CEMException`. При инициализации блока `Try` нужно определить переменную для текста исключения (не должна быть инициализирована ранее).

в языке EME-L пример обработки исключения:

```EME-L
Try (Error)
    Variable = 1/Variable;
Catch
If (~is_empty(Error))
    is_message("ОШИБКА!!!", Error, 1);
End If
```

### Оператор Switch ... Case ... End Switch

Оператор `Switch ... Case ... End Switch` применяется, когда не нужно значительно менять поведение программы на основе значения выражения. Ключевое слово `Default` — необязательно.

```EME-L
ClientRef;
PLAT_RECORD = 42;
Switch (Doc.GetRecord())
Case 48:
Case PLAT_RECORD:
   If (Doc.TuneVendorToRef())
      ClientRef = Doc.GetVendorToRef();
   Else
      ClientRef = Doc.GetClientRef();
   End If
Default:
   If (Doc.GetPayer().IsVendor())
      ClientRef = Doc.GetRecipientRef();
   Else
    ClientRef = Doc.GetPayerRef();
   End If
End Switch
```

### Оператор Select ... End Select

Оператор `Select ... End Select` выполняет запрос, написанный непосредственно в коде класса EME-L. С ключевым словом `AsExternal` выполненный запрос доступен из других запросов.

в системе EME.WMS пример формирования запросов через EME-L код:

```EME-L
Select (Query AsExternal)
    SELECT
    [@Тип документа].[Наименование] AS [Тип]
    FROM
    [Документы]
End Select

Select (Query2)
    SELECT
    [Тип]
    FROM
    [Query]
    GROUP BY
    [Тип]
End Select
```

### Оператор Procedure .. End Procedure

Оператор `Procedure .. End Procedure` выполняет на сервере процедуру, написанную в коде класса EME-L. Текст процедуры — блок кода на EME-L. Процедура компилируется на рабочей станции одновременно с кодом класса, хранится в коде класса и передается для выполнения на сервер. Возвращаемое значение всегда равно `EMPTY`.

в языке EME-L параметры оператора Procedure могут быть трех типов:
- **Выражение** — доступ через `argsN` (N — номер, начиная с 0).
- **Имя переменной** — доступ напрямую по имени.
- **Переменная по ссылке** (`RefOf`) — изменения передаются с сервера.

```EME-L
x AsExternal = 1;
y = 2;
z = 3;
procedure (y, RefOf z, z + 1)
z = x + y + z + args2;
end procedure
is_message("Результат выполнения процедуры", z, 1);
```
