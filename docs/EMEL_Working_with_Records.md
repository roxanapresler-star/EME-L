# Работа с записями базы данных в EME-L

**Описание:** Записи базы данных (Records) — основные единицы хранения данных в EME.WMS. Работа с записями осуществляется через объекты класса CEMERec и его наследников.
**Сленг:** запись, рекорд, CEMERec, строка записи, таблица БД

## Создание и структура записей

Записи создаются через диалог "Структура Базы Данных" (Ctrl-Shift-S). Каждая запись содержит поля определённых типов.

**Синонимы для поиска:** создание записи, структура записи, новая запись, поля записи, типы полей, редактор структуры БД

### Создание новой записи

1. Открыть диалог: Система → Ресурсы → Структура Базы Данных
2. Нажать зелёный крестик или Ctrl-N
3. Ввести имя записи
4. Сохранить: Ctrl-S

### Создание полей записи

1. Выбрать запись в диалоге структуры БД
2. Нажать "Редактирование полей"
3. Указать название поля и атрибут (тип данных)
4. Сохранить: Ctrl-S

### Типы атрибутов полей в EME БД

| Атрибут | Размер | Описание |
|---------|--------|----------|
| Целое число | 2 или 4 байта | Целочисленные значения |
| Число с плавающей точкой | 8 байт | Дробные числа |
| Строка текста | Задаётся | Текстовые данные |
| Массив | Задаётся | Массив значений |
| Деньги | 8 байт | Денежные значения |
| Ссылка | 4 байта | Ссылка на строку другой записи |
| Дата | 4 байта | Дата |
| Время | 4 байта | Время |

## Режимы обхода записей в EME-L

Для работы с записями в EME-L используются различные режимы обхода, которые определяют способ навигации по строкам.

**Синонимы для поиска:** обход записи, SetRealMode, SetSkipMode, SetChainMode, SetFindMode, перебор строк, итерация по записи

### Сравнение режимов обхода

| Режим | Скорость | Использование в цикле | Применение |
|-------|----------|----------------------|------------|
| SetRealMode | Медленно | Да | Полный перебор без условий |
| SetSkipMode | Средне | Нет | Поиск по идентификатору |
| SetSkipMode("Индекс") | Быстро | Да | Поиск по индексу |
| SetChainMode | Очень быстро | Да | Перебор строк одного документа |
| SetFindMode | Быстро | Нет | Поиск ≥10 значений без индекса |

### SetRealMode — обход по физике

```eme-l
' Обход всех строк записи подряд в EME-L
r_GoodsItem = Object("dsDB", "GoodsItem");
r_GoodsItem.SetRealMode();

For (r_GoodsItem.SetFirstLine(); r_GoodsItem.IsValidLine(); r_GoodsItem.SetNextLine())
    ' Обработка каждой строки
    name = r_GoodsItem.GetNameFld();
    ' ... бизнес-логика ...
End For
```

**Важно:** `SetRealMode()` читает все строки, включая удалённые. Используйте только для полного перебора без условий.

### SetSkipMode — обход с условиями

```eme-l
' Поиск документов по типу и дате в EME-L
r_Document = Object("dsDB", "Document");
r_Document.SetSkipMode();

' Добавляем условия фильтрации
r_Document.GetDocumentTypeRefFld().MustBeRefEQ(0); ' Тип = Приход
r_Document.GetCreateDateFld().MustBeGE(startDate);
r_Document.GetCreateDateFld().MustBeLE(endDate);
r_Document.MustBeValid(); ' Только неудалённые

' Устанавливаем на первую найденную строку
r_Document.SetFirstLine();

While (r_Document.IsValidLine())
    ' Обработка найденного документа
    docNo = r_Document.GetRegNo();
    
    ' Переход к следующей найденной строке
    r_Document.SetNextLine();
End While
```

**Важно:** `SetSkipMode()` нельзя использовать внутри цикла! Режим устанавливается один раз перед циклом.

### SetSkipMode по индексу

```eme-l
' Быстрый поиск по индексированному полю в EME-L
r_Stamps = Object("dsDB", "Stamps");
r_Stamps.SetSkipMode("Индекс номер марки");

' Условие по индексированному полю
r_Stamps.GetNumberFld().MustBeEQ(stampNumber);
r_Stamps.SetFirstLine();

If (r_Stamps.IsValidLine())
    ' Марка найдена
    Return r_Stamps.GetLine();
End If
```

### SetSkipMode по нескольким полям индекса

```eme-l
' Поиск по составному индексу в EME-L
r_Stamps = Object("dsDB", "Stamps");
r_Stamps.SetSkipMode("Индекс партия - номер");

' Используем MustBeIndexEQ для нескольких полей
r_Stamps.MustBeIndexEQ(lotRef, serialNo);
r_Stamps.SetFirstLine();
```

### SetChainMode — обход по цепочке

```eme-l
' Обход строк документа по цепочке в EME-L
r_Document = Object("dsDB", "Document");
r_Document.SetLine(docRef);

' Получаем строки документа через реляционный путь
r_DocLines = r_Document.GetDocLines();

' Перебираем строки в цепочке (очень быстро!)
Loop(r_DocLines)
{
    ' Обработка строки документа
    qty = r_DocLines.GetQtyN();
    goodsItem = r_DocLines.GetGoodsItemRefFld();
    
    ' ... бизнес-логика ...
}
```

**Важно:** `SetChainMode()` — самый быстрый способ обхода. Используйте для перебора строк одного документа, заказа и т.д.

### SetFindMode — поиск по хэш-таблице

```eme-l
' Поиск нескольких значений с построением хэш-таблицы в EME-L
r_GoodsItem = Object("dsDB", "GoodsItem");
r_GoodsItem.SetFindMode("CodeFld", "NameFld");

' Поиск выполняется один раз перед циклом
' При первом поиске строится хэш-таблица

For (arrCodes.SetFirstLine(); arrCodes.IsValidLine(); arrCodes.SetNextLine())
    code = arrCodes.Get();
    
    ' Поиск в хэш-таблице (быстро!)
    r_GoodsItem.FindByValues(code);
    
    If (r_GoodsItem.IsValidLine())
        ' Товар найден
    End If
End For
```

## Методы фильтрации в SetSkipMode

При использовании `SetSkipMode()` можно накладывать различные условия фильтрации.

**Синонимы для поиска:** фильтрация, MustBe, условия поиска, фильтр записи, условия отбора

### Основные методы фильтрации

| Метод | Описание |
|-------|----------|
| `MustBeEQ(значение)` | Равно значению |
| `MustBeNE(значение)` | Не равно значению |
| `MustBeGE(значение)` | Больше или равно |
| `MustBeLE(значение)` | Меньше или равно |
| `MustBeGT(значение)` | Строго больше |
| `MustBeLT(значение)` | Строго меньше |
| `MustBeRefEQ(ссылка)` | Ссылка равна |
| `MustBeRefNE(ссылка)` | Ссылка не равна |
| `MustBeRefInBitBuffer(bb)` | Ссылка в битовом буфере |
| `MustBeBitSet(бит)` | Бит установлен |
| `MustBeValid()` | Строка не удалена |
| `MustBeInRange(от, до)` | В диапазоне |

### Пример сложной фильтрации

```eme-l
' Многоуровневая фильтрация в EME-L
r_DocLines = Object("dsDB", "DocLines");
r_DocLines.SetSkipMode();

' Условия фильтрации (важен порядок!)
r_DocLines.GetDocumentRefFld().MustBeRefEQ(docRef);  ' Строки документа
r_DocLines.GetGoodsItemRefFld().MustBeRefInBitBuffer(bbGoods); ' Товары из списка
r_DocLines.GetQtyFld().MustBeGT(0);  ' Количество > 0
r_DocLines.MustBeValid();  ' Не удалённые

r_DocLines.SetFirstLine();
```

## Ограничение области поиска

Для оптимизации поиска можно ограничить область поиска методами:

**Синонимы для поиска:** ограничение поиска, SetLinesLimit, SetLine, оптимизация SkipMode

### Методы ограничения

```eme-l
' Ограничение области поиска в EME-L
r_Lines = Object("dsDB", "LargeRecord");
r_Lines.SetSkipMode();
r_Lines.GetNameFld().MustBeEQ(name);

' Ограничение начальной позиции
nStartLine = r_Lines.GetNoOfLines() - 4000000;
If (nStartLine > 0)
    r_Lines.SetLine(nStartLine);
    r_Lines.SetLinesLimit(2000000);
    r_Lines.SetNextLine();
Else
    r_Lines.SetFirstLine();
End If
```

**Важно:** При использовании `SetLine()` нельзя вызывать `SetFirstLine()`!

## Работа со ссылочными полями

Ссылочные поля обеспечивают связи между записями в EME БД.

**Синонимы для поиска:** ссылка, reference, ссылочное поле, связь записей, цепочка

### Типы ссылочных полей

| Тип | Описание | Префикс |
|-----|----------|---------|
| Ссылка на начало цепочки | Первая строка в цепочке | @ |
| Ссылка на заголовок цепочки | Родительская запись | @ |
| Ссылка (простая) | Произвольная строка | @ |

### Создание связи "один-ко-многим"

```eme-l
' Пример структуры связи в EME БД:
' Запись "Клиенты":
'   Поле "@Звонки клиентов" - Ссылка на начало цепочки
'   Указывает на запись "Звонки клиентов", поле "@Клиент"

' Запись "Звонки клиентов":
'   Поле "@Клиент" - Ссылка на заголовок цепочки
'   Поле "Сортировка по клиенту" - Сортировка-цепочка
'     Первое поле: @Клиент
'     Второе поле: Дата
```

### Навигация по ссылкам

```eme-l
' Чтение по реляционному пути в EME-L
r_Document = Object("dsDB", "Document");
r_Document.SetLine(docRef);

' Получение связанных данных через Get-методы
warehouseName = r_Document.GetWarehouse().GetName();
gatesCode = r_Document.GetGates().GetCode();
clientName = r_Document.GetClient().GetNameFld();
```

## Создание и удаление строк

Операции создания и удаления строк выполняются внутри транзакций.

**Синонимы для поиска:** создание строки, удаление строки, CreateNewLine, ApplyNewLine, DeleteLine

### Создание новой строки

```eme-l
' Создание новой строки записи в EME-L
is_transaction(1, "Создание нового товара");

r_GoodsItem = Object("dsDB", "GoodsItem");

' Создание новой строки
r_GoodsItem.CreateNewLine();

' Заполнение полей
r_GoodsItem.PutCodeFld("ART-001");
r_GoodsItem.PutNameFld("Товар А");
r_GoodsItem.PutBaseUnitRefFld(unitRef);

' Применение новой строки
r_GoodsItem.ApplyNewLine();

is_transaction(-1);
```

### Удаление строки

```eme-l
' Удаление строки записи в EME-L
is_transaction(1, "Удаление товара");

r_GoodsItem = Object("dsDB", "GoodsItem");
r_GoodsItem.SetLine(goodsRef);

' Проверка перед удалением
If (r_GoodsItem.IsValidLine())
    ' Удаление строки
    r_GoodsItem.DeleteLine();
End If

is_transaction(-1);
```

### Мягкое удаление (SetDeleted)

```eme-l
' Мягкое удаление (без физического удаления) в EME-L
r_DocLines = Object("dsDB", "DocLines");
r_DocLines.SetLine(lineRef);
r_DocLines.SetDeleted();  ' Помечаем как удалённую

' Преимущества: 
' - Быстрее физического удаления
' - Можно восстановить
' - Не нарушает сортировки
```

## Влияние удалённых строк на производительность

При работе с записями, содержащими много удалённых строк, производительность может снижаться.

**Синонимы для поиска:** удалённые строки, производительность, оптимизация больших записей, очистка записи

### Проблемы с удалёнными строками

- В `SetRealMode` читаются все удалённые строки
- В `SetSkipMode` первое условие должно отсекать удалённые строки
- Удалённые строки содержат нули и NULL_REF во всех полях

### Решение

```eme-l
' Правильная работа с записью, где много удалённых строк
r_Lines = Object("dsDB", "LargeRecord");
r_Lines.SetSkipMode();

' Первое условие должно отсекать удалённые строки!
r_Lines.GetGoodsItemRefFld().MustBeRefEQ(validRef);  ' NULL_REF будет отсечён
' или
r_Lines.GetFlagsFld().MustBeBitSet(validBit);

r_Lines.SetFirstLine();
```

## Методы Put и Get для полей

Для чтения и записи значений полей используются методы Put и Get.

**Синонимы для поиска:** Put метод, Get метод, чтение поля, запись поля, PutFld, GetFld

### Соглашение об именовании

- `PutFieldNameFld(значение)` — запись значения в поле
- `GetFieldNameFld()` — чтение значения поля
- `PutFieldNameRefFld(ссылка)` — запись ссылки
- `GetFieldNameRefFld()` — чтение ссылки

### Примеры Put и Get

```eme-l
' Работа с полями записи в EME-L
r_Document = Object("dsDB", "Document");
r_Document.SetLine(docRef);

' Чтение значений полей
regNo = r_Document.GetRegNoFld();
createDate = r_Document.GetCreateDateFld();
docType = r_Document.GetDocumentTypeRefFld();

' Запись значений полей
r_Document.PutRegNoFld("DOC-12345");
r_Document.PutCreateDateFld(is_date());
r_Document.PutDocumentTypeRefFld(0);
```

## Совместное использование даты и времени в SkipMode

Для фильтрации по диапазону дат и времени в SkipMode используйте метод `AttachTimeFld()` для связывания поля даты с полем времени.

**Синонимы для поиска:** дата-время, AttachTimeFld, пропуск по времени, фильтр по дате и времени

### Метод AttachTimeFld

Метод `AttachTimeFld()` привязывает поле времени к полю даты, что позволяет использовать `MustBeGT` и `MustBeLT` с полными значениями дата+время.

```eme-l
' Совместное использование даты-времени в SkipMode
r_Document = Object("dsDB", "Document");
r_Document.SetSkipMode();

' Привязываем поле времени к полю даты
r_Document.GetCreateDateFld().AttachTimeFld();

' Теперь можно фильтровать по дате и времени
r_Document.GetCreateDateFld().MustBeGT(is_date(19, 11, 2018, 11, 0));
r_Document.GetCreateDateFld().MustBeLT(is_date(20, 11, 2018, 12, 0));
r_Document.SetFirstLine();
```

**Варианты вызова AttachTimeFld:**

- `AttachTimeFld()` — автоматически ищет поле времени по имени (заменяет "Дата" на "Время")
- `AttachTimeFld(r_Document.GetCreateTimeFld())` — явно указывает поле времени
- `AttachTimeFld("Время создания")` — указывает имя поля времени
- `AttachTimeFld(61)` — указывает номер поля времени

## Особенности вычисления дробного количества

Количество в WMS хранится в поле типа double, но при арифметических действиях к количеству должна применяться целочисленная логика с оговоркой, что количество может содержать заданное число знаков после запятой.

**Синонимы для поиска:** дробное количество, округление, RoundDn, Round, распределение количества

### Проблема округления

При делении дробного количества может получиться остаток, который нельзя отбросить или распределить равномерно. Например, 5.1 / 2 = 2.55, после округления до одного знака получается 2.5, и остаток 0.1 нужно куда-то распределить.

Типовая ошибка — работать с дробным количеством как с числами с плавающей точкой, забывая про округление, или округлять в конце без учёта остатка и накапливаемой погрешности.

### Алгоритм распределения количества

```eme-l
' Распределение количества qty на N строк
qtyLine = r_GoodsItemMU.RoundDn(qty / N);
nLine = 0;

Loop(r_Doclines)
{
    nLine = nLine + 1;
    If (nLine < N)
        ' Запишем вычисленное кол-во во все строки, кроме последней
        r_Doclines.PutQty(qtyLine);
        qty = qty - qtyLine;
    Else
        ' В последнюю строку записываем весь остаток
        ' Округляем, так как при многократном вычитании погрешность накапливается
        qtyLine = r_GoodsItemMU.Round(qty);
        r_Doclines.PutQty(qtyLine);
    End If
End Loop
```

## Хранилище (Storage) — новый тип поля БД

Тип поля "Хранилище" предназначен для хранения больших объёмов данных (изображения, файлы, документы). Это альтернатива устаревшему типу VLP.

**Синонимы для поиска:** хранилище, Storage, большое поле, BLOB, изображение, файл в БД

### Назначение

Поле типа "Хранилище" позволяет хранить в БД данные произвольного размера: изображения, документы, файлы. В отличие от текстовых полей, данные хранилища хранятся отдельно от основных данных записи, что не замедляет работу с записью.

### Миграция с VLP

При переходе с полей типа VLP на тип "Хранилище" данные автоматически мигрируются при первом обращении к полю. Обратная совместимость сохраняется.

## Классы записей (продвинутые вопросы)

Классы записей в EME.WMS строятся на базе CEMERec с помощью макросов Си++.

**Синонимы для поиска:** CEMERec, классы записей, макросы, INNER, PROXY, метапроект

### IsValidLine — когда нужно проверять

При чтении по реляционному пути **никогда не нужно** проверять объект на `IsValidLine()`. Ссылка может быть `NULL_REF`, и ошибки не будет — методы вернут пустое значение.

Проверять на `IsValidLine()` нужно только если объект настраивается на строку методами `SetLine()`, `SetFirstLine()`, `SetNextLine()`.

```eme-l
' НЕ НУЖНО проверять при чтении по реляционному пути
code = r_Document.GetGates().GetCode();  ' Верно, даже если NULL_REF

' НУЖНО проверять после SetLine
r_Document.SetLine(docRef);
If (r_Document.IsValidLine())  ' Обязательно!
    ' Обработка
End If
```
