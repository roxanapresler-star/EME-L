## Объектная модель EME-L

Объектная модель EME-L имеет следующие важнейшие характеристики по созданию, выполнению, управлению и доступу к объектам:

- **Песочница (sand box):** для выполнения каждого объекта EME-L создается своя копия виртуальной машины EME-L.
- **Многопоточность:** объектная модель EME-L поддерживает многопоточность. Каждый поток создает свои, независимые от других потоков, объекты (threading model — apartment). Интерфейс пользователя EME БД является многодокументным (основан на MDI), поэтому объекты EME-L интерфейсного потока образуют независимые группы, определяемые по окну основного документа.
- **Управление объектами:** сочетание учета ссылок и сборки мусора.
- **Доступ in process:** доступ к объектам EME-L возможен только из ЕМЕ БД. Доступ из другого процесса невозможен.

в языке EME-L возможно создание объектов только экспортируемых в EME-L и зарегистрированных классов C++ метапроекта ЕМЕ БД.

### Регистрация классов в EME-L

Регистрация классов, экспортируемых в EME-L, осуществляется в файле `CEMEPrj.cpp` в проекте `MetC_Sys`. Уникальным идентификатором является имя класса.

Классы EME-L хранятся в специальной записи БД под названием "Классы". Косвенно эта запись выполняет функции регистрации классов EME-L — можно создать объект только такого класса EME-L, который есть в этой записи.

### Создание объектов в EME-L

в языке EME-L для создания объекта используется ключевое слово `Object`. Указывается название класса и аргументы конструктора.

```EME-L
Object("Map", 1013, String);
```

### Класс IClass — доступ к другому классу EME-L

Чтобы внутри класса EME-L обратиться к другому классу, используется объект экспортируемого в EME-L класса `IClass`, который инкапсулирует заданный в аргументах конструктора класс EME-L.

Конструктор `IClass` имеет 3 аргумента:

1. **Объект** — имя объекта класса EME-L.
2. **Класс** — ссылка на класс EME-L в записи "Классы" или имя класса EME-L.
3. **Признак окна** — предназначен для идентификации объектов с одинаковыми именами, имеющих окна (например, расходная накладная). Идентификация осуществляется по apartment id — дескриптору окна корневого родительского диалога. По умолчанию равен `TRUE` (окно обязано существовать на момент создания объекта). Для глобального объекта должен быть равен `FALSE`, тогда apartment id будет равен `NULL`.

в системе EME.WMS рекомендуется в конструкторе `IClass` использовать ссылку на класс, а не имя класса, чтобы избежать неоправданного поиска в записи "Классы".

```EME-L
'Внутри класса CompleteOperations обращаемся к классу CompleteInventOperations'
objEmelCompleteInventOperations = Object("IClass", "CompleteInventOperations", "CompleteInventOperations");
objEmelCompleteInventOperations.OnInit(m_Doc, m_bGetListOrdersKeys, m_TerminalRef, m_CurrTermVer);
objEmelCompleteInventOperations.InitQuery();
```

### Управление объектами в EME-L

в языке EME-L важнейшей характеристикой объектной модели по управлению объектами является сочетание сборки мусора с частичным учетом ссылок. Для управления объектами классов, экспортируемых в EME-L, средствами виртуальной машины EME-L, используется метод виртуальной машины `create_object`.

## Интерфейсы объекта в EME-L

Все классы, экспортируемые в EME-L, являются наследниками класса `CEME` (в C++). В этом классе определены виртуальные методы, являющиеся интерфейсами EME-L.

### Основные интерфейсные методы в языке EME-L

- **FindMethod** — возвращает идентификатор метода (целое число >= 0), если в классе есть метод, признаки которого (имя, количество аргументов, имена аргументов) соответствуют заданным.

- **CallMethod** — вызывает метод класса по его идентификатору и возвращает значение, возвращаемое методом.

- **GetMethodInfo** — используется при статическом связывании для получения информации о всех методах классов C++, экспортируемых в EME-L. Возвращает доступность или недоступность метода для статического связывания.

- **GetNoOfMethods** — возвращает количество методов в классе. Необходим для статического связывания и определения идентификатора метода в классе-наследнике.

- **GetClassName** — возвращает имя класса. Используется в отладчике EME-L.

- **ObjectIsCreated** — применяется в фабрике объекта класса, экспортируемого в EME-L.

- **EnumProperties** — возвращает массив строк имен методов, для вызова которых не требуются аргументы. Такие методы считаются свойствами (properties), если они не изменяют значение данных.

- **GetHelpItem** — создает объект, обслуживающий URL-указатель в интерактивной справочной системе EME БД.

- **GetHelpText** — возвращает HTML-страницу с информацией об объекте.

- **GetHelpPath** — возвращает URL-указатель на HTML-страницу для определенного ключа.

## Контекст объекта в EME-L

в языке EME-L понятие контекста применяется к классам и объектам классов. Класс, экспортируемый в EME-L, может быть или не быть контекстно-зависимым. Объект контекстно-зависимого класса имеет контекст.

### Контекстно-независимые классы

Например, классы `Array` (класс массива) или `String` (класс строки) не являются контекстно-зависимыми.

### Контекстно-зависимые классы

в системе EME.WMS класс `Control` (орган диалога) является контекстно-зависимым. Объект такого класса имеет текущий контекст EME БД. Контекст включает в себя:

- Номер сообщения ядра EME БД
- Указатель на объект диалога EME БД
- Указатель на объект броузера EME БД
- Указатель на объект запроса EME БД
- Указатель на объект отчета EME БД
- Номер записи EME БД
- Номер строки в записи EME БД и другие параметры

в языке EME-L контекст является текущим — значения всех параметров соответствуют определенному текущему состоянию EME БД. Не все параметры контекста могут быть заполнены.

Текущий контекст EME БД можно увидеть в правом верхнем окне отладчика EME-L.
