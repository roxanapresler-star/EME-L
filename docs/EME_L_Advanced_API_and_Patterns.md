## Класс String в языке EME-L

Класс `String` предназначен для работы со строковыми данными. Предоставляет методы для выполнения базовых операций со строками, конвертаций/преобразований, форматирования, проверок символов, поиска данных и их редактирования. в языке EME-L класс String является основным инструментом обработки текста.

### Инициализация объекта String

```EME-L
csString = Object("String")
csString = Object("String", arg)
```

Если строка не задана в конструкторе, её нужно задать методом `SetString`:

```EME-L
csString.SetString(arg_2);
```

Для отладки используется метод `GetContent` — выводит содержимое строки.

### Методы базовых операций

- **Add** — конкатенация строк. Принимает строку, возвращает результирующую строку. Изменяет содержимое объекта.
- **Compare** / **CompareNoCase** — проверка на равенство строк (с учётом и без учёта регистра). Возвращают `True`/`False`.
- **GetLength** — возвращает длину строки.
- **IsEmpty** — проверка на пустую строку. Возвращает `True`/`False`.
- **IsNumericOnly** — проверка, что строка является числом (не работает для чисел с плавающей точкой).

### Методы преобразований и редактирования

- **AnsiToISO88595**, **AnsiToOem**, **AnsiToUTF8** — конвертация из ANSI в целевые кодировки. Изменяют содержимое объекта.
- **AsDateTime**, **AsInteger**, **AsReal** — преобразование строки в различные типы данных.
- **AsString** — конвертация объекта в строку.
- **Format** — форматирование строки с неограниченным числом параметров.

```EME-L
str_temp.Format("ASN %s. Печать товарных этикеток для позиций: %d", RegNo, CountLines);
```

- **MakeCapitalLetter** — первый символ заглавный.
- **MakeLower** — все символы строчные.
- **MakeUpper** — все символы заглавные.
- **MakeReverse** — перевернуть строку.
- **RemoveSymbol** — удаление заданных символов из строки.
- **Replace** — замена всех вхождений подстроки.
- **Split** — разделение строки на подстроки и добавление в массив.
- **TrimLeft** / **TrimRight** — удаление символов с левой и правой части строки.
- **SetAt** — замена символа по указанному индексу.

### Методы поиска данных

- **ExtractUpto** / **ExtractUptoString** — срез строки от начала до символа/подстроки.
- **ExtractUptoEx** — срез с пропуском символов за спецсимволом.
- **Find** — поиск подстроки, возвращает индекс или -1.
- **FindChar** — поиск символа с указанного индекса.
- **FindNoCase** — поиск подстроки без учёта регистра.
- **FindOneOf** — поиск символа без использования индексов.
- **GetAt** — получение символа по индексу.
- **Left** / **Right** — подстрока слева/справа от индекса.
- **Mid** — срез строки с указанного индекса.
- **ReverseFind** / **ReverseFindSubStr** — поиск с конца строки.

## Класс PropertyTree в языке EME-L

Класс `PropertyTree` предназначен для работы с древовидными структурами данных Boost Property Tree в формате JSON. в языке EME-L этот класс является основным для работы с JSON-данными и формирования ответов HTTP-сервера.

### Инициализация

```EME-L
ResultJSON = Object("PropertyTree");
```

### Методы редактирования дерева

- **AddChild** — добавляет новый узел. Возвращает ссылку на объект `CIPropertyTree`.
- **AddQuery** — добавляет узел запроса вместе с полем данных запроса. Принимает объект `Query`.
- **Delete** — удаляет текущий узел (или узел по названию).
- **Empty** — полностью очищает дерево.
- **Put** — добавляет элемент в узел (строка + значение).
- **PutArray** — добавляет массив в узел.
- **PutJSON** — добавляет строку в формате JSON.
- **PutMap** — добавляет хэш-таблицу (объект `CIMap`) в узел.
- **SetConst** — делает дерево константным (неизменяемым).
- **PushBack** — добавляет пустой узел внутрь массива.

### Методы получения данных

- **Get** — возвращает значение текущего или дочернего узла.
- **GetArray** — возвращает значение узла как массив.
- **GetJSON** — возвращает строковое представление дерева в формате JSON.
- **GetKey** — возвращает название текущего узла при итерировании.
- **GetMap** — возвращает хэш-таблицу из узла.
- **GetParams** — возвращает значения параметров из дерева.
- **GetQuery** — заполняет объект запроса данными из узла дерева.
- **SetFirstLine** / **SetNextLine** — получение первого и последующих узлов при итерировании.

### Методы проверки

- **IsConst** — возвращает `True`, если дерево константное.
- **IsEmpty** — возвращает `True`, если в дереве нет ни одного узла.
- **IsValidLine** — проверка корректности текущего узла при обходе.

## Создание ботов в EME БД

Ботом в EME БД является процедура, которая запускает складской процесс либо по расписанию, либо по какому-то событию. в системе EME.WMS боты автоматизируют складские процессы.

### Пошаговая инструкция создания бота

1. Определить вид события для запуска бота.
2. Реализовать базовый EME-L класс бота и прописать таймерную процедуру.
3. Реализовать наследуемый EME-L класс с конструктором и методами бота.
4. Создать событие планировщика.
5. Сформировать бота через «Классификатор боты процессов».
6. Создать событие процесса.
7. Создать бот-процесс через «Описание бизнес-процессов».

### Виды событий

События в ЕМЕ БД делятся на два типа:

- **Таймерные** — генерируются на мозговом компьютере и выполняются по временному расписанию (1 час, 5 часов, 1 минута).
- **Функциональные** — генерируются в определённый момент времени по логике задачи.

### Реализация EME-L класса для бота

**Базовый класс (таймерная процедура):**

```EME-L
TD01()
{
    Object("dsDB", "QualifierEvents").OnEvent("TD01", NULL_RECORD, NULL_REF);
}
```

**Наследуемый класс (конструктор):**

```EME-L
objDeliveryOrdersImport = Object("IClass", "DeliveryOrdersImport", "DeliveryOrdersImport");
```

**Метод запуска бота:**

```EME-L
RunBot(BotRec AS "ProcessBotClassifier", ContextParams)
{
    Params = Object("Parameters");
    Params.SetString(ContextParams);

    r_Orders    = Object("dsDB", "Orders");
    ParamRecord = Params.GetLongParam("Record", NULL_RECORD);
    ParamRef    = Params.GetLongParam("Line", NULL_REF);
    If (r_Orders.GetRecord() == ParamRecord)
        r_Orders.SetLine(ParamRef);

        is_transaction(1, tr("Доктор импорта для заказов на доставку"));

        res = objDeliveryOrdersImport.CreateDeliveryOrdersByOrders(r_Orders);

        If (BotRec.IsNeedAddHistory())
            Switch(res)
            Case 0:
                message = is_format(tr("Выгружен заказ: %s."), r_Orders.GetItemText());
                BotRec.AddHistoryLine(BotRec.GetStatusNoError(), message, ContextParams, NULL_RECORD, NULL_REF);
                Break
            Case 1:
                message = is_format(tr("Для  заказа %s уже создан \\"Заказ на доставку\\"."), r_Orders.GetItemText());
                BotRec.AddHistoryLine(BotRec.GetStatusError(), message, ContextParams, NULL_RECORD, NULL_REF);
                Break
            End Switch
        End If

        is_transaction(-1);
    End If
}
```

### Создание события планировщика

1. Открыть «Утилиты БД -> Навигатор канальной схемы -> Редактор событий».
2. Создать событие с параметрами:
   - Имя события
   - Тип события: Будильник
   - DLL: MetC_SyS.dll
   - Функция: rf_run_EMEL
   - Состояние: Активно
3. В параметрах функции указать базовый класс бота и метод с таймерной процедурой.
4. В параметрах типа события указать период выполнения.

### Формирование бота через классификатор

1. Открыть диалог «Классификатор боты процессов».
2. Создать бота с названием и наследуемым EME-L классом.
3. При необходимости добавить параметры бота (название, способ ввода, запись ввода, поле ввода).
4. На вкладке «Обработчики» указать название метода.
5. Установить галочку «Вести историю робота».

### Создание события процесса

1. Открыть диалог «События процессов».
2. Создать событие:
   - Название события
   - Обработчик: выполнить бот
   - Событие: указать планировщик
   - Бот: указать бота из классификатора

### Создание бот-процесса

1. Открыть диалог «Описание бизнес процессов».
2. В папке «Боты» нажать «Создать бот-процесс».
3. Заполнить параметры — нажать «Введите событие процесса».
4. В «Событиях процессов» выставить для события сформированный бот-процесс.
5. Активировать бота — нажать «Статус: Прототип» для смены на «Статус: Утвержден».
