# Репликация ЕМЕ БД на SQL-сервер

> **Поисковые синонимы:** репликация БД, EME БД реплика, SQL-сервер, PostgreSQL, ODBC, статическая выгрузка, динамическая выгрузка, ядерный экспорт, ERPReplication, d_replica, db_box/sys/export, строка подключения, CheckDupKeys, ResolveDupKeys, DoImport, ReplicationLoop, DeleteOldExportFiles, первичная выгрузка, реплика БД, SQL реплика, форматирование записей, цепочка файлов импорта

---

## Назначение модуля репликации

Модуль репликации БД EME обеспечивает наличие полной копии БД EME на SQL-сервере. Репликация EME БД развёртывается через интерфейс рабочей станции через диалог «Репликация БД ЕМЕ на SQL-сервере» (меню «Утилиты БД → СИСТЕМА → Репликация БД»).

### Две стадии репликации в системе EME.WMS

| Стадия | Описание |
|---|---|
| **Статическая** | Первичная выгрузка данных из WMS. Для обновления в реальном времени нужна динамическая выгрузка |
| **Динамическая** | Обновление данных в SQL базе на основе хронологической цепочки файлов ядерного экспорта |

### Пошаговая инструкция развертывания реплики

1. Указать строку подключения к SQL базе данных в настройках модулей
2. Настроить динамическую выгрузку (задать директорию ядерного импорта в настройках модулей)
3. Выполнить первоначальную выгрузку данных из EME БД в SQL базу (диалог «Репликация БД»)
4. Перезапустить рабочую станцию с ключом `/d_replica`
5. Импортировать вручную первый файл ядерного экспорта (кнопка «Импорт файла»)
6. Добавить расширение `.bak` к имени импортированного файла
7. Произвести дополнительные настройки в классе `ERPReplication` при необходимости

## Настройка подключения к SQL базе данных

Строка подключения задаётся в настройках модулей (`Ctrl-Shift-M`): «Предпочтения → Репликация → Строка подключения к БД».

Пример строки подключения к PostgreSQL:

```
ODBC:Driver={PostgreSQL ANSI(x64)};Server=localhost;DataBase=WMS_REPLICA;UID=postgres;PWD=123;
```

Параметр `Driver` зависит от используемой СУБД. После ввода строки проверьте соединение кнопкой «Проверить подключение» в диалоге репликации.

---

## Первоначальная (статическая) выгрузка данных

Нажмите «Выгрузить данные» в диалоге репликации. Откроется браузер по записям — выберите весь перечень записей БД для выгрузки и нажмите зелёную галочку.

### Формат данных в SQL-реплике

- **Имя таблицы SQL:** `{Номер записи БД EME}.{Имя записи БД EME}`
- **Имя колонки таблицы:** `{Номер поля БД EME}.{Имя поля БД EME}`

---

## Настройка динамической выгрузки данных

1. В настройках модулей (`Ctrl-Shift-M`) → «Предпочтения → Репликация → Директория импорта» укажите полный путь к `/db_box/sys/export` (включая букву диска)
2. Сохраните настройку и перезапустите ЕМЕ БД с ключом `/d_replica`
3. После перезапуска в `/db_box/sys/export` автоматически создастся первый файл ядерного экспорта
4. В диалоге репликации нажмите «Импорт файла» и выберите этот файл
5. После импорта добавьте расширение `.bak` к имени файла

## Дополнительные настройки в классе ERPReplication

Откройте редактор классов (`Ctrl-Shift-C`) и найдите класс **ERPReplication**.

### 1. Цепочка файлов импорта для системного филиала

В методе `DoImport()` измените строку:

```
FileMask = "*.000";    →    FileMask = "*.127"
```

Это формирует цепочку файлов импорта при использовании системного филиала.

### 2. Период обновления данных в реплике

В методе `ReplicationLoop()` измените строку:

```
Period = 60 60 1000    →    Period = 5 60 1000
```

Это уменьшает цикл обновления с 1 часа до 5 минут. Для быстрого real-time обновления:

```
Period = 10000
```

Данные будут обновляться каждые 2 секунды.

### 3. Удаление старых файлов импорта

В методе `DeleteOldExportFiles()` измените значение переменной `Depth`. Изначально — 48 часов. При `Period = 10000` (генерация файлов каждые 2 секунды) рекомендуется установить 1–2 часа.

## Решение проблем при выгрузке данных

### Ошибка дублирующихся ключей при статической выгрузке

В классе `ERPReplication()` найдите методы `CheckDupKeys()` и `ResolveDupKeys()`:

1. В переменную `RecordNo` введите ID проблемной записи
2. Сохраните изменения
3. Наведите курсор в тело каждого метода и запустите их по очереди

### Ошибка кодировки (UTF-8 / Windows-1251)

Не включайте в реплику запись, вызывающую ошибку кодировки (не ставьте галочку при выборе записей для статической выгрузки в диалоге репликации ЕМЕ БД).
