## Обзор is-функций в языке EME-L

is-функции в EME-L являются внешними встроенными функциями, которые не могут быть реализованы на EME-L. Вызов is-функции имеет следующий формат:

```EME-L
{is-функция}([{Аргумент1}, [{Аргумент2} ...]])
```

is-функции языка EME-L подразделяются на функциональные группы: Bit, Browser, Context и другие.

---

## is-функции группы Bit — битовые операции

is-функции группы Bit отвечают за работу с битовыми признаками в языке EME-L.

### is_bit — проверка установленного бита

в языке EME-L функция `is_bit` выполняет проверку на установленный бит. Принимает на вход поле битов (число типа `long`) и номер бита. Возвращает `TRUE`, если указанный бит установлен, иначе — `FALSE`.

```EME-L
If (is_bit(Flags, 3))
    'Бит 3 установлен'
End If
```

### is_clear_bit и is_set_bit — управление битами

в системе EME.WMS функции `is_clear_bit` и `is_set_bit` выполняют снятие бита из поля и добавление бита в поле соответственно. Входные параметры — поле битов и номер бита. Возвращают обновленное поле битов.

```EME-L
Flags = is_set_bit(Flags, 5);    'Установить бит 5'
Flags = is_clear_bit(Flags, 5);  'Снять бит 5'
```

### is_reverse_bit — инверсия бита

в языке EME-L функция `is_reverse_bit` инвертирует номер бита в поле. На вход подается поле битов и номер бита. Возвращает поле бита с инвертированным номером.

```EME-L
Flags = is_reverse_bit(Flags, 3);
```

### is_and, is_or, is_xor — побитовые операции

в EME БД функции `is_and`, `is_or`, `is_xor` выполняют побитовые операции "И", "ИЛИ", "Исключающее ИЛИ" соответственно. Принимают любое количество параметров. Возвращают результат операции.

```EME-L
Result = is_and(Flags1, Flags2);
Result = is_or(Flags1, Flags2, Flags3);
Result = is_xor(Flags1, Flags2);
```

### is_not — унарное побитовое НЕ

в языке EME-L функция `is_not` выполняет унарную операцию "НЕ". Принимает на вход число. Возвращает результат операции.

```EME-L
Result = is_not(Flags);
```

### is_mask — формирование побитовой маски

в системе EME.WMS функция `is_mask` формирует побитовую маску. Принимает на вход число или последовательность чисел через запятую.

```EME-L
Mask = is_mask(1, 3, 5);
```

### is_shift_left и is_shift_right — сдвиг битов

в языке EME-L функции `is_shift_left` и `is_shift_right` выполняют сдвиг поля битов влево и вправо соответственно. Входные параметры — поле битов и значение сдвига. Возвращают обновленное поле битов.

```EME-L
Shifted = is_shift_left(Flags, 2);
Shifted = is_shift_right(Flags, 2);
```

## is-функции группы Browser — работа с браузерами

is-функции группы Browser отвечают за работу с браузерами в EME БД.

### is_hewed и is_hew_item — проверка выделения строки

в языке EME-L функции `is_hewed` и `is_hew_item` отвечают за проверку выделения строки в браузере. Принимают на вход переменную выделения, номер строки и объект. Возвращают `True`, если строка выделена, иначе — `False`.

### is_empty_hew — проверка отсутствия выделения

в системе EME.WMS функция `is_empty_hew` отвечает за обработку случая отсутствия выделения строки браузера. Принимает на вход переменную выделения и объект. Возвращает `True`, если в переменной нет ни одной строки, иначе — `False`.

### is_browse — номер выбранной строки браузера

в языке EME-L функция `is_browse` принимает на вход браузер. Возвращает номер строки, выбранной в браузере. Можно передать несколько браузеров — вернется номер строки для последнего.

```EME-L
Line = is_browse(Browser1, Browser2);
```

### is_browse_data — содержимое колонки браузера

в EME БД функция `is_browse_data` принимает на вход название колонки и строку браузера (необязательный параметр). Возвращает содержимое колонки.

```EME-L
Value = is_browse_data("ColumnName", Line);
```

### is_browser_base_record — базовая запись браузера

в языке EME-L функция `is_browser_base_record` принимает на вход номер строки записи "Шаблоны BROWSE", а возвращает номер базовой записи браузера.

```EME-L
BaseRecord = is_browser_base_record(TemplateLine);
```

### is_run_browse — запуск браузера записи БД

в системе EME.WMS функция `is_run_browse` запускает браузер записи БД. Принимает на вход номер записи БД, возвращает номер выбранной строки браузера.

```EME-L
SelectedLine = is_run_browse(RecordNumber);
```

### is_keydown_flag — считывание нажатия клавиши

в языке EME-L функция `is_keydown_flag` считывает нажатие клавиши в интерфейсе браузера. На вход ничего не принимает, возвращает целочисленное значение `keydown_flag`.

```EME-L
KeyFlag = is_keydown_flag();
```

## is-функции группы Context — работа с контекстом

is-функции группы Context отвечают за работу с текущим контекстом в EME БД (открытые ресурсы: диалоги, отчёты, классы и т.д.).

### is_put_context — формирование контекста

в языке EME-L функция `is_put_context` отвечает за формирование текущего контекста программы. На вход принимает имя контекста и параметр для его запоминания.

```EME-L
is_put_context("MyContext", Value);
```

### is_get_context_as_string — получение контекста как строки

в системе EME.WMS функция `is_get_context_as_string` позволяет получить параметр контекста в формате строки. На вход принимает имя контекста.

```EME-L
Value = is_get_context_as_string("MyContext");
```

### is_get_context_as_ref — получение контекста как числа

в языке EME-L функция `is_get_context_as_ref` позволяет получить параметр контекста в формате целого числа. На вход принимает имя контекста.

```EME-L
Ref = is_get_context_as_ref("MyContext");
```

### is_del_context — удаление контекста

в EME БД функция `is_del_context` удаляет ячейку контекста по его имени. На вход принимает имя контекста, ничего не возвращает.

```EME-L
is_del_context("MyContext");
```

### is_check_context — проверка контекста

в языке EME-L функция `is_check_context` проверяет текущий контекст программы. Поведение зависит от количества параметров:

- **1 параметр** — проверка, является ли контекст диалогом (`"Dialog"`) или запросом (`"Query"`).
- **2 параметра** — дополнительно проверяется имя диалога/запроса.

Возвращает `TRUE` или `FALSE`.

```EME-L
If (is_check_context("Dialog"))
    'Текущий контекст — диалог'
End If

If (is_check_context("Dialog", "Clients.Dialog"))
    'Текущий контекст — диалог Clients.Dialog'
End If
```

### is_get_context — получение текущего контекста

в системе EME.WMS функция `is_get_context` получает текущий контекст программы. Допустимые параметры: `"Dialog"`, `"Query"` или `"Report"`. Возвращает имя ресурса, считанного из текущего контекста. Может вернуть пустую строку, если контекст не является диалогом, запросом или отчетом.

```EME-L
DialogName = is_get_context("Dialog");
QueryName = is_get_context("Query");
ReportName = is_get_context("Report");
```
