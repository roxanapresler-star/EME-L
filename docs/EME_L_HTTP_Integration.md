## HTTP-сервер и тонкий клиент в EME.WMS

В 2001 году в EME был реализован полноформатный HTTP-сервер, полностью проинтегрированный в базу данных. Это позволяет реализовывать проекты с «тонким клиентом». В режиме тонкого клиента организуется работа терминалов сбора данных (ТСД) с Wi-Fi-точкой доступа. в системе EME.WMS HTTP-сервер является основным коммуникационным узлом между ТСД, Монитором Заданий и сервером БД.

### Архитектура обмена данными

Сервер http (http-сервер) — это сервер для клиентских устройств (Монитор Заданий и ТСД), добавляющий цепочку физических событий бизнес-процесса в очередь транзакций. в языке EME-L HTTP-запросы обрабатываются специализированными классами-обработчиками.

ТСД работает со списком приказов, загруженных в память с сервера. Фоновый поток на ТСД периодически отправляет запросы на сервер, чтобы обновить список приказов. Обновление происходит либо по таймеру каждые 10 секунд, либо после действий пользователя.

HTTP-запросы поступают в ядро, и оно решает, какой обработчик выполняет тот или иной запрос. Будет ли выполнен код на C++, или вызван обработчик на EME-L. Все запросы от ТСД идут на виртуальный каталог `terminals.html`, используется класс `Terminals`.

### Пошаговое описание обновления приказов на ТСД

1. В методе `CThreadData::UpdateData()` осуществляется отправка запроса с ТСД на сервер (формируется очередь запросов и параллельная отправка на транзакционный и нетранзакционный порты).
2. Перед формированием списка проверяется регистрация ТСД (`Terminals::CheckTerminal()`) и авторизация кладовщика (`CommonTerminal::CheckOperator()`).
3. Вызывается `DrawListOrders::DrawListOrdersEx()`, формирующий список значений общих настроек системы.
4. Для каждого приказа формируется блок с параметрами (тип, номер документа и т.д.). Начало блока — `Order=Start`, окончание — `Order=End`.
5. После обработки сервером вызывается `CThreadData::OnRequestDone()`. Данные передаются в объекте `CHttpData`.
6. Данные сохраняются в глобальный объект `EMEGlobalData` методом `SaveServerData()`. При изменении данных посылается сигнал `ServerDataChanged()`.
7. `CThreadData::OnRequestDone()` анализирует, нужно ли отправлять следующий запрос незамедлительно или подождать 10 секунд.

## Оптимизация обмена данными между ТСД и сервером

Каждые 10 секунд ТСД отправляет на сервер запрос на получение списка приказов, используя команду `GetListOrders`. В ответ сервер присылает `TakeListOrders` со списком приказов.

Сервер не каждый раз присылает полную информацию — может прислать «урезанные» данные, содержащие только номера строк и изменяемую информацию. Полные данные запрашиваются при обнаружении расхождений.

Для добавления данных, которые могут меняться на ТСД одновременно с сервером:

1. В EME-L классе `DrawListOrders` поле добавляется в запрос методом `AddOutputKeys`.
2. В методе `IsSameOrders()` класса `EMEGlobalData` делается сравнение для требуемого поля.

### Механизм оптимизации

1. ТСД определяет, нужны ли полные данные (`EMEGlobalData::IsGetOnlyOrdersKeys()`).
2. Если полные данные не нужны — передаётся признак `GetListOrdersKeys`.
3. Сервер формирует ответ с учётом признака.
4. ТСД анализирует ответ и сравнивает с локальным списком (`EMEGlobalData::IsSameOrders()`).
5. При расхождениях — незамедлительно отправляется запрос на полные данные.

## Пакетная обработка HTTP-запросов от ТСД

На больших складах ежесуточно выполняется более 150 тысяч серверных транзакций, инициируемых с ТСД. в системе EME.WMS для уменьшения нагрузки несколько запросов с разных ТСД объединяются в один запрос с одной общей транзакцией. Обработка запросов перенесена из потоков HTTP-сервера в поток сервера заданий.

### Формирование HTTP-запросов на стороне ТСД

- **CHttpData::GetRequestString()** — формирует строку запроса. Все запросы делятся на транзакционные и нетранзакционные.
- **CEMEHttp::HttpRequest** — формирует номер порта, URL и отправляет запрос. Сервер не выполняет запрос сразу — возвращает ID запроса. ТСД запоминает его в `ServerJobNo` и циклично отправляет запрос статуса задания.
- **CThreadData::OnRequestDone** — формирует интервал между отправками запросов статусов.

### Обработка HTTP-запросов на стороне сервера в EME-L

- **Document_OnBatchRespond(Doc As "HTMLDocument")** — сервер заданий получает наименование класса по пришедшей команде.
- **Document_OnBatchRespond(arrHtmlDocs As "Array", arrEMELObjects As "Array", ErrArray As "Array")** — обработка пачки запросов от сервера заданий.
- **ParseCommand()** — определяет EME-L класс-обработчик по наименованию команды.
- **CheckTerminalBatch()** — пакетный поиск терминалов и проверка версии.

## Отправка веб-запросов из EME-L

В языке EME-L для отправки HTTP-запросов во внешние системы используется декларация метода:

```EME-L
SendWebRequest(address, Params)
```

Этот механизм позволяет EME-L коду обращаться к внешним HTTP-сервисам и API.

## Формирование коротких URL

В ЕМЕ БД может возникнуть необходимость создания укороченных ссылок, например для QR-кодов или скрытия параметров запроса. в системе EME.WMS для этого используется «Классификатор WEB объектов» (меню «Утилиты БД -> СИСТЕМА -> Классификатор WEB объектов»).

### Пример создания короткого URL

Исходный URL: `http://192.168.1:8383/terminals.html?Command=GetListPrinters&Version=1.763&OnlySystemPrinters=1`

1. Создать корневой каталог «Shortcuts».
2. Создать подкаталог «PrintersList».
3. Поставить галочку «Ярлык».
4. Заполнить «Реальный путь для ярлыка» названием модуля (`Terminals.html`).
5. Заполнить «Дополнительные команды» параметрами после названия модуля.

Результат: `http://192.168.1:8383/Shortcuts/PrintersList`

**Важно:** короткие ссылки нельзя создавать для компонентов, ведущих к стороннему HTTP-серверу (например, 3PL).

## Оптимизация: печать стека потоков на HTTP-сервере

Довольно простым и эффективным способом обнаружения неоптимального кода HTTP-запроса на сервере является печать стека потоков. Это позволяет выявить медленные обработчики и оптимизировать производительность EME.WMS.

## Настройка HTTP-сервера

Параметры HTTP-сервера задаются в настройках рабочей станции (Ctrl-Shift-W) или в файле `eme.config`:

```
"dataslice_config" = /mono "C:\db_63\dat0ws\eme.hdr" /bc /http /d_dataslice
```

Ключ `/http` включает HTTP-сервер. в системе EME.WMS также указываются IP-адрес и порт HTTP-сервера.
