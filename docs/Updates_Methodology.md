# Методика выполнения обновлений EME.WMS на работающем складе

> **Синонимы для поиска:** обновление EME WMS, обновление склада, обновление ядра, обновление метапроекта, обновление ТСД, EME-L обновление, бэкап WMS, тест структуры БД, тест целостности, тест интерфейсов, перекомпиляция классов, мастер раздел, рабочий раздел, заливка EME-L, пакет обновления

---

## Общие требования

К моменту обновления промышленно функционирующей EME.WMS у руководителя проекта на руках должен быть **пакет обновления**, протестированный в интерактивном режиме. Именно этот пакет устанавливается без каких-либо изменений.

в системе EME.WMS обновлениям могут подвергаться следующие компоненты:

- **Ядро**
- **Метапроект**
- **ТСД**
- **EME-L** (структура базы данных, классы, диалоги, браузеры, отчёты, запросы EME-SQL)
- **Монитор Заданий**

Все службы рабочих разделов должны запускаться каждый со своего ядра и метапроекта — это единственно правильный вариант для автоматического развёртывания.

---

## Пошаговая инструкция обновления

### Шаг 1: Согласование времени

в системе EME.WMS необходимо согласовать время остановки серверов и проговорить время простоя складов. При обновлении ядра закладывать **минимум 1 час**. Ответственные: менеджер проекта и руководитель проекта. Написать подтверждающее письмо с содержанием обновления. Получатели: заказчик, руководитель проекта, техническая поддержка, менеджер проекта, бригадир.

### Шаг 2: Перенос пакета обновления

Перенести пакет обновления на сервер Заказчика любым доступным способом.

### Шаг 3: Остановка рабочих разделов

Остановить **все службы рабочих разделов**. Это необходимо, чтобы после обновления мастер сервера остальные рабочие разделы не подхватывали фрагменты обновления.

### Шаг 4: Остановка мастер раздела

Остановить службу **мастер раздела**.

### Шаг 5: Создание бэкапов

Создать бэкапы серверных данных мастер раздела, ядра и метапроекта.

### Шаг 6: Сборка

Собрать ядро и метапроект.

### Шаг 7: Запуск мастер раздела

Запустить данные мастер раздела в режиме службы.

### Шаг 8: Подключение рабочей станции

Открыть рабочую станцию, подключённую к мастер разделу.

### Шаг 9: Внесение изменений EME-L

в языке EME-L изменения вносятся через **модуль заливок** — все изменения по структуре БД, классам, диалогам, браузерам, отчётам и запросам EME-SQL.

### Шаг 10: Тестирование внутри БД

Запустить следующие тесты:

| Тест | Путь |
|------|------|
| Тест структуры базы данных | «Система → Ресурсы → Структура базы данных → Тест структуры раздела» |
| Тест целостности БД | «Система → Консоль сервера → Система → Тест целостности → Старт» |
| Тест интерфейсов | «Система → Ресурсы → Тестирование → Тест интерфейса» |
| Локальная перекомпиляция всех классов | «Система → Ресурсы → Редактор классов → Сервис → Все классы → Локальная → Применить» |

### Шаг 11: Проверка результатов

в EME БД проверить результаты выполнения тестов, включая записи в лог-файлах. **Все тесты должны пройти безошибочно.**

### Шаг 12: Запуск рабочих разделов

Запустить службы рабочих разделов.

### Шаг 13: Проверка логов

Выполнить проверку логов на предмет наличия ошибок.

### Шаг 14: Обновление ТСД и Монитора Заданий

Создать бэкап предыдущей версии ТСД и Монитора Заданий, установить новую версию на каждом разделе.

---

## Поведение системы после обновления

в системе EME.WMS после запуска каждой службы сервера рабочего раздела программа автоматически обнаружит обновление, загрузит файлы, выполнит обновление и запустится. Далее программа сделает выравнивание по мастер-транзакциям для получения изменений в прикладной части.
