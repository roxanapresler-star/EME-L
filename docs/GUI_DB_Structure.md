# Диалог «Структура БД» — настройки записей и полей в ЕМЕ БД

> **Поисковые синонимы:** структура базы данных, записи БД, поля записи, структура БД, флаги записи, редактор структуры, экспорт структуры, импорт структуры, атрибуты полей, ссылочные поля, цепочки, VLP-указатель, системная запись, мастер-запись, раздвоитель ключей, межфилиальный экспорт, EME DB structure, database structure dialog

## Общие сведения

Настройки записей и полей БД доступны в редакторе структуры БД, который открывается через пункты меню **«Система → Ресурсы → Структура Базы Данных»**. За настройки записей отвечают группы: «Флаги записи», «Утилиты» и «Разделы». Для перехода к настройкам полей необходимо нажать кнопку «Редактирование полей» или использовать горячую клавишу **Alt+R**. Настройки полей включают группы: «Описание поля записи», «Признаки поля», «Поля для заполнения».

в языке EME-L структура базы данных определяет базовые сущности, с которыми работают диалоги, запросы и классы. Правильная конфигурация записей и полей критически важна для корректной работы всей системы.

## Флаги записи в диалоге «Структура БД»

| Флаг | Назначение |
|------|-----------|
| **Системная запись** | Запись формирует фундамент БД, без которого она не запустится. Системные записи имеют символ «-» (минус) в номерном индексе. Создаются только программистами ЕМЕ БД. |
| **Не используется** | Запись никогда не используется при выполнении складских операций через WMS. |
| **Не копируется на сервер** | Данные записи не копируются на сервер WMS. |
| **Не проверять свой/чужой (не будет экспорт.)** | Применяется в механизме межфилиального экспорта/импорта данных. |
| **Не экспорт. в родителя** | Блокирует экспорт данных в родительский филиал. |
| **Не экспорт. в дочки** | Блокирует экспорт данных в дочерние филиалы. |
| **Не импорт. из родителя** | Блокирует импорт данных из родительского филиала. |
| **Не импорт. из дочек** | Блокирует импорт данных из дочерних филиалов. |
| **Разрешить удалять после экспорта** | Разрешает удаление данных после завершения экспорта в механизме межфилиального обмена. |
| **Трассировка** | Настройка не используется. |
| **Участвует в переменных ссылках** | Оптимизация работы записей со ссылочными полями. Ссылки с переменным вектором могут обращаться к строкам из разных записей БД. |
| **Разрешить реанимацию удалённых строк** | Актуален для записей с механизмом очереди — удалённые строки возвращаются на исходное место, а не в конец записи. |
| **NET VM** | Отключает выравнивание данных записи с сервера, сокращая объём на рабочей станции. Системные записи обязательно выравниваются. Установить — в любом режиме, снять — только в монопольном. |
| **Мастер** | Запись принадлежит мастер-разделу БД. Системные записи всегда являются мастер-данными, поэтому флаг для них обязателен. |
| **Локальная для раздела (не будет экспорт.)** | Настройка не используется. |

в системе EME.WMS флаги записей управляют поведением данных при репликации между филиалами и определяют приоритет записей при синхронизации с сервером.

## Утилиты в диалоге «Структура БД»

### Экспорт структуры

При нажатии открывается окно экспорта структуры БД в файл с расширением **.dbh**. Пользователь указывает путь к файлу, выбирает перечень записей, управляет включением зависимых записей и может выставить флаг «Запрашивать разрешение» для ручного отбора зависимых записей.

### Импорт структуры

При нажатии открывается окно импорта структуры БД из файла **.dbh**. Доступны настройки: заменять/не заменять/запрашивать подтверждение для существующих записей, создавать файл соответствий (.bat), переименовывать VLP-поле по имени VLF, перестраивать нумерацию записей (крайне нежелательно).

### Модификация строк системных записей

Запускает процесс поднятия флагов модификации для всех системных записей, что приводит к экспорту всех системных строк. Перед выполнением отображается предупреждение.

### Модификация строк данной записи

Поднимает флаги модификации для выбранной записи. Модифицированные строки попадают в файл экспорта .dbh. Для больших записей увеличивается время экспорта.

### Удаление всех строк записи

Полная очистка записи от всех внесённых данных.

### Просмотр всех строк записи

Открывает броузер для просмотра данных, внесённых в запись.

### Раздвоитель ключей

Настройка не используется.

## Разделы в диалоге «Структура БД»

| Действие | Описание |
|----------|----------|
| **Тест структуры раздела** | Открывает диалог для тестирования структуры раздела. Рекомендуется выполнять перед архивацией. |
| **Удаление данных раздела** | Очищает данные в записях, принадлежащих разделу. Требует подтверждения. |
| **Изменение флагов на текущем филиале** | Открывает диалог для управления галочками принадлежности записей к разделу и изменения признаков полей. |

в EME БД разделы позволяют группировать записи по функциональным областям и управлять ими централизованно, включая независимое тестирование и очистку.

## Описание настроек полей записи

### Описание поля записи

- **Изменение признаков на текущем филиале** — открывает диалог для управления принадлежностью поля к разделу.
- **Атрибут поля** — задаёт тип данных поля.
- **Размер поля** — задаёт размер поля.
- **Переменная длина** — формирует атрибут переменной длины. Актуально для полей с VLP-указателем и для ссылок с переменным вектором.

в языке EME-L типы данных полей определяют, какие операции допустимы со значениями при программной обработке в классах и запросах.

### Признаки поля

| Признак | Описание |
|---------|----------|
| **Поле не используется** | Поле никогда не используется в складских операциях WMS. |
| **NetVM** | Отключает выравнивание данных поля с сервера. Поля системных записей должны выравниваться. |
| **Защищено контрольной суммой** | Обеспечивается целостность данных поля контрольной суммой. |
| **Не шифровать** | Настройка не используется. |
| **Не копируется на сервер** | Данные поля не копируются на сервер WMS. |
| **Раздел** | Поле является полем раздела БД. |
| **Не экспорт. в родителя** | Блокирует экспорт поля в родительский филиал. |
| **Не экспорт. в дочки** | Блокирует экспорт поля в дочерние филиалы. |
| **Не импорт. в родителя** | Блокирует импорт поля из родительского филиала. |
| **Не импорт. из дочек** | Блокирует импорт поля из дочерних филиалов. |
| **Не проверять владельца (не будет эксп.)** | Применяется в механизме межфилиального экспорта/импорта. |
| **Локальное для раздела (не будет эксп.)** | Настройка не используется. |
| **Трассировка обращений** | Настройка не используется. |
| **Проверять на NULL_REF** | Только для ссылочных полей. Проверка на пустое значение при чтении по реляционному пути. |

### Поля для заполнения

- **Запись, на которую ссылается данное поле** — указывается название записи, на которую ссылается поле.
- **Номер поля-ссылки цепочки** — указывается поле с атрибутом «Ссылка на начало цепочки» из связанной записи. Если у текущего поля выставлен атрибут «Ссылка на начало цепочки», параметр переименовывается в «Поле-ссылка на заголовок» и требует указания поля с атрибутом «Ссылка на заголовок цепочки».
- **Номер поля-указателя VLP** — актуально для полей с переменной длиной. Указывается поле с атрибутом «VLP-указатель», хранящее длину. Для поля с типом «сортировка-цепочка» переименовывается в «Поле вторичной сортировки».
- **Следующее поле в цепи языков** — настройка не используется.
- **Поле основного счётчика** — настройка не используется.
- **Директория хранилища данных** — актуально для полей с типом «Хранилище данных». Указывается путь к папке загрузки файлов, создаваемой строго в папке dat сервера.

в системе EME.WMS ссылочные поля и цепочки обеспечивают навигацию по реляционной структуре БД, позволяя связывать данные из разных записей без дублирования.
