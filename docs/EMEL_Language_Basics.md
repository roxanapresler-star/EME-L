# Основы языка программирования EME-L в системе EME.WMS

**Описание:** Язык EME-L (EME Language) — проприетарный язык программирования для разработки в системе EME.WMS. Используется для создания бизнес-логики складских процессов, диалогов, отчётов и интеграций.
**Сленг:** емель, емельный код, EME-L скрипт, емельный класс, программирование на EME-L

## Синтаксис и структура EME-L кода

Язык EME-L в системе EME.WMS использует синтаксис в стиле Pascal с блочными конструкциями. Код хранится в EME-L классах, которые создаются через редактор классов (сочетание клавиш Ctrl-Shift-C).

**Синонимы для поиска:** синтаксис EME-L, структура кода EME-L, написание кода на EME-L, программирование на EME-L, правила синтаксиса, структура программы

### Комментарии в EME-L коде

В языке EME-L комментарии обозначаются одинарной кавычкой `'`. Комментарии могут быть однострочными и располагаться как на отдельной строке, так и после кода.

```eme-l
'Это пример комментария в EME-L'
'Комментарии используются для пояснения кода'

r_Document = Object("dsDB", "Document"); 'Комментарий после строки кода'
r_Document.SetFirstLine(); 'Установка на первую строку записи'
```

### Операторы в EME-L

В EME-L используются следующие ключевые операторы (все операторы начинаются с заглавной буквы):

- `If / End If` — условный оператор (если / конец если)
- `Else` — альтернативная ветка условия (иначе)
- `For / End For` — цикл с параметром
- `Loop / End Loop` — цикл по записям
- `While / End While` — цикл с предусловием
- `Switch / Case / Break / End Switch` — оператор множественного выбора
- `Return` — возврат значения из метода
- `Continue` — переход к следующей итерации цикла

**Синонимы для поиска:** операторы EME-L, условные операторы, циклы в EME-L, if else EME-L, for loop EME-L, конструкции языка

### Пример условного оператора в EME-L

```eme-l
'Проверка валидности строки документа в EME-L'
If (r_Document.IsValidLine())
    'Строка валидна - получаем номер документа'
    docNo = r_Document.GetRegNo();
Else
    'Строка не валидна - выводим сообщение'
    is_message("Внимание", "Документ не найден", "OK", "STOP");
End If
```

### Пример цикла For в EME-L

```eme-l
'Перебор строк документа в EME-L'
r_DocLines = Object("dsDB", "DocLines");
r_DocLines.SetSkipMode();
r_DocLines.GetDocumentRefFld().MustBeRefEQ(docRef);

For (r_DocLines.SetFirstLine(); r_DocLines.IsValidLine(); r_DocLines.SetNextLine())
    'Обработка каждой строки документа'
    qty = r_DocLines.GetQtyN();
    goodsItem = r_DocLines.GetGoodsItemRefFld();
    
    'Подсчёт общего количества'
    totalQty = totalQty + qty;
End For
```

## Создание объектов в EME-L

В EME-L объекты создаются с помощью функции `Object()`. Это основной способ создания экземпляров классов и работы с данными в системе EME.WMS.

**Синонимы для поиска:** создание объекта EME-L, Object функция, инициализация объекта, экземпляр класса EME-L, создание экземпляра

### Синтаксис создания объекта

```eme-l
Object("ИмяКласса", "ИмяПеременной")
Object("ИмяКласса") 'Без имени переменной'
```

### Основные типы объектов в EME-L

- `Object("dsDB", "ИмяЗаписи")` — объект записи базы данных
- `Object("Dialog")` — объект диалога
- `Object("Control",, "ИмяОргана")` — орган управления в диалоге
- `Object("Array")` — массив
- `Object("Map", размер)` — хэш-таблица
- `Object("Query")` — запрос
- `Object("BitBuffer", размер)` — битовый буфер

### Пример создания объектов в EME-L

```eme-l
'Создание объекта записи "Документы" в EME-L'
r_Document = Object("dsDB", "Document");

'Создание объекта записи "Строки документа" в EME-L'
r_DocLines = Object("dsDB", "DocLines");

'Создание объекта диалога в EME-L'
Dialog = Object("Dialog");

'Создание массива в EME-L'
arrOrders = Object("Array");

'Создание хэш-таблицы с размером 1000 элементов в EME-L'
mapGoods = Object("Map", 1000);
```

## Работа с транзакциями в EME-L

Транзакции в EME-L обеспечивают целостность данных при выполнении операций. Все изменения в базе данных должны выполняться внутри транзакций.

**Синонимы для поиска:** транзакция EME-L, is_transaction, работа с транзакциями, целостность данных, открытие транзакции, закрытие транзакции

### Синтаксис работы с транзакциями

```eme-l
'Начало транзакции в EME-L'
is_transaction(1, "Описание транзакции");

'Код внутри транзакции'
'... операции с данными ...'

'Завершение транзакции в EME-L'
is_transaction(-1);
```

### Пример транзакции в EME-L

```eme-l
'Создание нового документа прихода в EME-L'
CreateIncomeDocument()
{
    'Открываем транзакцию с описанием'
    is_transaction(1, "Создание документа прихода");
    
    'Создаём объект записи Документы'
    r_Document = Object("dsDB", "Document");
    r_Document.CreateNewLine();
    
    'Заполняем поля документа'
    r_Document.PutDocumentTypeRefFld(0); 'Тип документа = Приход'
    r_Document.PutCreateDateFld(is_date());
    r_Document.PutRegNoFld(GenerateRegNo());
    
    'Сохраняем документ'
    r_Document.ApplyNewLine();
    
    'Закрываем транзакцию'
    is_transaction(-1);
    
    'Возвращаем ссылку на созданный документ'
    Return r_Document.GetLine();
}
```

## Работа с записями базы данных в EME-L

Записи базы данных (Records) — это таблицы данных в EME.WMS. Работа с записями осуществляется через классы, производные от CEMERec.

**Синонимы для поиска:** работа с записями, CEMERec, доступ к данным, чтение записи, запись данных, работа с БД EME-L

### Режимы обхода записей

В EME-L существует несколько режимов обхода записей:

- `SetRealMode()` — обход по физике (все строки подряд)
- `SetSkipMode()` — обход с наложением условий
- `SetSkipMode("Поле индекса")` — обход по индексу
- `SetChainMode()` — обход по цепочке
- `SetFindMode()` — поиск по хэш-таблице

### Пример работы с записью в EME-L

```eme-l
'Поиск документа по номеру в EME-L'
FindDocumentByRegNo(RegNo)
{
    r_Document = Object("dsDB", "Document");
    
    'Устанавливаем режим SkipMode для поиска'
    r_Document.SetSkipMode();
    
    'Добавляем условие поиска'
    r_Document.GetRegNoFld().MustBeEQ(RegNo);
    
    'Устанавливаем на первую найденную строку'
    r_Document.SetFirstLine();
    
    'Проверяем, найден ли документ'
    If (r_Document.IsValidLine())
        Return r_Document.GetLine();
    Else
        Return NULL_REF;
    End If
}
```

### Пример обхода по цепочке в EME-L

```eme-l
'Обход строк документа по цепочке в EME-L'
ProcessDocumentLines(docRef)
{
    r_Document = Object("dsDB", "Document");

    }
```

## Чтение по реляционному пути в EME-L

В EME-L поддерживается чтение данных по реляционному пути — это получение связанных данных через ссылочные поля.

**Синонимы для поиска:** реляционный путь, чтение по ссылке, связанные данные, Get методы, навигация по связям

### Пример чтения по реляционному пути

```eme-l
'Чтение данных по реляционному пути в EME-L'
r_Document = Object("dsDB", "Document");
r_Document.SetLine(90);

'Получение ворот через реляционный путь'
code = r_Document.GetGates().GetCode();

'Получение склада через реляционный путь'
warehouseName = r_Document.GetWarehouse().GetName();
```

### Важные правила реляционного пути

1. При первом вызове метода (например, `GetGates()`) создаётся объект и сохраняется в массиве
2. При последующих вызовах возвращается тот же объект (не создаётся новый)
3. Объект перенастраивается на нужную строку при каждом вызове

```eme-l
'Правильное использование реляционного пути в цикле EME-L'
Loop(r_Document)
{
    'GetGates() возвращает один и тот же объект, но перенастроенный'
    gatesCode = r_Document.GetGates().GetCode();
}

'НЕПРАВИЛЬНОЕ использование:'
r_Document.SetFirstLine();
r_Gates1 = r_Document.GetGates();
r_Document.SetNextLine();
r_Gates2 = r_Document.GetGates();
'Ошибка: r_Gates1 и r_Gates2 - это один и тот же объект!'
```

## Обработчики событий диалогов в EME-L

Диалоги в EME.WMS имеют специальные методы-обработчики, которые вызываются при определённых событиях.

**Синонимы для поиска:** обработчики событий, Dialog_OnLoad, Dialog_OnCheck, Dialog_OnClose, события диалога, ончек, онлоад

### Основные обработчики диалогов

| Обработчик | Описание | Сленг |
|-----------|----------|-------|
| `Dialog_OnLoad()` | Вызывается при загрузке диалога | онлоад |
| `Dialog_OnCheck(Transaction)` | Вызывается перед сохранением | ончек |
| `Dialog_OnClose(CloseType)` | Вызывается при закрытии диалога | онклоуз |
| `Dialog_OnUpdate()` | Вызывается после сохранения | онапдейт |
| `Dialog_OnAfterUpdate()` | Вызывается после обновления | онэфтерапдейт |

### Пример обработчика Dialog_OnCheck

```eme-l
'Валидация данных перед сохранением в EME-L (ончек)'
Dialog_OnCheck(Transaction)
{
    'Если транзакция уже открыта, пропускаем валидацию'
    If (~Transaction)
        'Проверка заполнения обязательных полей'
        If (Name.IsEmpty())
            is_message("Внимание!", "Не заполнено поле \"Наименование\"", 1);
            Name.SetFocus();
            Return 1; 'Возвращаем 1 для отмены сохранения'
        End If
        
        'Проверка количества'
        If (Qty <= 0)
            is_message("Ошибка", "Количество должно быть больше нуля", "OK", "STOP");
            Qty.SetFocus();
            Return 1;
        End If
    End If
    
    Return 0; 'Возвращаем 0 для разрешения сохранения'
}
```

### Пример обработчика Dialog_OnLoad

```eme-l
'Инициализация диалога при загрузке в EME-L (онлоад)'
Dialog_OnLoad()
{
    Dialog = Object("Dialog");
    
    'Установка значений по умолчанию'
    If (is_edit_data("Report.StartDate") == 0)
        StartDate.SetValue(is_date());
    End If
    
    'Загрузка драйвера сканера'
    Dialog.LoadScannerDriver(0);
    
    'Настройка диапазона дат'
    ctrlStartDate = Object("Control",,"StartDate");
    ctrlEndDate = Object("Control",,"EndDate");
    Dialog.SetControlDateRange(ctrlStartDate, ctrlEndDate);
}
```

## Оптимизация EME-L кода

При написании кода на EME-L важно учитывать производительность, особенно при работе с большими объёмами данных.

**Синонимы для поиска:** оптимизация EME-L, производительность, ускорение кода, Const оператор, эффективный код

### Правила оптимизации

1. Не использовать `Object()` внутри цикла
2. Использовать оператор `Const` для константных объектов
3. Предпочитать `SetChainMode()` вместо `SetSkipMode()`
4. Ограничивать данные на всех этапах работы

### Пример оптимизации с Const

```eme-l
'НЕОПТИМАЛЬНЫЙ код (объект создаётся в каждой строке запроса)'
{
    dbRegs = Object("dsDB", "Registers");
    dbRegs.SetLine(is_query(,"@"));
    Return dbRegs.GetQtyN();
}:FLOAT AS [Кол-во]

'ОПТИМАЛЬНЫЙ код (объект создаётся один раз)'
{
    dbRegs = Const(Object("dsDB", "Registers"));
    dbRegs.SetLine(is_query(,"@"));
    Return dbRegs.GetQtyN();
}:FLOAT AS [Кол-во]
```

### Пример правильной работы с Map

```eme-l
'НЕПРАВИЛЬНО: создание объектов в цикле'
For (r_Lots.SetFirstLine(); r_Lots.IsValidLine(); r_Lots.SetNextLine())
    array = mapLotRef.Lookup(line, Object("Array")); 'Утечка памяти!'
    array.Add(r_Lots.GetLine());
    mapLotRef.SetAt(line, array);
End For

'ПРАВИЛЬНО: без создания объектов в параметре Lookup'
For (r_Lots.SetFirstLine(); r_Lots.IsValidLine(); r_Lots.SetNextLine())
    array = mapLotRef.Lookup(line);
    If (~array.IsValid())
        array = Object("Array");
    End If
    array.Add(r_Lots.GetLine());
    mapLotRef.SetAt(line, array);
End For
```

## Передача аргументов по ссылке в EME-L

В EME-L поддерживается передача аргументов по ссылке, что позволяет возвращать несколько значений из метода.

**Синонимы для поиска:** передача по ссылке, ByRef, ByVal, RefOf, ValOf, возврат нескольких значений

### Синтаксис работы со ссылками

- `ByRef` — объявление параметра как ссылки
- `ByVal` — присваивание значения ссылке
- `RefOf` — передача переменной по ссылке
- `ValOf` — получение значения из ссылки

### Пример передачи по ссылке

```eme-l
'Метод с параметром-ссылкой в EME-L'
CalculateTotal(Quantity ByRef, Price ByRef)
{
    'Получение значений из ссылок'
    qty = ValOf Quantity;
    price = ValOf Price;
    
    'Изменение значений через ссылки'
    Quantity ByVal = qty * 2;
    Price ByVal = price * 1.1;
}

'Вызов метода с передачей по ссылке'
total = CalculateTotal(RefOf myQty, RefOf myPrice);
```

### Правила использования tr()

```eme-l
'ПРАВИЛЬНОЕ использование tr() в EME-L'
is_message(tr("Внимание"), tr("Документ не найден"), "OK", "STOP");
```

## Статическая линковка в EME-L

Статическая линковка позволяет компилятору проверять корректность имён методов во время компиляции, а не во время выполнения.

**Синонимы для поиска:** статическая линковка, динамическая линковка, As, тип объекта, наследование, Dynamic

### Динамическая линковка

При динамической линковке компилятор не знает о типах объектов и не проверяет наличие методов. Во время первого выполнения кода программа ищет метод по названию, получает его номер и сохраняет в скомпилированный код. При последующих вызовах метод вызывается по номеру.

### Динамический вызов (Dynamic и Invariant)

Для объектов `IClass` статическая линковка невозможна. Ранее использовались `FindMethod` и `CallMethod` (теперь запрещены). Вместо них:

```eme-l
MyDynamicObject As "Dynamic" = MyIClassObject;
```

Имена методов и количество аргументов не проверяются при компиляции. Поиск метода выполняется **перед каждым вызовом** — код работает медленнее.

```eme-l
MyDynamicObject As "Invariant" = MyIClassObject;
```

Имена так же не проверяются, но поиск метода выполняется **только при первом вызове** — предпочтительный вариант, работает быстрее `Dynamic`.

### Статическая линковка

При статической линковке компилятор знает типы объектов и проверяет наличие методов. Если тип объекта не определён или метод не найден, компилятор выдаёт ошибку.

```eme-l
'Объявление типа объекта в методе'
GetDocumentByRegNo(DocNoIncom)
{
    r_Doc As "Document" = Const(Object("dsDB", "Document"));
    '...'
}

'Указание типа объекта для параметра'
Integrator_OnSelect(Rec As "CEMERec")
{
    If (Rec.IsValidLine())
        bIsGroup = Rec.GetRecordName() != "Оборудование";
    End If
}
```

### Тип Dynamic

Для случаев, когда нужно вызвать метод у объектов разных типов, используется тип `"Dynamic"`. При этом компилятор не проверяет наличие метода, а во время выполнения метод ищется по названию.

```eme-l
'Использование типа Dynamic (нежелательно, замедляет работу)'
GetTotalSize(arrObjects As "Array")
{
    TotalSize = 0;
    Loop(arrObjects)
    {
        objClass As "Dynamic" = arrObjects.Get();
        TotalSize = TotalSize + objClass.GetSize();
    End Loop
    Return TotalSize;
}
```

**Важно:** Использование типа `"Dynamic"` нежелательно — программа работает медленнее, и компилятор не проверяет наличие методов.

### Разименоввание типа объекта

Если метод возвращает объект базового класса, программист может конкретизировать класс с помощью ключевого слова `As`:

```eme-l
'Разименовывание типа объекта'
r_User As "Users" = r_Doc.GetDBObject("@Оператор");
```

## Удалённый вызов процедур (Procedure)

Оператор `Procedure` позволяет вызывать EME-L код на удалённом сервере EME через HTTP.

**Синонимы для поиска:** удалённый вызов, Procedure, HTTP, интеграция, удалённый сервер

### Ограничения

- Внутри блока `Procedure / End Procedure` нельзя вызывать методы EME-L классов
- Код внутри блока компилируется динамической линковкой, отдельно от остального кода класса
- Можно использовать только Си-классы и is-функции, имеющиеся на обеих базах
- Аргументы, переданные без `RefOf`, не возвращаются обратно (экономия трафика)
- Аргументы с `RefOf` возвращаются в вызывающую базу

### Пример вызова процедуры на удалённом сервере

```eme-l
'Вызов процедуры на удалённом сервере EME'
Procedure <"10.1.6.153", 18080> (RefOf Query, UserRef)
    'Этот код выполняется на удалённом сервере'
    r_Menu = Object("dsDB", "UserMenu");
    r_Menu.SetChain("@Заголовок (@Пользователь)", UserRef);

    Loop (r_Menu)
        Query.AddLine();
        Query.Put("Название", r_Menu.GetName());
    End Loop
End Procedure
```

## HTTP-запросы из EME-L

Класс `System` позволяет выполнять HTTP-запросы к внешним серверам из EME-L.

**Синонимы для поиска:** HTTP, HttpRequest, веб-запрос, SMS, внешний API

### Пример отправки SMS через HTTP

```eme-l
'Отправка SMS через внешний API'
SendSMS(Text, Phone)
{
    'Замена 8 на 7 в начале номера'
    If (is_strncmp(Phone, "8") == 0)
        Phone = "7" + is_mid(Phone, 1);
    End If

    Result = Object("String");
    api_id = "your-api-id";

    Return Object("System").HttpRequest("sms.ru", 80, "/sms/send", 0,
        Result, "",,,"api_id=" + api_id, "text=" + Text, "to=" + Phone);
}
```

## Правила оформления программного кода в EME

При написании кода на EME-L следует соблюдать правила оформления.

**Синонимы для поиска:** стиль кода, оформление кода, naming conventions, правил кодирования

### Именование переменных

| Префикс | Тип данных | Пример |
|---------|-----------|--------|
| `r_` | Объект CEMERec (запись БД) | `r_Document`, `r_DocLines` |
| `d_` | Объект CEMEData | `d_Data` |
| `f_` | Объект CEMEFilter | `f_Filter` |
| `arr` | Объект CEMEArray | `arrOrders` |
| `map` | Объект Map | `mapGoods` |
| `b_` | Объект CEMEBuffer | `b_Buffer` |
| `v_` | Объект CDialogVar | `v_Variable` |
| `m_` | Переменная-член класса | `m_r_Document` |
| `obj` | Внешний объект | `objQuery` |
| `cs` | Строка (CString) | `csText` |
| `n` или `i` | Число (int) | `nCount`, `iIndex` |
| `b` | Логическое значение | `bIsValid` |

### Правила оформления

- Отступы — 4 пробела
- Имена методов и переменных в стиле CamelCase
- Имена классов записей БД начинаются с заглавной буквы
- Имена методов записей БД начинаются с глагола (Get, Put, Set, Is и т.д.)
- Комментарии начинаются с одинарной кавычки `'`
- Одна строка — один оператор

### Обязательные требования к комментариям

1. Перед каждым методом — дата создания и фамилия автора: `' #KPI Колударов 01.02.2024`
2. Перед методом — высокоуровневое описание решаемой задачи (что принимает, что возвращает)
3. Каждое исправление сопровождается комментарием **над** блоком (не сбоку, не снизу) с инициалами, датой и причиной: `' #NPM 30.11.2023 По заявке №12345: изменена логика объединения приказов`
4. Если код скопипастен — старые комментарии актуализируются, старые даты и фамилии удаляются
5. Каждый блок кода сопровождается комментарием — для чего он нужен в контексте задачи
6. Чтение из БД и запись в БД выделены в разные блоки (сначала чтение, потом запись, в разных циклах)
7. Код пишется с учётом больших данных, в комментариях указано, почему выбран тот или иной способ
8. Встроенная самодиагностика: проверка промежуточных результатов и значений из БД

Пример заголовка класса:

```eme-l
'#KPI 06.04.2012 Колударов'
'Модуль: Консолидация приказов'
'Заказчик: 21-й век'
'Заявка: объединение приказов на подборку по ячейке'
```

Пример комментария к исправлению:

```eme-l
'#NPM 30.11.2023'
'По заявке №12345: изменена логика объединения приказов'
'Старый код: Loop(r_Orders) ... End Loop'
'Причина: некорректная обработка пустых ссылок'
```

## Оптимизация памяти в EME-L

При написании кода на EME-L важно избегать утечек памяти и неэффективного использования ресурсов.

**Синонимы для поиска:** утечка памяти, оптимизация, Const, Object в цикле, производительность

### Создание объектов в цикле — утечка памяти

```eme-l
'НЕПРАВИЛЬНО: создание объектов в цикле'
For (r_Lots.SetFirstLine(); r_Lots.IsValidLine(); r_Lots.SetNextLine())
    array = mapLotRef.Lookup(line, Object("Array")); 'Утечка памяти!'
    array.Add(r_Lots.GetLine());
    mapLotRef.SetAt(line, array);
End For

'ПРАВИЛЬНО: без создания объектов в параметре Lookup'
For (r_Lots.SetFirstLine(); r_Lots.IsValidLine(); r_Lots.SetNextLine())
    array = mapLotRef.Lookup(line);
    If (~array.IsValid())
        array = Object("Array");
    End If
    array.Add(r_Lots.GetLine());
    mapLotRef.SetAt(line, array);
End For
```

### Const для кэширования объектов

В SQL-запросах объекты, создаваемые внутри фигурных скобок, создаются для каждой строки. Используйте `Const` для создания объекта один раз:

```eme-l
'НЕПРАВИЛЬНО: объект создаётся для каждой строки'
{
    dbRegs = Object("dsDB", "Registers");
    dbRegs.SetLine(is_query(,"@"));
    Return dbRegs.GetQtyN();
}:FLOAT

'ПРАВИЛЬНО: объект создаётся один раз'
{
    dbRegs = Const(Object("dsDB", "Registers"));
    dbRegs.SetLine(is_query(,"@"));
    Return dbRegs.GetQtyN();
}:FLOAT
```
