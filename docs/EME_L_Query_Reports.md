# Отчёты и печатные формы в EME.WMS

> Синонимы для поиска: отчёт EME БД, печатная форма EME-WMS, визуальный редактор отчётов, горизонтальный банд, вертикальный банд, потайной банд, разворачивающийся банд, динамический банд, rs_main_query rs_band_query rs_cell_query, DBField ячейка, BindBand BindCell, Report_OnBeginReport, Report_OnSave, Report_OnNewLine, редактируемый отчёт, нередактируемый отчёт, отчёт на диалоге, rs_main_proxy_ex, rs_put_report_data, CalcStockHistory

## Отличия отчётов от печатных форм в ЕМЕ-БД

В ЕМЕ-БД различают «отчёты» и «печатные формы». Печатная форма является одной из функций диалога и вызывается из тулбара диалога кнопкой с изображением принтера. К диалогу можно привязывать несколько печатных форм — для выбора нужной открывается меню.

Отчёт — самостоятельный модуль программы, который обычно имеет свой диалог для выбора параметров. Внутри же и то и другое — совершенно одно и то же: шаблоны отчётов и печатных форм конструируются в Визуальном Редакторе Отчетов и хранятся в БД в одной записи. в системе EME.WMS отчёты являются основным инструментом визуализации данных.

## Формирование простейшего отчёта с группировкой

### Создание диалога с параметрами отчёта

1. В «Редакторе диалогов» создаётся новый диалог с софтовым именем (например, `BuyersCallsReport.Dialog`).
2. В «Редакторе классов» создаётся EME-L класс (например, `BuyersCallsReport`).
3. Диалог привязывается к классу кнопкой «EME-L».
4. На шаблоне размещаются: статическое поле, текстовый редактор (для выбора даты), радио-кнопки для выбора группировки, кнопка «Отчёт».

Кнопка отчёта настраивается с функцией `ts_btn_show_reports` из `metd_sys.dll`. Параметр «Выражение для индекса отчёта»:

```EME-L
is_if(is_selected("BuyersCallsReport.GroupBy",0),0,1)
```

### Формирование шаблона отчёта

1. В «Редакторе отчетов» (Ctrl+Shift+R) создаётся новый отчёт.
2. В свойствах отчёта указываются:
   - Название отчёта
   - Диалог отчёта
   - Имя DLL: `metr_sys.dll`
   - Имя функции: `rs_main_query`
   - Запрос к ЕМЕ DB (имя SQL-запроса)

3. На шаблоне размещаются:
   - Статические текстовые поля (заголовки)
   - Ячейки DBField (поля данных из запроса)
   - Горизонтальный банд (для разворачивания строк)

### Передача данных в отчёт

Существует два способа передачи данных в отчёт в EME-L:

**Способ 1 — через «Переменную Результаты Запроса»:** создаётся переменная диалога типа «Указатель(void*)», данные из запроса помещаются в эту переменную. Достоинство — при переименовании запроса не нужно менять все ячейки.

**Способ 2 — прямое указание имени запроса:** в каждой ячейке и банде указывается имя запроса к EME DB. Недостаток — при переименовании запроса нужно обновлять все ячейки.

### Настройка ячеек DBField

Каждая ячейка DBField настраивается с функцией `rs_cell_query` из `metr_sys.dll`:

- Имя ячейки
- Тип поля (Дата, Текстовая строка, Целое число)
- Запрос к EME DB
- Поле запроса

### Настройка горизонтального банда

Горизонтальный банд настраивается с функцией `rs_band_query` из `metr_sys.dll`:

- Имя области: «SQL Отчёт»
- Тип: «Область данных»
- Запрос к EME DB

## Формирование печатной формы

Печатная форма создаётся аналогично отчёту через «Редактор отчетов». Для привязки к диалогу:

1. Открыть диалог в редакторе диалогов.
2. Нажать кнопку «Отчеты диалога».
3. Выбрать отчёт и перенести в левый список.
4. Установить флажок «Работать с отчетами».

После этого на диалоге появится кнопка с изображением принтера.

## Отчёт с разворачивающимся бандом (потайной банд)

в языке EME-L существует возможность создания отчётов с группировкой по 2 и более полям с возможностью скрытия/раскрытия вложенных групп.

### Использование переменной «Результат запроса»

В свойствах отчёта в параметрах функции `rs_main_query` устанавливается параметр «Переменная РЕЗУЛЬТАТЫ ЗАПРОСА» (например, «Результат»). Эта переменная создаётся в диалоге как «Переменная диалога» с типом «Указатель(void*)». Далее во всех ячейках и бандах вместо имени запроса указывается эта переменная.

### Потайной банд

Для реализации скрытых групп:

1. Добавляется вложенный горизонтальный банд внутри основного.
2. В свойствах нового банда устанавливается «Потайной банд: Скрывать».
3. В свойствах отчёта ставится галочка «Управление потайными бандами».

### Агрегация в разворачивающихся бандах

Для отображения суммы значений вложенной группы используется ячейка с функцией `rs_put_report_data` из `metr_sys.dll`:

- Параметр «Выражение»: `is_sum("Количество звонков")`
- Параметр «Обрабатывать RSM_RECALCULATE»: «Да»

## Нередактируемый отчёт на диалоге

в системе EME.WMS отчёт можно разместить непосредственно на диалоге.

### Настройка отчёта на диалоге

1. Создать отчёт с признаком «Использовать отчет как форму для ввода данных (отчет на диалоге)».
2. Привязать главную функцию `rs_main_proxy_ex` из `MetR_Sys.dll`.
3. В параметрах функции поставить «Сохраняемый отчёт» — «Да».
4. Добавить отчёт в «Отчеты диалога».
5. Разместить на диалоге орган типа «Рисунок» с привязкой к отчёту.
6. Растянуть орган на весь диалог (свойства «Верх» — «Низ», «Л» — «П»).

### Реализация EME-L класса для отчёта на диалоге

```EME-L
Select (HourQuery)
    SELECT
        {
            Hour = is_time(8 + is_line(), 0);
            Return Hour;
        }:TIME AS [Час]
    FROM
        NEW{11}
End Select

Query AS "Query";
```

Метод `Report_OnBeginReport()` формирует данные для отчёта:

```EME-L
Report_OnBeginReport()
{
    Report = Object("Report");

    Select (Query)
        SELECT
            [@Покупатель].[Фамилия] AS [Покупатель],
            [Дата],
            {
                Hour = is_time(is_hour(is_query(, "Время")), 0);
                Return Hour;
            }:TIME AS [Час],
            SUM({
                Return 1;
            }:INT) AS [Количество звонков]
        FROM
            [Звонки покупателей]
        GROUP BY
            [Покупатель], [Дата], [Час]
    End Select

    Report.BindBand("BuyersReport.BandDate", "Дата", Query, "Дата");
    Report.BindBand("BuyersReport.BandHour", "Час", HourQuery, "Час");
    Report.BindBand("BuyersReport.BandBuyer", "Покупатель", Query, "Покупатель");

    Report.BindCell("BuyersReport.Date", "Дата");
    Report.BindCell("BuyersReport.Hour", "Час");
    Report.BindCell("BuyersReport.Buyer", "Покупатель");
    Report.BindCell("BuyersReport.BuyerCount", Query, "Количество звонков");
}
```

**Важно:** для ячейки на пересечении горизонтального и вертикального бандов необходимо явно указать запрос, из которого будут взяты данные, иначе данные будут взяты из горизонтального банда.

## Редактируемый отчёт на диалоге

Для добавления возможности редактирования отчёта используется динамический банд — специальная категория бандов, разработанная для отчётов на диалоге.

### Особенности динамического банда

- Всегда имеет признак «Скрывать».
- Не разворачивается автоматически при формировании отчёта.
- Разворачивается в момент открытия его оператором.
- Позволяет сформировать «Отчёт в отчёте» (принцип «дрилл-даун»).

### Методы BindBand и BindCell для динамического банда

```EME-L
Report.BindBand("BuyersReport.BandCall", "Звонок", CallQuery, "Ключ", true, true);
Report.BindCell("BuyersReport.CallTime", "Время", true);
Report.BindCell("BuyersReport.CallText", "Содержание", true);
```

Два дополнительных параметра `BindBand` означают «Динамический» и «Добавление строк». Дополнительный параметр `BindCell` означает «Сохранять в БД».

### Управление динамическим бандом

```EME-L
BuyerCount_OnShowBand(Cell AS "Range")
{
    Date = Cell.GetVLine();
    Buyer = Cell.GetLine();
}

BandCall_OnBeforeDynamicOpen()
{
    Return 1;
}
```

### Механизм сохранения данных из редактируемого отчёта в БД

Для сохранения изменений реализуются следующие методы в EME-L:

- `BandCall_GetBlockID()` — возвращает уникальный идентификатор строк развернутого динамического банда.
- `Report_OnNewLine(blockID, Line)` — обработка добавления новой строки.
- `Report_OnDeleteLine(blockID, Line)` — обработка удаления строки.
- `Report_OnSaveLine(blockID, Line)` — возвращает номер реальной строки в БД.
- `Report_OnSave(blockID, Line, cellName, cellID, NewValue)` — запись изменений в БД.

```EME-L
BandCall_GetBlockID()
{
    Parameters.Empty();
    Parameters.PutParam("Банд","Звонок");
    Parameters.PutParam("Дата", Date);
    Parameters.PutParam("Покупатель", Buyer);
    Return Parameters.AsString();
}

Report_OnNewLine(blockID, Line)
{
    Parameters.SetString(blockID);
    Band = Parameters.GetParam("Банд");
    If (Band == "Звонок")
        CallRec.AppendAndSetLine();
        Return CallRec.GetLine();
    End If
}

Report_OnSave(blockID, Line, cellName, cellID, NewValue)
{
    Parameters.SetString(blockID);
    Band = Parameters.GetParam("Банд");
    If (Band == "Звонок")
        Date = Parameters.GetDateParam("Дата");
        Buyer = Parameters.GetParam("Покупатель");
        BuyerRec.GetFld("Фамилия").MustBeEQ(Buyer);
        BuyerRec.SetFirstLine();
        CallRec.SetLine(Line);
        CallRec.Put("@Покупатель", BuyerRec.GetLine());
        CallRec.Put("Дата", Date);
        If (cellName == "BuyersReport.CallTime")
            CallRec.Put("Время", NewValue);
        Else
            If (cellName == "BuyersReport.CallText")
                CallRec.Put("Содержание", NewValue);
            End If
        End If
    End If
}
```
