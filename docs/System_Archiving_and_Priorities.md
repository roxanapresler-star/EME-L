## Архивация ЕМЕ БД

в системе EME.WMS механизм архивации представляет собой формирование снимка состояния базы данных на указанные пользователем даты.

### Предварительные требования

- Пройти **тест целостности** (меню: «Система → Тест целостности базы данных») за 1–2 дня до архивации. Ошибки в структуре БД могут накапливаться, поэтому тест нужно проходить незадолго до начала.
- Иметь **не менее 50% свободного места** на диске, на котором работает ЕМЕ БД.

### Пошаговая инструкция архивации

1. **Отключить все сетевые рабочие станции** и убедиться, что никто не выполняет транзакции
2. **Остановить сервер EME.WMS**
3. **Запустить базу данных в монопольном режиме** с дополнительными ключами: `/bc` и `/si`
4. **Выполнить архивацию** через диалог **«Архивация БД»** (меню: «Утилиты БД → Архивация БД»)
5. **Переименовать** директорию со старыми данными по шаблону: `dat_год_месяц_дата`
6. **Запустить сервер** на новых данных (сгенерированных при архивации), проверить работоспособность
7. **Создать полный архив** папки с новыми данными (важно использовать архиватор, корректно работающий с временем модификации файлов — иначе будут проблемы с запуском рабочей станции)
8. **Распаковать архив** с новыми данными на всех рабочих станциях в соответствующие директории (предварительно удалив старые файлы)

в EME БД пункты 2, 7 и 8 актуальны **только для крупных складов**, работающих 24/7. В ином случае достаточно выровнять рабочие станции с сервером.

### Работа с диалогом «Архивация БД»

в системе EME.WMS диалог архивации доступен по меню: **«Утилиты БД → Архивация БД»**. Необходимо заполнить три параметра с датами:

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|----------------------|
| Дата отгрузки | Документы с датой отгрузки меньше указанной будут удалены | Прошлогоднее значение текущей даты |
| BBD строк | Если в строках документа нет дат BBD больше указанной — они будут удалены | Год с текущей даты |
| Фиксированные BBD | Для документов с фиксированными BBD (например 31.12.2099) | Указать эту дату или год с текущей даты |

После заполнения нажать **«Выполнить»** и дождаться завершения.

в EME БД результат архивации — новая папка данных с суффиксом **`Prun`** в корневом каталоге, после чего база запускается на этих данных. Текущие данные **не затрагиваются**.

## Системные приоритеты в EME.WMS

в системе EME.WMS системные приоритеты определяют критерии важности применения правил резервирования товара в приёмке/подборке в рамках формирования стратегий размещения.

### Расположение и настройка

Диалог **«Системные приоритеты»** доступен по меню: **«Утилиты БД → Система → Системные приоритеты»**. В диалоге указывается:

- Название класса с реализацией приоритета (в стиле **Camel Case**)
- Комментарий к приоритету
- Галочка **«Уход»** — если приоритет нужно использовать в параметрах подборки для стратегии размещения

в языке EME-L значения в данном диалоге могут редактироваться **только на филиале разработчика**. Системные приоритеты формируются в метапроекте и **недоступны в EME-L**.

Приоритеты используются в таблицах стратегий размещения (параметры прихода/подборки). Порядок расстановки определяется программой по умолчанию или вручную через браузер **«Приоритеты системные»**. Для управления последовательностью выставляются значения в колонке **«П»**.

### Создание нового системного приоритета

в системе EME.WMS для создания нового приоритета:

1. Создать новый файл приоритета в метапроекте (за образец брать `sc_ByRef.cpp`)
2. Придумать название класса приоритета, например: `CSearchClassBy — объект приоритета`
3. В метод `INNER(PriorityOperations)::FillAlgoStringArrayDefaultOutcome` (файл `r_PriorityOperations.cpp`) добавить две строки инициализации приоритета в соответствующее место
4. В рабочей станции (меню: «Утилиты БД → Система → Системные приоритеты») объявить новый приоритет, проставить галочку «Уход» при необходимости
5. Проверить, что приоритет доступен в стратегиях размещения (параметры прихода/подборки)

## Фильтры и алгоритмы пополнения/оптимизации в EME.WMS

в системе EME.WMS фильтры и алгоритмы пополнения/оптимизации настраиваются в диалоге по меню: **«Утилиты БД → Система → Алгоритмы пополнения»**. Значения редактируются **только на филиале разработчика**.

в языке EME-L алгоритмы отображаются в браузерах специализированных диалогов как применяемые параметры (например, параметр «Алгоритм оптимизации» в диалоге «Оптимизация склада»).

### Параметры алгоритма

| Параметр | Описание |
|----------|----------|
| Не используется | Алгоритм нельзя выбрать ни в одном браузере |
| PPU | Алгоритм для пополнения пикинга (параметр «Алгоритм» в диалоге «Задание на пополнение пикинга») |
| Фильтр прихода | Поиск ячейки для размещения товара (параметр «Дополнительный фильтр» в параметрах прихода) |
| Фильтр подборки | Резервирование ячеек при подборе заказов (параметр в параметрах подборки) |
| Заполнение приоритетов | Динамическое вычисление приоритетов типов ячеек для каждого товара при приходе (параметр «Динамическое заполнение приоритетов» в параметрах продукта) |
| Код | Наименование алгоритма (рекомендуется на русском языке) |
| Класс | EME-L класс реализации алгоритма |
| Метод | Метод EME-L класса с реализацией |
| ! | Флаг использования алгоритма по умолчанию |
| Описание | Текстовое описание |

### Классификация алгоритмов

в системе EME.WMS если у алгоритма **не проставлена ни одна** из галочек (PPU, Фильтр прихода, Фильтр подборки, Заполнение приоритетов) — он автоматически классифицируется как **алгоритм оптимизации склада**. Такие алгоритмы доступны в браузере диалога **«Оптимизация склада»**.
