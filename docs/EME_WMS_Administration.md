# Администрирование EME.WMS

**Описание:** Руководство по администрированию системы EME.WMS: установка, настройка, обновление, мониторинг и решение проблем.
**Сленг:** администрирование, настройка WMS, развёртывание, деплой, обновление

## Установка и развёртывание EME.WMS

Установка EME.WMS выполняется из релиз-версии или путём ручного развёртывания компонентов.

**Синонимы для поиска:** установка EME.WMS, развёртывание, деплой, инсталляция, настройка установки

### Варианты установки

| Вариант | Описание |
|---------|----------|
| Профессиональная версия | Полный набор модулей: сервер, рабстанция, ТСД, веб-порталы |
| Стандартная версия | Базовый набор: сервер, рабстанция |
| Демо-версия | Пробная версия с ограничениями |
| Только рабочая станция | Клиентская часть |

### Автоматическая установка

1. Запустить установщик EME.WMS
2. Выбрать вариант установки
3. Указать директорию установки
4. Дождаться завершения установки

### Ручная установка

1. Разархивировать `db_wms_release` в рабочую директорию
2. Убедиться, что в пути нет пробелов
3. Настроить ярлык `db.exe`:
   - Объект: путь к db.exe с параметрами
   - Рабочая папка: путь к metaproj/exe

## Создание сервера EME.WMS

Сервер EME.WMS создаётся через утилиту service.exe.

**Синонимы для поиска:** создание сервера, настройка сервера, служба WMS, service.exe, сервер транзакций

### Компоненты сервера

| Служба | Ключ | Описание |
|--------|------|----------|
| Сервер | /server | Сервер транзакций |
| Мозговой компьютер | /bc | Выполнение регламентных заданий |
| HTTP-сервер | /http | Сервер для ТСД и Монитора Заданий |

### Создание службы

1. Запустить `db.exe` от имени администратора
2. В диалоге запуска нажать "Добавить"
3. Выбрать "Выбрать папку с данными и параметры запуска"
4. Указать параметры:
   - Сетевой режим: сервер / рабочая станция /fullvm
   - Сервисы: /bc и/или /http
   - Заголовок БД: путь к .hdr файлу
   - Наименование службы
   - Путь к файлу параметров
5. Нажать "Установить службу"

### Настройка IP-адреса сервера

1. Остановить службу
2. Запустить рабочую станцию в моно-режиме
3. Открыть настройки: Ctrl-Shift-W
4. Вкладка "Сеть":
   - IP адрес сервера
   - Порт прослушивания
5. Сохранить и перезапустить

### Запуск службы через service.exe

1. Перейти в `kernel.x64/exe`
2. Запустить `service.exe`
3. В диалоге "ЕМЕ БД сервер менеджер":
   - Выбрать службу из списка
   - Нажать "Запустить"
4. Проверить CATASTRF.log:
   - "Сервер транзакций загружен"
   - "WEB-сервер загружен"
   - "Мозговой компьютер EME загружен"

## Обновление EME.WMS

Обновление промышленно функционирующей EME.WMS требует тщательной подготовки.

**Синонимы для поиска:** обновление WMS, апдейт, обновление версии, патч, обновление продакшена

### Компоненты для обновления

- Ядро
- Метапроект
- ТСД
- EME-L (классы, диалоги, запросы)
- Монитор Заданий

### Процедура обновления

1. **Согласовать время** с заказчиком и руководителем проекта
2. **Перенести пакет обновления** на сервер
3. **Остановить службы** рабочих разделов
4. **Остановить службу** мастер раздела
5. **Создать бэкапы** данных, ядра, метапроекта
6. **Собрать ядро и метапроект**
7. **Запустить мастер раздел** в режиме службы
8. **Внести изменения** по EME-L через модуль заливок
9. **Выполнить тесты**:
   - Тест структуры БД
   - Тест целостности
   - Тест интерфейсов
   - Перекомпиляция классов
10. **Запустить службы** рабочих разделов
11. **Проверить логи** на ошибки
12. **Установить новые версии** ТСД и Монитора Заданий

### Тесты после обновления

```eme-l
'Тест структуры БД: Система -> Ресурсы -> Структура БД -> Тест структуры'
'Тест целостности: Система -> Консоль сервера -> Система -> Тест целостности'
'Тест интерфейсов: Система -> Ресурсы -> Тестирование -> Тест интерфейса'
'Перекомпиляция: Система -> Ресурсы -> Редактор классов -> Сервис -> Все классы'
```

## Архивация БД

Архивация — удаление старых данных для оптимизации размера БД.

**Синонимы для поиска:** архивация, очистка БД, prune, удаление старых данных, оптимизация БД

### Подготовка к архивации

1. Пройти тест целостности (за 1-2 дня до архивации)
2. Убедиться в наличии 50% свободного места на диске
3. Отключить все рабочие станции

### Процедура архивации

1. Остановить сервер
2. Запустить БД в моно-режиме с ключами /bc и /si
3. Открыть диалог: Утилиты БД → Архивация БД
4. Заполнить параметры:
   - Дата для удаления документов
   - Дата для BBD
   - Дата для фиксированных BBD
5. Нажать "Выполнить"
6. Переименовать старую директорию данных
7. Запустить сервер на новых данных
8. Проверить работоспособность
9. Создать архив новых данных

## Хэш-БД

Хэш-БД предназначена для синхронизации работы кладовщиков с ТСД и поиска марок.

**Синонимы для поиска:** Хэш-БД, HashDB, синхронизация ТСД, марки, честный знак, ЕГАИС

### Назначение

- Блокировка параллельных работ по одному приказу
- Поиск ссылки на марку по штрих-коду
- Обязательна для работы с маркированным товаром

### Развёртывание Хэш-БД

1. Перейти в директорию HASHBD
2. Настроить ярлык запуска:
   - Путь к db.exe
   - Режим: /mono или /fullvm
   - Путь к HDR-файлу
   - Ключ: /http
3. Запустить Хэш-БД
4. Настроить сетевые параметры:
   - IP-адрес: адрес компьютера
   - Порт: 7777
5. Перезапустить сервер

## Репликация БД

Репликация обеспечивает наличие копии БД на SQL-сервере.

**Синонимы для поиска:** репликация, PostgreSQL, SQL-реплика, выгрузка в SQL, ExternalDB

### Настройка репликации

1. **Указать строку подключения** в настройках модулей:
   - Предпочтения → Репликация → Строка подключения
   - Пример: `ODBC:Driver={PostgreSQL ANSI(x64)};Server=localhost;DataBase=WMS_REPLICA;UID=postgres;PWD=123;`

2. **Выполнить статическую выгрузку**:
   - Диалог "Репликация БД"
   - Проверить подключение
   - Нажать "Выгрузить данные"
   - Выбрать записи для выгрузки

3. **Настроить динамическую выгрузку**:
   - Предпочтения → Репликация → Директория импорта
   - Путь к `/db_box/sys/export`

4. **Перезапустить с ключом /d_replica**

5. **Импортировать первый файл экспорта**

### Дополнительные настройки в ERPReplication

```eme-l
'Метод DoImport():'
'Для системного филиала: FileMask = "*.127"'

'Метод ReplicationLoop():'
'Period = 5 60 1000 ' Обновление каждые 5 минут

'Метод DeleteOldExportFiles():'
'Depth = 2 ' Удаление файлов старше 2 часов
```

### Формат таблиц в SQL-реплике

Имя таблицы SQL-базы формируется по шаблону: `{Номер записи БД EME}.{Имя записи БД EME}`

Имя колонки таблицы формируется по шаблону: `{Номер поля БД EME}.{Имя поля БД EME}`

### Решение проблем при репликации

| Проблема | Решение |
|----------|---------|
| Ошибка дублирующихся ключей при статической выгрузке | В классе `ERPReplication` найти методы `CheckDupKeys()` и `ResolveDupKeys()`, в переменную RecordNo ввести id проблемной записи, запустить методы по очереди |
| Ошибка кодировки (UTF-8 не найдено соответствие в Windows-1251) | Не включать в реплику запись, вызывающую ошибку |

## Развертывание промежуточной SQL базы данных (интерфейс интеграции ERP-WMS)

Для развертывания промежуточной SQL-ориентированной базы данных, посредством которой осуществляется обмен данными между ERP и WMS, необходимо выполнить следующие шаги:

**Синонимы для поиска:** интеграция ERP-WMS, промежуточная база, транзитная база, SQL Server для интеграции, ERPEngine.xml

### Порядок развертывания

1. Установить Microsoft SQL Server 2008 R2 (установщик доступен по ссылке: https://www.microsoft.com/ru-ru/download/details.aspx?id=30438).
2. Выполнить настройку подключения промежуточной базы данных к EME.WMS, используя Диспетчер конфигурации SQL Server.
3. Выполнить настройку правила брандмауэра для SQL-сервера, используя Монитор брандмауэра Защитника Windows.
4. Создать базу данных в Microsoft SQL Server, используя SQL Server Management Studio и прописать строку подключения к ней в EME.WMS (используя диалог "Классификатор складов").
5. Выполнить автоматическую настройку интерфейса ERP-WMS в EME.WMS, используя диалог "Интерфейс ERP-WMS".
6. Выполнить синхронизацию структуры промежуточной базы данных с диалогом "Интерфейс ERP-WMS" и удостовериться, что промежуточная база содержит все таблицы механизма интеграции.

### Ручная настройка интерфейса ERP-WMS

Пользователь может выполнять настройку интерфейса ERP-WMS вручную:

1. Произвести настройку экспорта документов в EME.WMS, используя настройки модулей, а также диалоги "Интерфейс ERP-WMS", "Классификатор складов", "Типы экспорта документов" и "Классификатор транзакций".
2. Выполнить настройку событий планировщика для интерфейса ERP-WMS, используя диалог "Навигатор канальной схемы" или диалог "Мастер настроек интерфейса ERP-WMS".
3. Выполнить настройку EME-L обработчиков для интерфейса ERP-WMS, используя диалог "Организация-собственник" или диалог "Мастер настроек интерфейса ERP-WMS".

### Установка Microsoft SQL Server

Чтобы установить Microsoft SQL Server, на сайте Microsoft необходимо скачать файл "SQLEXPRWT_x64_RUS.exe", поскольку он содержит все необходимые компоненты для работы сервера (в том числе и SQL Server Management Studio). После скачивания - нужно запустить установщик.

**Шаги установки:**

1. В открывшемся окне установщика выбрать пункт "Установка" и перейти по ссылке "Новая установка или добавление компонентов к существующей установке".
2. В окне "Тип установки" выбрать "Новая установка или добавление общих компонентов" и нажать на кнопку "Далее".
3. Согласиться с лицензионным соглашением.
4. Выбрать компоненты для установки SQL Server (оставить все галочки, которые уже проставлены).
5. Настройка экземпляра SQL-сервера: можно выбрать "Экземпляр по умолчанию". Имя экземпляра необходимо запомнить, оно требуется для формирования строки подключения к СУБД.
6. Конфигурация SQL-сервера: для службы "SQL Server Database Engine" указать имя учетной записи (СИСТЕМА).
7. Настройка компонента Database Engine: указать смешанный режим проверки подлинности (как по учетной записи Windows, так и по паролю) и ввести пароль для учетной записи системного администратора SQL Server (пользователь sa).
8. Согласиться или не согласиться отправлять отчеты об ошибках в Microsoft и нажать на кнопку "Далее".
9. Дождаться завершения процесса установки Microsoft SQL Server.

### Настройка подключения Microsoft SQL Server к EME.WMS

По умолчанию, после установки SQL Server - подключиться к EME.WMS будет невозможно. Поэтому необходимо произвести настройку подключения.

**Шаги настройки:**

1. Открыть Диспетчер конфигурации SQL Server и в левом окне выбрать "Службы SQL Server".
2. Зайти в свойства службы "SQL Server, обозреватель" и установить встроенную учетную запись (выбрать соответствующую радио-кнопку).
3. Перейти на вкладку "Служба" и выставить режим запуска на "Авто". Для применения изменений нажать на кнопку "ОК".
4. Произвести запуск службы: нажать правую кнопку мыши на строке со службой и выбрать "Запустить".
5. Перейти на вкладку "Сетевая конфигурация SQL Server -> Протоколы для SQLEXPRESS" и включить протокол TCP/IP.
6. Перезапустить службу SQL Server для применения изменений.
7. Вновь перейти на вкладку "Сетевая конфигурация SQL Server -> Протоколы для SQLEXPRESS", открыть правой кнопкой мыши свойства протокола TCP/IP и указать, что нужно прослушивать все IP-адреса.
8. На вкладке "IP-адреса" (раздел IPAll) найти настройку "TCP-порт" и указать произвольный незанятый статический порт (например - 49158). При этом настройку "Динамические TCP-порты" нужно очистить.

### Настройка правила брандмауэра для SQL-сервера

Для стабильной работы SQL-сервера требуется создать правило для брандмауэра.

**Шаги создания правила:**

1. Перейти в брандмауэр Windows (Монитор брандмауэра Защитника Windows) и в правой части окна нажать на кнопку "Создать правило".
2. Указать, что нужно создать правило для программы.
3. Выбрать радио-кнопку "Путь программы" и прописать путь к .exe файлу SQL-сервера: `C:\Program Files\Microsoft SQL Server\MSSQL_10{название экземпляра}\MSSQL\Binn\sqlservr.exe`.
4. На вкладке "Действие" выбрать радиокнопку "Разрешить подключение".
5. На вкладке "Профиль" оставить все галочки так, как они проставлены изначально.
6. На вкладке "Имя" написать имя правила брандмауэра.
7. Найти в Мониторе брандмауэра созданное правило "SQL Server", нажать на него правой кнопкой мыши и выбрать "Свойства".
8. Выбрать вкладку "Протоколы и порты", в настройке "Локальный порт" выбрать "Специальные порты" и номер статического порта, который зарезервирован под SQL-сервер.

### Создание базы данных в Microsoft SQL Server

Перед настройкой интерфейса ERP-WMS в EME.WMS необходимо вручную создать базу данных в СУБД Microsoft SQL Server.

**Шаги создания базы:**

1. Открыть SQL Server Management Studio и пройти аутентификацию на SQL-сервере.
2. Нажать правую кнопку мыши на ветке "Базы данных" и в открывшемся контекстном меню выбрать пункт "Создать базу данных".

### Формирование строки подключения

Если EME.WMS и ERP находятся в разных локальных сетях, необходимо определиться с расположением SQL сервера. Рекомендуется располагать SQL сервер там же, где запускается база EME.WMS. Для того, чтобы транзитная база была доступна для ERP - необходимо пробросить порт SQL сервера наружу.

**Примеры строк подключения:**

- База расположена на том же хосте, что и запущенная EME.WMS: `ODBC:Driver={SQL Server};Server=WS132\\SQLEXPRESS;Database=WMS_ERP;Uid=sa;Pwd=***;`
- База расположена на удаленном хосте: `ODBC:Driver={SQL Server};Server=X.X.X.X\\SQLEXPRESS,56231;Database=WMS_ERP_BOX2;Uid=sa;Pwd=***;` (здесь явно указывается статический порт SQL-сервера).

В файле ERPEngine.xml для использования в ERP должна быть прописана строка подключения через внешний IP-адрес. Для этого в диалоге "Интерфейс ERP-WMS" (вкладка "Настройки") нужно прописать тестовую строку подключения, где в параметре Server указывается внешний IP-адрес.

## Сборка метапроекта в Windows

Метапроект представляет собой расширение емельных программ на языке С++, то есть программ на внутреннем языке EME-L и обеспечивает работу одной версии с несколькими проектами. Каждому проекту соответствует свой dll – драйвер проекта, входящий в состав метапроекта, и данные БД.

**Синонимы для поиска:** сборка метапроекта, Visual Studio, metaproj, драйвер проекта, EMEdb, kernel.x64

### Подготовка директории для сборки

В системе Windows необходимо подготовить директорию для сборки метапроекта (например - EMEdb), которая в обязательном порядке содержит следующие поддиректории:

- kernel.x64 - директория с файлами ядра.
- metaproj - директория с файлами метапроекта.

### Требования для сборки

Для сборки метапроекта требуются:

- Исходный код метапроекта для базовой версии ЕМЕ БД: http://svn.eme.ru/metaproj/trunk/metaproj/
- Библиотеки Qt: http://svn.eme.ru/metaproj/trunk/qt/
- Библиотеки Boost: http://svn.eme.ru/metaproj/trunk/boost/
- Исходный код ядра: http://svn.eme.ru/metaproj/kernel/kernel.x64-51.1/

### Пошаговая инструкция по сборке метапроекта в Windows

1. Выгрузить из SVN исходный код библиотек Qt и Boost с помощью TortoiseSVN.
2. Задать в свойствах системы Windows переменные среды с указанием каталогов библиотек Qt и Boost.
3. Выполнить выгрузку исходного кода метапроекта из SVN в составе семи директорий (MetC_Sys, MetC_WMS, MetD_Sys, MetR_Sys, _Lib_x64, _Obj_x64 и outerWMS) с помощью TortoiseSVN.
4. Выполнить выгрузку исходного кода ядра из SVN в составе двух директорий: src и exe с помощью TortoiseSVN.
5. Установить Visual Studio.
6. В директории outerWMS открыть файл outerWMS.sln (файл решения).
7. Произвести настройку параметров отладки outerWMS и остальных проектов в решении, используя свойства проекта в Visual Studio.
8. Назначить outerWMS автозагружаемым проектом в Visual Studio.
9. Запустить процесс сборки метапроекта, указав конфигурацию Release x64.

### Переменные среды для библиотеки Qt

Для библиотеки Qt необходимо настроить следующие переменные среды:

- QMAKESPEC - \\qt\\qt-4.8.6-x64-msvc2010\\mkspecs.
- QT_INCLUDE - \\qt\\qt-4.8.6-x64-msvc2010\\include.
- QT_INCLUDE_V142 - \\qt\\qt-everywhere-src-5.15.2\\qtbase\\include.
- QT_LIB - \\qt\\qt-4.8.6-x64-msvc2010\\lib.
- QT_LIB_V142 - \\qt\\qt-everywhere-src-5.15.2\\qtbase\\lib.
- QT_LIB_X64 - \\qt\\qt-4.8.6-x64-msvc2010\\lib.
- QT5_CMAKE_DIR - \\qt\\5.13.2\\android_x86\\lib\\cmake.
- QT5_DIR - \\qt\\5.13.2.
- QT5CORE_DIR - \\qt\\5.13.2\\android_armv7\\lib\\cmake\\Qt5Core.

### Переменные среды для библиотеки Boost

Для библиотеки Boost необходимо настроить следующие переменные среды:

- BOOST_DIR - \\boost\\boost_1_53_0.
- BOOST_DIR_V142 - \\boost\\boost_1_75_0.

### Настройка параметров отладки метапроекта в Visual Studio

Для каждого проекта необходимо задать следующие параметры отладки:

- Команда - {Рабочий каталог ЕМЕ БД}\\kernel.x64\\exe\\debug\\db.exe.
- Аргументы команды - /mono "{Рабочий каталог ЕМЕ БД}\\dat0ws/eme.hdr" /d_emel /d_dataslice.
- Рабочий каталог - ..\\..\\exe\\Debug.

### Решение проблем при сборке метапроекта в Visual Studio

| Проблема | Решение |
|----------|---------|
| Ошибка отсутствия заголовочного файла | Установить пакет MFC для C++ через Visual Studio Installer: компонент "MFC C++ для средств сборки версии 142 (x64 и x86)" |
| Предупреждение "svn не является внутренней или внешней командой" | Установить инструменты командной строки в TortoiseSVN: установщик -> Modify -> command line client tools |
| Не найденный проект MetC_CLR в обозревателе решений | Удалить из решения, поскольку он необходим только для сборки 3PL-портала |

## Сборка метапроекта на Linux

Помимо операционной системы Windows, метапроект можно собрать и в системе Linux. В данной инструкции используется виртуальная машина Astra Linux, развернутая на Hyper-V.

**Синонимы для поиска:** сборка метапроекта Linux, Astra Linux, deploy.sh, alm, config.json

### Подготовка директории для сборки

В системе Linux необходимо подготовить директорию для сборки метапроекта (например - eme), и поместить её внутрь служебной директории /opt. Директория eme в обязательном порядке содержит следующие поддиректории:

- project_name - директория, которая должна содержать файл конфигурации системы, бинарные файлы системы, скрипты для разворачивания системы, а также директории данных для служб (мастер-сервер, сервер раздела, мозговой компьютер, http-сервер).
- azure - директория, которая должна содержать склонированный репозиторий для развертывания и обновления системы.

### Пошаговая инструкция по сборке метапроекта на Linux

1. В системе Windows подготовить данные для сборки метапроекта - данные мастер сервера и сервера раздела, задав требуемые разделы БД (используя настройки модулей).
2. Подготовить систему Linux для сборки метапроекта.
3. Подготовить директорию eme для сборки метапроекта.
4. Отредактировать конфигурационный файл config.json в директории /opt/eme/alm.
5. Сгенерировать службы системы, выполнив скрипт alm/generate_services.sh и произвести настройку 4-х служб в моно-режиме.
6. Произвести стартовый запуск всех 4-х служб в сетевом режиме и удостовериться в их корректной работе.
7. После запуска службы мастер-сервера - подключиться через рабочую станцию (в системе Windows) к мастер серверу и включить событие системного импорта (используя Навигатор канальной схемы).
8. После запуска службы сервера раздела - подключиться через рабочую станцию (в системе Windows) к серверу раздела и включить события интеграционного импорта/экспорта (используя Навигатор канальной схемы).
9. Запустить процесс сборки метапроекта, выполнив скрипт alm/deploy.sh (при этом две созданные рабочие станции в системе Windows должны оставаться открытыми).

### Подготовка данных для сборки метапроекта на Linux

В системе Windows необходимо взять исходники рабочей станции EME.WMS и на их основе сформировать две директории данных:

- dat_master - содержит данные мастер-сервера.
- dat_server - содержит данные сервера раздела.

Для этого достаточно два раза продублировать исходную директорию с данными (dat0ws) и назвать директории соответствующими названиями.
