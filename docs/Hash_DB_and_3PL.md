## Развертывание Хэш-БД

### Назначение Хэш-БД в системе EME.WMS

Хэш-БД предназначена для синхронизации одновременной работы кладовщиков с разных ТСД или рабочих станций. Например, при контроле одного заказа или сканировании одной марки Честного Знака/ЕГАИС. Если два кладовщика одновременно начинают выполнять один приказ — первому разрешается выполнение, второму блокируется. Хэш-БД обеспечивает моментальную блокировку параллельных работ.

Вторая задача Хэш-БД — поиск ссылки на марку (Честный знак/ЕГАИС) по штрих-коду товара. Хэш-БД **обязательна к развертыванию**, если в EME.WMS загружается срез с ТТН маркированного товара.

Хэш-БД реализована как отдельная база данных, не встроена в EME.WMS, запускается отдельным HTTP-сервером с ключом `/http`. Работает на версии ядра **47.3** (без лицензий). Для сборки используется классический метапроект с тремя DLL: `Metc_sys.dll`, `Metd_sys.dll`, `Metr_sys.dll`.

Архив с последней версией Хэш-БД: `\\srv002\db_box_5.0\HashDB.zip`.

### Пошаговая инструкция развертывания Хэш-БД

1. Настроить параметры запуска Хэш-БД через `db.exe`
2. Настроить сетевые параметры через настройки рабочей станции
3. Перезапустить сервер Хэш-БД и проверить настройки

### Запуск Хэш-БД

Перейдите в директорию `HASHBD` внутри рабочего каталога базы EME.WMS, затем в `/kernel.x64/exe` запустите `db.exe`.

Параметры запуска:
- Режим: `fullvm`, `ws` или `mono`
- Путь до HDR-файла с данными Хэш-БД
- Флаг `/http` для запуска в режиме сервера

Пример ярлыка (строка «Объект»):
```
C:\wms_release\HASHBD\kernel.x64\exe\db.exe /mono "C:\wms_release\HASHBD\dat_ws\EME.HDR" /http
```

Строка «Рабочая папка» (путь к скомпилированным исходникам метапроекта):
```
C:\wms_release_2025\HASHBD\metaproj\exe\
```

### Настройка сетевых параметров Хэш-БД

При загрузке срезов с ТТН маркированного товара EME.WMS выдаёт предупреждение о необходимости развернуть Хэш-БД на **порту 7777**.

При первом запуске Хэш-БД разворачивается на локальном IP-адресе. Для настройки:

1. Запустив сервер, нажмите `Ctrl-Shift-Enter` для входа в рабочую станцию
2. Откройте настройки рабочей станции (`Ctrl-Shift-W` или «Система → Ресурсы → Настройки рабочей станции»)
3. В «HTTP-сервер → IP адрес» укажите необходимый IP, нажмите «Применить»
4. Перезагрузите сервер (вкладка «HTTP» → «Перезагрузить»)
5. Проверьте обновлённые сетевые настройки при повторном запуске

## Ручная выгрузка данных по SSCC в Хэш-БД

Ручная выгрузка актуальна при проблемах с маркированным товаром, чаще всего — SSCC отсутствует в Хэш-БД.

**Пример проблемы:** при инвентаризации маркированного товара на ТСД отображается «Партия привязана к другому продукту», хотя штрих-код и SSCC отсканированы корректно.

### Порядок действий

1. Откройте диалог «Просмотр SSCC» («Регистры → SSCC»). Через браузер (`F3`) выберите требуемый номер SSCC
2. Выставьте галочку «Выгрузить в Хэш-БД» и сохраните (`Ctrl-S`)
3. Откройте редактор классов (`Ctrl-Shift-C`), найдите класс **EGAIS** → потомок **ByTime** → метод `AddKeysByTime()`
4. Найдите фрагмент кода в языке EME-L:

```emel
mapSSCCRef = Object("Map", 1013);
mapSSCCRef.SetKeyType(1);
mapSSCCHSRef = Object("Map", 1013);
mapSSCCHSRef.SetKeyType(1);
mapBoxesRef = Object("Map", 1013);
mapBoxesRef.SetKeyType(1);
mapStampsRef = Object("Map", 1013);
mapStampsRef.SetKeyType(1);
```

5. Установите курсор на этот фрагмент и нажмите `F5` для выполнения

В `CATASTRF.log` должны появиться сообщения:
- «Добавление в HashDB SSCC с признаком»
- Начало транзакции «Очистка признака SendToHash»

Ошибок быть не должно. После перезагрузки диалога «Просмотр SSCC» галочка «Выгрузить в Хэш-БД» должна исчезнуть у выгруженного номера.

## Эмулятор Хэш-БД

В системе EME.WMS имеется эмулятор Хэш-БД, позволяющий имитировать запросы для работы срезов с маркированными товарами без развёртывания полноценной Хэш-БД. Эмулятор стартует автоматически при запуске WMS.

За эмуляцию отвечает EME-L класс **HashDBEmulator**, наследуемый от класса **WEB**. Методы эмулятора:

- **FindKey** — эмуляция запроса на поиск ключей
- **AddValue** — эмуляция запроса на добавление ключа
- **SetValue** — эмуляция запроса на изменение значения ключа
- **CopyHead** — копирует в ответ шапку (номер модуля, срок жизни ключа, максимальная длина ключа)
- **IsTryFind** — поиск ключа (находится нужная строка в записи и осуществляется проверка)

### Настройка эмулятора

Проверьте настройки модулей (`Ctrl-Shift-M`): вкладка «Система → Хеш-БД». IP-адрес должен совпадать с адресом компьютера, порт — с портом HTTP-сервера рабочей станции. Иначе при работе со срезами маркированного товара будут возникать ошибки таймаута подключения.

## Сборка веб-портала 3PL

### Предварительные требования

Для сборки 3PL в Windows необходимы:
- **.NetFramework 4.5**
- **ASP.NET MVC4**
- **.NET Framework 4.6.1 SDK**

Сертификаты: `http://svn.eme.ru/metaproj/trunk/metaproj/src/MetC_CLR/Certs/`

Исходный код сайта 3PL: `http://svn.eme.ru/metaproj/trunk/metaproj/src/.NET/web.3pl.Kendo/`

Также требуется собранный метапроект с добавленным модулем **MetC_CLR** (`http://svn.eme.ru/metaproj/trunk/metaproj/src/MetC_CLR`).

### Пошаговая инструкция сборки 3PL

1. Установить IIS-сервер и импортировать сертификаты через MMC
2. Пересобрать метапроект в Visual Studio с модулем MetC_CLR
3. Подготовить рабочую станцию EME.WMS к использованию 3PL
4. Проверить работоспособность сайта в Visual Studio, собрать и скопировать в `inetpub`
5. Добавить сайт на сервер IIS и проверить работу

### Установка IIS и импорт сертификатов

Установите IIS через «Включение или отключение компонентов Windows» (галочка «Службы IIS»). После установки выполните в командной строке от имени администратора:

```
cd C:\Windows\Microsoft.NET\Framework\v4.0.30319
dism /online /enable-feature /featurename:IIS-ASPNET45
iisreset
```

Импорт сертификатов через консоль MMC (`Win+R` → `mmc`):

1. «Файл → Добавить или удалить оснастку» → «Сертификаты» → «Для учетной записи компьютера» → «Локальный компьютер»
2. «Доверенные корневые центры сертификации» → «Все задачи → Импорт»
3. Импортируйте три сертификата: `WcfServer.cer`, `WcfServer.crl`, `WcfServer.pfx` (пароль: `32543254`)
4. Поместите в хранилище «Доверенные корневые центры сертификации»

### Сборка метапроекта с MetC_CLR

1. Откройте `outerWMS.sln`
2. «Файл → Добавить → Существующий проект» → выберите директорию MetC_CLR
3. В файле `p_WMS.h` (путь `metaproj\src\MetC_WMS\innerREC`) раскомментируйте `include "MetC_CLR/innerRec/p_CLR.h"`
4. Соберите MetC_CLR, затем outerWMS. Результат — новая DLL `metc_clr`

### Подготовка рабочей станции EME.WMS к 3PL

Запустите рабочую станцию с ключом `/http` и настройте:

1. **Настройки рабочей станции** (`Ctrl-Shift-W`): «Система → Включить CLR» = **Да**
2. **Настройки модулей** (`Ctrl-Shift-M`): «Предпочтения → 3pl → Включить CLR» = **Да**
3. **Навигатор канальной схемы**: «Программные модули» — MetC_CLR должна быть в перечне

После перезапуска проверьте `CATASTRF.log`: строка «Eme.Services.WcfServer: Сервер запущен!» подтверждает готовность.

### Сборка и публикация сайта 3PL

1. Откройте `web.3pl` в Visual Studio
2. Проверьте ссылку на `Kendo.Mvc` (если нет — добавьте: `..\Packages\TelerikMvcKendo.Q2.2015\lib\Kendo.Mvc.dll`, «Копировать локально» = True)
3. Соберите сайт (`F5`)
4. Запустите `publish.bat` внутри директории `web.3pl`
5. Переименуйте появившуюся директорию `Publish` в `3PL` и переместите в `C:\inetpub`

### Добавление сайта на IIS

1. Откройте «Диспетчер служб IIS»
2. «Сайты → Добавить веб-сайт»: имя **3PL**, пул приложений **.NET v4.5**, физический путь `C:\inetpub`, свободный порт (например, **444**)
3. В «Дополнительные параметры» укажите пул **.NET v4.5**
4. Нажмите «Обзор» — введите логин и пароль от рабочей станции для входа в интерфейс портала 3PL
