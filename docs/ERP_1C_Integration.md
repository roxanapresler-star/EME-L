## Развертывание промежуточной SQL базы данных (интерфейс интеграции ERP-WMS)

Для организации обмена данными между ERP и WMS в системе EME.WMS используется промежуточная SQL-ориентированная база данных. Данные между системами передаются через экспорт и импорт сообщений, записываемых в таблицы этой транзитной базы.

В системе EME.WMS интерфейс интеграции ERP-WMS позволяет настроить автоматический или ручной обмен данными с внешней ERP-системой. Промежуточная база выступает буфером, в который WMS записывает исходящие сообщения и из которого считывает входящие сообщения от ERP.

### Пошаговая процедура развертывания

1. Установить Microsoft SQL Server 2008 R2 (рекомендуется файл `SQLEXPRWT_x64_RUS.exe`, содержащий все необходимые компоненты, включая SQL Server Management Studio).

2. Выполнить настройку подключения промежуточной базы данных к EME.WMS через Диспетчер конфигурации SQL Server.

3. Создать правило брандмауэра для SQL-сервера через Монитор брандмауэра Защитника Windows.

4. Создать базу данных в Microsoft SQL Server через SQL Server Management Studio и прописать строку подключения в EME.WMS (через диалог «Классификатор складов»).

5. Выполнить автоматическую настройку интерфейса ERP-WMS через диалог «Интерфейс ERP-WMS».

6. Выполнить синхронизацию структуры промежуточной базы с диалогом «Интерфейс ERP-WMS» и убедиться, что база содержит все таблицы механизма интеграции.

При необходимости пользователь может выполнять настройку интерфейса ERP-WMS вручную:

1. Настроить экспорт документов через настройки модулей и диалоги: «Интерфейс ERP-WMS», «Классификатор складов», «Типы экспорта документов», «Классификатор транзакций».
2. Настроить события планировщика через «Навигатор канальной схемы» или «Мастер настроек интерфейса ERP-WMS».
3. Настроить EME-L обработчики через диалог «Организация-собственник» или «Мастер настроек интерфейса ERP-WMS».

## Установка и настройка Microsoft SQL Server

### Установка Microsoft SQL Server

Для установки необходимо скачать файл `SQLEXPRWT_x64_RUS.exe` с сайта Microsoft. Данный установщик содержит все компоненты, включая SQL Server Management Studio.

**Ключевые шаги установки:**

- Выбрать пункт «Новая установка или добавление компонентов к существующей установке».
- Выбрать тип установки: «Новая установка или добавление общих компонентов».
- Принять лицензионное соглашение.
- Оставить все предлагаемые компоненты для установки (не снимать галочки).
- Настроить экземпляр SQL-сервера — можно выбрать «Экземпляр по умолчанию». Имя экземпляра потребуется для формирования строки подключения к СУБД.
- Для службы «SQL Server Database Engine» указать имя учетной записи «СИСТЕМА».
- Указать смешанный режим проверки подлинности (по учетной записи Windows и по паролю) и задать пароль для учетной записи системного администратора SQL Server (пользователь `sa`).

В системе EME.WMS строка подключения к SQL Server формируется на основе имени экземпляра и учётных данных администратора `sa`.

### Настройка подключения Microsoft SQL Server к EME.WMS

По умолчанию после установки SQL Server подключение из EME.WMS невозможно. Необходима дополнительная настройка через Диспетчер конфигурации SQL Server:

1. В разделе «Службы SQL Server» зайти в свойства службы «SQL Server, обозреватель» и установить встроенную учетную запись.
2. На вкладке «Служба» выставить режим запуска на «Авто» для автоматического старта после перезагрузки компьютера.
3. Запустить службу через контекстное меню.
4. В разделе «Сетевая конфигурация SQL Server → Протоколы для SQLEXPRESS» включить протокол TCP/IP.
5. Перезапустить службу SQL Server для применения изменений.
6. Открыть свойства протокола TCP/IP и указать прослушивание всех IP-адресов.
7. На вкладке «IP-адреса» в разделе IPAll задать произвольный незанятый статический TCP-порт (например, `49158`). Настройку «Динамические TCP-порты» необходимо очистить.

В системе EME.WMS статический порт используется в строке подключения к удалённому SQL-серверу.

### Настройка правила брандмауэра для SQL-сервера

Для стабильной работы SQL-сервера требуется создать правило входящего подключения в Мониторе брандмауэра Защитника Windows:

1. Нажать «Создать правило» → выбрать правило для программы.
2. Указать путь к исполняемому файлу SQL-сервера: `C:\Program Files\Microsoft SQL Server\MSSQL_10.{название_экземпляра}\MSSQL\Binn\sqlservr.exe`.
3. Выбрать действие «Разрешить подключение».
4. Оставить все профили по умолчанию.
5. Задать имя правила (например, «SQL Server»).
6. В свойствах созданного правила на вкладке «Протоколы и порты» указать специальный локальный порт — номер статического порта, зарезервированного под SQL-сервер.

## Создание базы данных и формирование строки подключения

### Создание базы данных в SQL Server Management Studio

Перед настройкой интерфейса ERP-WMS необходимо вручную создать базу данных в СУБД Microsoft SQL Server:

1. Открыть SQL Server Management Studio и пройти аутентификацию на SQL-сервере.
2. Нажать правую кнопку мыши на ветке «Базы данных» → «Создать базу данных».

### Формат строки подключения в EME.WMS

Строка подключения к внешней базе данных прописывается в карточке склада (диалог «Классификатор складов»). Формат строки подключения:

- **База на том же хосте, что и EME.WMS:**
  ```
  ODBC:Driver={SQL Server};Server=WS132\SQLEXPRESS;Database=WMS_ERP;Uid=sa;Pwd=123;
  ```

- **База на удалённом хосте (с явным указанием статического порта):**
  ```
  ODBC:Driver={SQL Server};Server=10.1.6.4\SQLEXPRESS,56231;Database=WMS_ERP_BOX2;Uid=sa;Pwd=123;
  ```

Если EME.WMS и ERP находятся в разных локальных сетях, SQL-сервер рекомендуется размещать там же, где запускается база EME.WMS, а для ERP пробросить порт SQL-сервера наружу. В файле `ERPEngine.xml` (используемом на стороне ERP) строка подключения должна указывать внешний IP-адрес.

Для выгрузки `ERPEngine.xml` в диалоге «Интерфейс ERP-WMS» на вкладке «Настройки» нужно нажать кнопку «Выгрузить структуру сообщений» и выбрать вариант выгрузки в `.xml`.

## Автоматическая настройка интерфейса ERP-WMS

В системе EME.WMS предусмотрена утилита автоматической настройки интерфейса ERP-WMS. При нажатии кнопки «Автонастройка интерфейса» (диалог «Интерфейс ERP-WMS», вкладка «Настройки») автоматически формируются:

- Настройка диагностики и типа экспорта документов «EME-L».
- Событие «Импорт сообщений ERP».
- Событие «Выгрузка товаров в ERP».
- Событие «Выгрузка партий товаров в ERP».
- Событие «Выгрузка остатков товара в ERP».
- Событие «Сборщик мусора в интерфейсе ERP-WMS».
- Обработчик `RUN_EMEL_CLOSE_DOCUMENT`.
- Обработчик `RUN_EMEL_CLOSE_REPACKING`.
- Обработчик `RUN_EMEL_NEXT_ORDER_STATUS`.
- Настройка экспорта незакрытых документов.
- Настройка запрещенных транзакций.

**Важно:** Перед автонастройкой обязательно должна быть задана строка подключения к внешней базе данных.

После успешной автонастройки в лог-файле `CATASTRF.log` должны появиться сообщения об изменении параметров всех перечисленных событий.

## Ручная настройка экспорта документов в интерфейсе ERP-WMS

### Настройки модулей

Для ручной настройки интерфейса ERP-WMS в системе EME.WMS необходимо выставить настройки модулей:

- **EANCOM** — значение «Включено».
- **EANCOM.ErrorEmail** — почтовый адрес для получения диагностики ошибок в работе интерфейса.

### Тип экспорта документов

Необходимо проверить наличие типа экспорта «EMEL» в диалоге «Типы экспорта документов» (меню: «Классификаторы → EANCOM → Типы экспорта документов»). Если тип отсутствует — добавить его запуском EME-L доктора: класс — `ExportProtocolEMEL`, метод — `RegistryExportProtocolEMEL`.

### Экспорт незакрытых документов

Стандартная постановка документа в очередь экспорта происходит при закрытии документа. Для экспорта на каждой размещённой/отгруженной строке (например, при длительной приёмке) нужно в диалоге «Классификатор транзакций» (меню: «Классификаторы → Складские операции → Транзакции») на вкладке «Интерфейсы» выставить галочку «Разрешить экспорт незакрытых документов».

### Запрещённые транзакции

Для запрета экспорта на уровне отдельных транзакций нужно в диалоге «Типы экспорта документов» выбрать тип «EMEL», нажать «Параметры экспорта» и заполнить поле «Локальные параметры» перечнем запрещённых транзакций. Пример:

```
Запрещенные транзакции = 207, 210-219
```

## Настройка событий планировщика для интерфейса ERP-WMS

В системе EME.WMS события планировщика настраиваются через диалог «Навигатор канальной схемы» (меню: «Утилиты БД → Навигатор канальной схемы»). Все события имеют тип «Будильник», DLL — `MetC_Sys.dll`, функция реакции — `rf_run_EMEL`.

| Событие | Класс | Метод |
|---------|-------|-------|
| Импорт сообщений ERP | `ERPUtilities` | `ImportERPMessages` |
| Выгрузка товаров | `ERPUtilities` | `ExportGoodsItems` |
| Выгрузка партий | `ERPUtilities` | `ExportGoodsLots` |
| Выгрузка остатков товара | `ERPUtilities` | `ExportWarehouseRemains` |
| Сборщик мусора | `ERPUtilities` | `RunGarbageCollector` |

**Глубина сбора мусора** настраивается через параметр модулей «Предпочтения.Интеграция. Глубина сбора мусора». По умолчанию — 7 дней. Если значение равно 0, сообщения интерфейса ERP-WMS не удаляются никогда.

События планировщика также можно настроить через «Мастер настроек интерфейса ERP-WMS» (вкладка «Планировщик»).

---

## Настройка EME-L обработчиков для интерфейса ERP-WMS

EME-L обработчики настраиваются в диалоге «Организация-собственник» (меню: «Классификаторы → Разное → Организации»), вкладка «Дополнительные соответствия».

| Обработчик | Назначение | Класс | Метод |
|------------|-----------|-------|-------|
| `RUN_EMEL_CLOSE_DOCUMENT` | Закрытие документа | `ERPUtilities` | `OnCloseDocument` |
| `RUN_EMEL_CLOSE_REPACKING` | Закрытие переупаковки | `ERPUtilities` | `OnCloseRepacking` |
| `RUN_EMEL_NEXT_ORDER_STATUS` | Изменение статуса заказа | `ERPUtilities` | `OnNextOrderStatus` |

Для обработчика `RUN_EMEL_NEXT_ORDER_STATUS` указываются статусы: «Перемещается в пикинг/OA | Частично подобран | Подобран | Ушел со склада | Подтвержден | Закрыт».

Обработчики также настраиваются через «Мастер настроек интерфейса ERP-WMS» (вкладка «Очередь документов») с помощью галочек: «Документы», «Переупаковки», «Заказы на статусе».

## Синхронизация структуры промежуточной базы данных

Для формирования структуры таблиц промежуточной базы в системе EME.WMS:

1. Открыть диалог «Интерфейс ERP-WMS» (меню: «Утилиты БД → Импорт данных → Интерфейс ERP-WMS»).
2. Перейти на вкладку «Сообщения» и нажать кнопку «Формат».
3. Выбрать «Обновить структуру».

В результате отобразится сообщение «Структура внешней базы данных успешно обновлена» с указанием количества созданных таблиц. В SQL Server Management Studio в подключённой базе данных появятся таблицы, созданные при синхронизации.

## Базовое расширение интеграции 1С с EME.WMS

Базовое расширение интеграции 1С с EME.WMS обеспечивает взаимодействие между системой управления складом EME.WMS и системой управления производством 1C-ERP. Расширение работает через промежуточную SQL-базу данных посредством экспорта/импорта сообщений.

Расширение представляет собой дополнение к существующей конфигурации 1С, автоматически объединяемое с основной конфигурацией. Оно облегчает добавление нового функционала и последующую поддержку.

### Системные требования

Расширение совместимо с конфигурациями:

- 1C ERP версии 2.5
- 1C «Комплексная автоматизация» версии 2.5
- 1C «Управление торговлей» версии 11.5

### Установка расширения

1. Перейти в раздел «НСИ и администрирование» → «Администрирование» → «Расширения».
2. Нажать «Добавить из файла» и выбрать файл расширения (`.cf` / `.cfe`).
3. После установки расширение запросит перезайти в 1С.
4. Для доступа к основному диалогу: «НСИ и администрирование» → «Сервис» → «Интерфейс ERP-WMS».

### Первоначальная настройка расширения

**На стороне EME.WMS:**

1. В диалоге «Интерфейс ERP-WMS» на вкладке «Настройки» нажать «Выгрузить структуру сообщений» → «в файл ERPEngine.xml».

**На стороне 1С:**

1. Открыть интерфейс ERP-WMS, вкладка «Настройки», указать путь к файлу `ERPEngine.xml`.
2. Задать дату запуска интеграции (документы ранее этой даты не регистрируются и не выгружаются).
3. Проставить галочки «Обмен включен» и «Пометка к экспорту включена».
4. Через кнопки «Маппинг организаций» и «Маппинг складов» установить соответствие организаций и складов по единому коду.
5. Нажать «Установить структуру сообщений» для применения структуры из `ERPEngine.xml`.

На стороне EME.WMS настройка выполняется в одно действие: в диалоге «Интерфейс ERP-WMS» нажать «Автонастройка интерфейса».

### Экспорт и импорт данных

После первоначальной настройки доступен ручной экспорт/импорт данных для отладки. Для перехода в автоматический режим на вкладке «Утилиты» создаются и активируются регламентные задания.

**Предустановленные регламентные задания:**

- Раз в сутки выгружать остатки.
- Раз в сутки выполнять экспорт сообщений в EME.WMS.
- Раз в сутки выполнять импорт сообщений из EME.WMS.
- Фоновое проведение импортированных документов (приходные и расходные ордера).

Новые объекты 1С (справочники, контрагенты, документы) автоматически регистрируются к экспорту и выгружаются в EME.WMS.

### Первый запуск в продуктивной среде

При первом запуске справочники могут содержать десятки тысяч записей. Для оптимизации начальной выгрузки рекомендуется:

1. На вкладке «Остатки» в интерфейсе ERP-WMS произвести выгрузку остатков за последние 3 месяца (−1 квартал).
2. Повторить для следующих 3 месяцев (−2 квартала).

В результате формируется полная номенклатура, оборачивавшаяся за последние полгода, без значительных временных затрат. При запуске расширения формируются начальные остатки в EME.WMS и актуальная номенклатура товаров.

### Адаптация расширения под конкретные задачи

Все работы по адаптации производятся на основе документа «План интеграции» (описание информационных потоков между ERP и WMS).

**Пример:** В документе 1С «Перемещение товаров» отсутствуют марки, но на складе требуется работа с маркированным товаром. Приходные ордера на товары с марками поддерживаются.

**Два пути решения:**

1. **Использовать ордерную схему** (рекомендуется) — движение товаров фиксируется складскими документами «Приходный ордер на товары» и «Расходный ордер на товары». Это позволяет максимально детализировать процесс и использовать марки без значительных трудозатрат.

2. **Настроить функциональные опции экспорта/импорта** в 1С с последующей отладкой — значительно более трудоёмкий процесс.

При использовании ордерной схемы в системе EME.WMS процесс движения товаров становится прозрачным, а интеграция с 1C-WMS для маркированного товара работает без программных доработок в 1С. Если ордерная схема не используется, движение фиксируется накладными: «Приобретение товаров и услуг», «Реализация товаров и услуг», «Перемещение товаров».
